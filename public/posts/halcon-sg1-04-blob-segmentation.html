Blob Analysis 개요

Blob Analysis는 이미지에서 연결된 영역을 추출하고, 그 영역의 feature를 계산해 판단하는 방식이다. HALCON에서 가장 기본적이면서도 산업 현장에서 매우 많이 쓰이는 접근이다.

대표 흐름:

Image acquisition
→ ROI/domain 제한
→ preprocessing
→ segmentation
→ connection
→ region selection
→ feature extraction
→ decision

Blob 분석은 단순해 보이지만, 실제 성능은 segmentation 품질과 feature margin에 의해 결정된다.

Segmentation

Segmentation은 image를 object와 background로 나누는 단계이다.

2.1 Threshold

가장 기본적인 segmentation 방법이다.

threshold(Image, Region, MinGray, MaxGray)

장점:

  • 빠름
  • 직관적
  • parameter가 단순

한계:

  • 조명 변화에 취약
  • 표면 반사에 취약
  • 배경 gray가 겹치면 실패
  • 제품 색/재질 변화에 취약

실무 포인트

threshold 값은 "눈으로 보기 좋은 값"이 아니라 histogram margin으로 결정해야 한다.

background max < threshold < object min
margin이 작으면 흔들린다 이 margin이 작으면 현장에서는 반드시 흔들린다. 양품·불량 모두에서 background와 object의 gray 분포 사이 간격이 충분한지 histogram으로 확인하세요.

2.2 Adaptive / Dynamic Threshold

조명 불균일이나 배경 변화가 있을 때 사용한다.

개념:

local background estimate
→ original image와 비교
→ local contrast 기반 segmentation

적합한 경우:

  • 배경 밝기가 위치별로 다름
  • 표면 조명 gradient 존재
  • 작은 defect를 local contrast로 찾음

주의점:

  • window size 설정이 중요
  • defect 크기와 background 변화 scale을 구분해야 함
  • noise까지 검출할 수 있음

2.3 Color Segmentation

색상 기반 segmentation은 color space 선택이 중요하다.

  • RGB threshold
  • HSV hue/saturation
  • Lab color distance
  • learned classifier

실무에서는 조명 변동을 줄이기 위해 다음을 고려한다.

  • white balance 고정
  • flat-field correction
  • color calibration
  • diffuse illumination
  • reference patch 사용

Preprocessing

Segmentation 전 전처리는 안정성에 매우 중요하다.

3.1 Smoothing

Noise를 줄여 threshold 안정성을 높이다.

  • mean filter
  • gaussian filter
  • median filter
필터 특징
mean빠르지만 edge blur
gaussian자연스러운 smoothing
mediansalt-and-pepper noise에 강함
과도한 smoothing 주의 과도한 smoothing은 작은 defect를 제거할 수 있다.

3.2 Background Correction

조명 불균일을 보정한다.

방법:

  • flat-field correction
  • large scale smoothing 후 subtraction
  • shading correction
  • reference image division
표면 검사에서의 우선순위 표면 검사에서는 background correction이 defect detection보다 중요할 수 있다.

3.3 Morphology

Region 형태를 보정한다.

주요 연산:

  • opening: 작은 noise 제거
  • closing: 작은 hole 메움
  • dilation: 영역 확장
  • erosion: 영역 축소
  • fill_up: hole 채움

주의점

측정 목적에서의 morphology Morphology는 결과를 보기 좋게 만들 수 있지만, 실제 결함 크기를 왜곡할 수 있다. 측정 목적이라면 morphology 후 feature가 실제 물리량을 유지하는지 확인해야 한다.

Connected Components

Threshold 결과 region은 여러 연결 성분을 포함할 수 있다. connection을 통해 개별 blob으로 분리하다.

threshold
→ connection
→ select_shape

중요 feature:

  • area
  • row/column center
  • width/height
  • orientation
  • circularity
  • rectangularity
  • compactness
  • convexity

Feature Extraction

Blob 판단은 feature selection이 핵심이다.

5.1 Area

가장 많이 사용하는 feature이다.

사용 예:

  • 부품 존재 여부
  • 이물 크기
  • hole 크기
  • 누락 판단

주의점:

  • threshold에 매우 민감
  • 조명 변화에 따라 area가 변함
  • morphology 적용 시 실제 면적과 달라짐

5.2 Position

중심 좌표를 이용한다.

  • area_center
  • smallest rectangle
  • ellipse fitting
  • moments

부품 위치 판정에서는 center만 보는 것보다 orientation과 bounding box를 함께 보는 것이 좋다.

5.3 Shape Features

rectangularity사각형성chip, pad
compactness조밀도이물/파편 구분

select_shape 계열로 feature 조건에 맞는 region을 선택한다.

단일 조건보다 복합 조건이 안정적이다.

area range
+ circularity
+ position range

실무 판정 기준

threshold는 nominal 값이 아니라 양품/불량 데이터 분포로 잡아야 한다.

양품 feature 분포
불량 feature 분포
환경 변화 분포

Blob Analysis의 한계

Blob은 segmentation 기반이므로 다음 경우 약하다.

  • object와 background gray가 비슷함
  • 조명 변화가 큼
  • 표면 반사가 강함
  • object edge가 흐림
  • defect contrast가 낮음
  • 주변 구조와 연결됨

이 경우 대안:

  • edge/XLD 기반 분석
  • matching 기반 ROI 정렬
  • local contrast detection
  • texture feature
  • deep learning segmentation/anomaly

Blob과 Measuring의 차이

Blob은 영역 기반이다. Measuring은 edge 위치 기반이다.

항목 Blob Measuring
기준region area/shapeedge position
정밀도pixel 수준subpixel 가능
속도빠름빠름~중간
조명 의존threshold 의존edge contrast 의존
용도존재/개수/대략 형상폭/간격/치수
정밀 치수의 선택 정밀 치수는 blob area보다 edge/measure 기반이 적합하다.

핵심 정리

  • Blob Analysis는 HALCON의 기본이지만 segmentation 품질에 절대 의존한다.
  • threshold 값보다 gray 분포 margin이 중요하다.
  • morphology는 강력하지만 feature 왜곡 가능성을 항상 확인해야 한다.
  • area 하나만으로 판정하지 말고 shape/position/context feature를 함께 쓰는 것이 안정적이다.
  • Blob이 불안정하면 조명/광학/ROI부터 다시 봐야 한다.
설계 감각 Blob은 "쉬운 만큼 가장 먼저 고장 나는" 처리이기도 한다. 잘못된 segmentation을 후단 feature/rule로 보정하지 말고, 조명·광학·ROI 단계로 거슬러 올라가서 해결하는 습관이 안정적인 검사 시스템을 만든다.
