GenICam이란?

GenICam(Generic Interface for Cameras)은 EMVA(European Machine Vision Association)에서 제정한 산업용 카메라 제어 표준이다. 카메라 제조사에 관계없이 동일한 API와 Feature 이름으로 카메라를 제어할 수 있도록 설계되었다.

왜 GenICam이 필요한가?

GenICam 이전에는 카메라 제조사마다 독자적인 SDK와 API를 제공했다. Basler, FLIR, Hikvision, Daheng 등 각 제조사의 SDK를 따로 학습해야 했고, 카메라를 교체하면 소프트웨어를 처음부터 다시 작성해야 하는 경우가 많았다. GenICam은 이러한 제조사 종속(Vendor Lock-in) 문제를 해결하기 위해 탄생했다.

GenICam 이전

  • 제조사마다 다른 SDK 설치
  • 노출 시간 설정 함수 이름이 제각각
  • 카메라 교체 시 코드 전면 수정
  • 멀티 벤더 환경 구성 어려움

GenICam 이후

  • 표준 API로 모든 카메라 제어
  • ExposureTime 하나로 통일
  • 카메라 교체해도 코드 변경 최소
  • 서로 다른 제조사 카메라 혼용 가능
GenICam과 전송 표준의 관계 GenICam은 카메라 제어 방식에 대한 표준이지, 데이터 전송 방식에 대한 표준이 아니다. GigE Vision, USB3 Vision, Camera Link 등은 전송 표준이고, GenICam은 이들 위에서 동작하는 제어 계층이다. 즉, 어떤 전송 표준을 사용하든 GenICam으로 동일하게 카메라를 제어할 수 있다.

GenICam 모듈 구조

GenICam은 단일 규격이 아니라 4개의 핵심 모듈로 구성된 프레임워크다. 각 모듈은 독립적이면서도 서로 연동하여 카메라의 발견, 연결, 제어, 이미지 전송까지의 전 과정을 표준화한다.

그림 1. GenICam 프레임워크의 계층 구조 — 전송 표준 위에 제어 표준이 위치한다

GenAPI

GenICam의 핵심 엔진이다. 카메라가 제공하는 XML 디스크립션 파일을 파싱하여 Feature 트리를 구성하고, 애플리케이션이 Feature 값을 읽거나 쓸 수 있는 표준 인터페이스를 제공한다. 노드(Node) 기반으로 동작하며, 각 Feature는 IInteger, IFloat, IEnumeration, IBoolean, ICommand 등의 타입을 갖는다.

SFNC (Standard Features Naming Convention)

카메라 Feature의 이름과 의미를 표준화한 규약이다. 예를 들어 모든 GenICam 호환 카메라에서 노출 시간은 ExposureTime, 이미지 폭은 Width라는 이름으로 접근한다. 다음 섹션에서 자세히 다룬다.

GenTL (Generic Transport Layer)

카메라의 발견, 연결, 이미지 스트림 수신을 담당하는 전송 계층 표준이다. Producer(카메라/드라이버 측)와 Consumer(SDK/애플리케이션 측) 모델로 동작한다. GenTL Producer는 .cti 파일 형태로 배포되며, SDK는 이 파일을 로드하여 카메라에 접근한다.

GenTL Producer (.cti 파일) 각 전송 표준(GigE, USB3 등)에 대해 제조사가 GenTL Producer DLL을 제공한다. 환경변수 GENICAM_GENTL64_PATH에 .cti 파일 경로를 등록하면, GenICam 호환 SDK가 자동으로 해당 카메라를 검색할 수 있다.

GenCP (Generic Control Protocol)

카메라 레지스터에 대한 읽기/쓰기 프로토콜을 정의한다. GenAPI가 Feature 값을 변경하면, 실제로는 GenCP를 통해 카메라 레지스터에 바이트 단위로 데이터가 전달된다. Bootstrap Register(장치 정보, XML 파일 위치 등)도 GenCP로 접근한다.

SFNC (Standard Features Naming Convention)

SFNC는 GenICam에서 가장 실무에 직접적인 모듈이다. 카메라가 제공하는 기능(Feature)에 대해 표준화된 이름, 타입, 동작 방식을 정의한다. 개발자는 제조사에 관계없이 동일한 Feature 이름으로 카메라를 제어할 수 있다.

이미지 포맷 관련 Feature

Feature 이름 타입 설명 예시 값
Width IInteger 이미지 가로 해상도 (픽셀) 1920
Height IInteger 이미지 세로 해상도 (픽셀) 1080
OffsetX IInteger ROI 시작 X 좌표 0
OffsetY IInteger ROI 시작 Y 좌표 0
PixelFormat IEnumeration 픽셀 데이터 형식 Mono8, BayerRG8

획득 제어 관련 Feature

Feature 이름 타입 설명 예시 값
ExposureTime IFloat 노출 시간 (마이크로초) 10000.0
Gain IFloat 아날로그/디지털 게인 (dB) 6.0
AcquisitionMode IEnumeration 촬영 모드 Continuous, SingleFrame
AcquisitionStart ICommand 이미지 획득 시작 명령 Execute()
AcquisitionStop ICommand 이미지 획득 중지 명령 Execute()

트리거 관련 Feature

Feature 이름 타입 설명 예시 값
TriggerMode IEnumeration 트리거 모드 ON/OFF On, Off
TriggerSource IEnumeration 트리거 입력 소스 Line1, Software
TriggerActivation IEnumeration 트리거 엣지 방향 RisingEdge, FallingEdge
TriggerSoftware ICommand 소프트웨어 트리거 실행 Execute()
SFNC 버전 확인 SFNC는 버전이 올라가면서 Feature가 추가되거나 이름이 변경될 수 있다. 예를 들어 구버전에서는 ExposureTimeAbs였던 것이 SFNC 2.0부터 ExposureTime으로 변경되었다. 카메라의 SFNC 버전을 확인하는 것이 중요하다.

XML 디스크립션 파일

GenICam 호환 카메라는 자신이 지원하는 모든 Feature를 XML 파일로 기술한다. 이 XML 파일은 카메라 내부 플래시 메모리에 저장되어 있으며, SDK가 카메라에 연결할 때 자동으로 다운로드하여 파싱한다.

XML 파일의 역할

XML 디스크립션 파일은 카메라의 "설명서"와 같다. 어떤 Feature를 지원하는지, 각 Feature의 타입/범위/기본값은 무엇인지, Feature 간 의존 관계는 어떤지를 모두 기술한다. SDK는 이 XML을 파싱하여 Feature 트리를 구성하고, 애플리케이션에 동적으로 인터페이스를 제공한다.

그림 2. XML 디스크립션 파일의 동작 흐름 — 카메라에서 SDK로 Feature 정보가 전달된다

XML 파일의 구조를 간략히 살펴보면 다음과 같다:

<!-- GenICam XML 디스크립션 파일 예시 (간략화) -->
<RegisterDescription>
  <Integer Name="Width">
    <Value>1920</Value>
    <Min>1</Min>
    <Max>4096</Max>
    <Inc>1</Inc>
    <pAddress>WidthReg</pAddress>
  </Integer>

  <Float Name="ExposureTime">
    <Value>10000.0</Value>
    <Min>10.0</Min>
    <Max>1000000.0</Max>
    <Unit>us</Unit>
    <pAddress>ExposureTimeReg</pAddress>
  </Float>

  <Enumeration Name="TriggerMode">
    <EnumEntry Name="Off">
      <Value>0</Value>
    </EnumEntry>
    <EnumEntry Name="On">
      <Value>1</Value>
    </EnumEntry>
  </Enumeration>
</RegisterDescription>
XML 파일을 직접 수정하지 마세요 카메라의 XML 파일은 제조사가 작성한 것이며, 사용자가 임의로 수정하면 카메라가 정상 동작하지 않을 수 있다. Feature 값 변경은 반드시 GenAPI 인터페이스를 통해 수행해야 한다.

GenICam의 장점

코드 재사용

한 번 작성한 카메라 제어 코드를 다른 제조사 카메라에도 그대로 사용할 수 있다. Feature 이름이 SFNC로 통일되어 있기 때문이다.

제조사 전환 용이

프로젝트 중간에 카메라 제조사를 변경해도 소프트웨어 수정이 최소화된다. 단가, 성능, 납기 등의 이유로 제조사를 바꿔야 할 때 큰 장점이다.

표준화된 Feature 이름

ExposureTime, Gain, Width 등 직관적인 이름으로 모든 카메라를 제어한다. 새 카메라를 접해도 학습 시간이 짧다.

전송 표준 독립

GigE Vision, USB3 Vision, Camera Link 등 어떤 전송 표준을 사용하든 동일한 GenICam API로 카메라를 제어할 수 있다.

멀티 벤더 환경의 실제 효과

하나의 검사 라인에 Basler(GigE), Hikvision(USB3), Daheng(GigE) 카메라가 혼용되는 경우에도, GenICam 기반 SDK를 사용하면 동일한 코드 패턴으로 모든 카메라를 제어할 수 있다. 이는 유지보수 비용 절감과 개발 생산성 향상으로 직결된다.

실무 팁

Feature 접근 코드 예시

GenICam 기반 SDK에서 카메라 Feature에 접근하는 일반적인 패턴은 다음과 같다 (의사 코드):

// 1. 카메라 연결 및 NodeMap 획득
Camera cam = OpenCamera("DeviceSerialNumber");
NodeMap nodeMap = cam.GetNodeMap();

// 2. 이미지 크기 설정 (IInteger)
nodeMap.GetIntegerNode("Width").SetValue(1920);
nodeMap.GetIntegerNode("Height").SetValue(1080);

// 3. 노출 시간 설정 (IFloat, 단위: us)
nodeMap.GetFloatNode("ExposureTime").SetValue(10000.0);

// 4. 게인 설정 (IFloat, 단위: dB)
nodeMap.GetFloatNode("Gain").SetValue(6.0);

// 5. 트리거 모드 설정 (IEnumeration)
nodeMap.GetEnumNode("TriggerMode").SetValue("On");
nodeMap.GetEnumNode("TriggerSource").SetValue("Line1");

// 6. 촬영 시작 (ICommand)
nodeMap.GetCommandNode("AcquisitionStart").Execute();

// 7. 이미지 수신
Image img = cam.GetNextImage(timeout: 5000);

// 8. 촬영 종료
nodeMap.GetCommandNode("AcquisitionStop").Execute();
cam.Close();

주의사항

Feature 유효성 확인 모든 카메라가 모든 SFNC Feature를 지원하지는 않는다. Feature에 접근하기 전에 IsAvailable(), IsReadable(), IsWritable() 체크를 반드시 수행한다.
값 범위 확인 GetMin(), GetMax(), GetInc()로 설정 가능한 범위를 확인한 후 값을 설정한다. 범위를 벗어나면 예외가 발생한다.
Feature 설정 순서 일부 Feature는 다른 Feature에 의존한다. 예를 들어 TriggerSource는 TriggerMode가 "On"인 경우에만 설정할 수 있다. 의존 관계를 고려하여 순서대로 설정한다.
스트리밍 중 설정 변경 Width, Height, PixelFormat 등 일부 Feature는 스트리밍 중에 변경할 수 없다. AcquisitionStop 후 변경하고 다시 시작해야 한다.
제조사 고유 Feature SFNC에 정의되지 않은 제조사 고유 Feature도 존재한다. 이런 Feature는 이름 앞에 벤더 prefix가 붙거나 (예: BslExposureTimeMode), XML에서 직접 확인해야 한다.
GenICam Browser 활용 대부분의 머신비전 SDK는 GenICam Feature Browser(또는 Feature Explorer)를 제공한다. 이 도구를 사용하면 카메라가 지원하는 모든 Feature를 트리 형태로 탐색하고, 값을 실시간으로 변경할 수 있다. 코드를 작성하기 전에 Browser에서 먼저 Feature를 확인하는 습관을 들이면 개발 속도가 크게 향상된다.
GenICam 호환 주요 SDK Basler Pylon, MVTec HALCON, Matrox MIL, Cognex VisionPro, STEMMER IMAGING CVB 등 대부분의 상용 머신비전 SDK가 GenICam을 지원한다. 오픈소스로는 Aravis(Linux)가 있다.
