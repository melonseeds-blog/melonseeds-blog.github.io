ISP란

ISP (Image Signal Processor)

이미지 센서에서 출력된 Raw 데이터를 사람이 보기에 자연스러운 최종 이미지로 변환하는 처리 장치(또는 처리 과정). 센서가 빛을 전기 신호로 변환하면, ISP가 노이즈 제거, 색상 보정, 밝기 조정 등 일련의 처리를 수행하여 최종 RGB 이미지를 만든다.

이미지 센서의 각 픽셀은 빛의 양(밝기)만 측정할 수 있다. 컬러 이미지를 만들려면 각 픽셀 위에 컬러 필터(Bayer 패턴)를 올려서 R, G, B 중 하나의 색상만 통과시킨 뒤, 나머지 두 색상 정보는 주변 픽셀에서 보간(interpolation)한다. 이 과정을 포함한 전체 신호 처리 체계가 바로 ISP 파이프라인이다.

Raw 데이터란? 센서에서 바로 읽어낸 가공되지 않은 원시 데이터. 각 픽셀에 Bayer 패턴의 한 채널(R/G/B) 값만 담겨 있으며, 보통 10~14bit 깊이를 가진다. 이 상태에서는 화면에 표시해도 어둡고 색이 이상하게 보인다.

ISP 파이프라인 전체 흐름

ISP는 여러 단계의 처리 블록이 순서대로 연결된 파이프라인(Pipeline) 구조로 동작한다. 각 단계의 출력이 다음 단계의 입력이 되며, 순서가 중요하다.

그림 1. ISP 파이프라인의 전체 처리 흐름
파이프라인 순서가 중요한 이유 Black Level은 반드시 Demosaic 이전에 수행해야 한다. 오프셋이 남아있으면 보간 결과가 왜곡되기 때문이다. 마찬가지로 감마 보정은 색 보정(CCM) 이후에 적용해야 색 재현성이 유지된다.

Black Level 보정

이미지 센서는 빛이 전혀 없는 상태에서도 0이 아닌 값을 출력한다. 이를 암전류(Dark Current)에 의한 오프셋이라 하며, Black Level 보정은 이 오프셋을 제거하는 첫 번째 단계이다.

OB (Optical Black) 영역

센서의 가장자리에는 빛이 차단된 OB(Optical Black) 픽셀이 배치되어 있다. 이 픽셀들은 항상 어두운 상태이므로, 여기서 측정된 평균값이 곧 현재 온도와 조건에서의 Black Level이 된다.

그림 2. 센서 내 OB(Optical Black) 영역의 위치
암전류 (Dark Current) 열에 의해 전자가 자연 발생하는 현상. 온도가 높을수록 커진다. 노출 시간이 길수록 축적량이 늘어난다.
OB Clamp OB 영역의 평균값을 실시간으로 계산하여 유효 화소에서 빼는 하드웨어 회로. 프레임마다 자동 보정한다.
Pedestal 의도적으로 추가하는 작은 오프셋. Black Level 보정 후 값이 음수가 되는 것을 방지하기 위해 사용한다.

화이트밸런스 (AWB)

화이트밸런스 (White Balance)

조명의 색온도에 관계없이 흰색이 흰색으로 보이도록 R/G/B 채널의 게인을 조정하는 과정. 사람의 눈은 자동으로 색 순응(Chromatic Adaptation)을 하지만, 센서는 조명의 색온도가 그대로 반영되어 이미지 전체가 편향된 색을 띤다.

Gray World 가정

가장 간단한 AWB 알고리즘은 Gray World 가정이다. "장면 전체의 평균 색상은 회색(Gray)이다"라는 가정 아래, R/G/B 채널의 평균값이 같아지도록 게인을 조정한다.

Gray World 게인 계산

G 채널을 기준으로 R, B 채널의 게인을 계산한다:

  • Gain_R = Avg_G / Avg_R
  • Gain_B = Avg_G / Avg_B
  • Gain_G = 1.0 (기준 채널)

예: R 평균이 80, G 평균이 120, B 평균이 100이면 → Gain_R = 1.5, Gain_B = 1.2

수동 화이트밸런스

산업용 카메라에서는 조명 환경이 일정하므로 수동(Manual) 화이트밸런스를 더 자주 사용한다. 흰색 기준 타겟(White Reference)을 촬영하여 게인 값을 한 번 설정하면, 이후 동일 조건에서 안정적인 색상을 유지할 수 있다.

AWB의 함정 자동 화이트밸런스(AWB)는 장면의 색 분포가 균일하지 않으면 오동작한다. 예를 들어 빨간 제품만 가득한 장면에서는 AWB가 빨간색을 줄이려 하여 색이 왜곡된다. 산업용에서는 반드시 수동 설정을 권장한다.

색상 보정 매트릭스 (CCM)

CCM (Color Correction Matrix)

센서의 컬러 필터는 이상적인 R/G/B 응답 특성을 갖지 못한다. 각 필터의 파장 투과율이 겹치기 때문에 채널 간 색상 누설(crosstalk)이 발생한다. CCM은 이를 보정하기 위해 3x3 행렬을 적용하여 색 재현성을 높인다.

그림 3. CCM 3x3 행렬에 의한 색상 보정
CCM 캘리브레이션 CCM 계수는 ColorChecker 같은 표준 색 차트를 촬영한 뒤, 측정값과 기준값의 차이를 최소화하도록 최적화(Least Squares)하여 결정한다. 센서마다, 조명마다 다른 CCM이 필요하다.
행렬 성분 역할 일반적인 값 범위
대각 성분 (C11, C22, C33) 해당 채널의 게인. 1보다 크면 강화 1.0 ~ 2.0
비대각 성분 채널 간 crosstalk 보정. 음수가 일반적 -0.5 ~ 0.0
행 합 (Row Sum) 전체 밝기 유지를 위해 각 행의 합이 ~1.0 0.9 ~ 1.1

감마 보정

감마 보정 (Gamma Correction)

이미지 센서의 출력은 빛의 양에 비례하는 선형(Linear) 값이지만, 사람의 눈은 어두운 영역의 변화에 더 민감하다. 감마 보정은 이 차이를 보상하기 위해 비선형(Non-linear) 변환을 적용하여, 제한된 비트 수로도 인간의 시각 특성에 맞는 밝기 분포를 만든다.

sRGB 감마

가장 널리 쓰이는 표준은 sRGB 감마로, 대략 Output = Input^(1/2.2)의 관계를 가진다. 이 변환을 거치면 어두운 영역에 더 많은 코드값이 할당되어 인간의 시각 특성에 맞는 자연스러운 이미지가 된다.

그림 4. 감마 인코딩(1/2.2)과 디코딩(2.2) 커브

LUT (Look-Up Table)

감마 변환은 수학적으로 거듭제곱 연산이지만, 실시간 처리를 위해 대부분 LUT(Look-Up Table)로 구현한다. 입력값(예: 0~4095)을 인덱스로 사용하여 미리 계산된 출력값을 즉시 읽어오는 방식으로, 하드웨어 ISP에서는 1 클럭 만에 변환이 완료된다.

커스텀 LUT 활용 감마 LUT를 사용자가 직접 설정할 수 있는 카메라도 많다. 특정 밝기 구간의 대비를 높이거나, 비선형 센서 응답을 선형으로 교정하는 데 활용할 수 있다.

노이즈 제거와 샤프닝

Spatial NR (공간 노이즈 제거)

한 프레임 내에서 주변 픽셀들의 값을 참고하여 노이즈를 줄이는 방식이다. 대표적으로 Bilateral Filter가 있으며, 밝기 차이가 작은 이웃 픽셀만 평균에 포함시켜 엣지(Edge)는 보존하면서 노이즈를 제거한다.

Spatial NR

  • 한 프레임 내 공간적 필터링
  • Gaussian, Bilateral, NLM 등
  • 강하게 적용하면 디테일 손실
  • 구현이 비교적 단순

Temporal NR

  • 여러 프레임을 시간축으로 평균
  • 랜덤 노이즈가 상쇄됨
  • 움직이는 물체에 잔상 발생
  • 프레임 버퍼 메모리 필요

Unsharp Mask (샤프닝)

노이즈 제거 후 약간 흐려진 이미지의 선명도를 회복하기 위해 Unsharp Mask를 적용한다. 원리는 간단하다:

Unsharp Mask 공식
  1. 원본 이미지를 블러(Blur) 처리하여 저주파 성분을 추출
  2. 원본에서 블러 이미지를 빼면 고주파 성분(엣지)만 남음
  3. 이 고주파 성분을 원본에 다시 더하면 엣지가 강조됨

Sharpened = Original + Amount * (Original - Blurred)

NR과 샤프닝은 트레이드오프 관계 노이즈 제거를 강하게 하면 디테일이 손실되고, 샤프닝을 강하게 하면 노이즈까지 증폭된다. ISP 튜닝에서 가장 어려운 부분이 바로 이 두 파라미터의 균형을 맞추는 것이다.

산업용 vs 소비자용 ISP 차이

스마트폰이나 디지털 카메라는 "보기 좋은 사진"을 목표로 ISP의 모든 기능을 최대한 활용한다. 반면 산업용 머신비전에서는 "정확한 측정"이 목표이므로 ISP를 최소화하거나 완전히 비활성화하는 경우가 많다.

항목 소비자용 (스마트폰/카메라) 산업용 (머신비전)
목표 보기 좋은 이미지 정확한 측정/판정
ISP 사용 모든 단계 활성화, 자동 조정 최소화 또는 비활성화
출력 형식 JPEG/HEIF (8bit 압축) Raw / Mono8 / BayerRG8 등
화이트밸런스 AWB (자동) 수동 설정 또는 비활성화
감마 sRGB 감마 적용 Linear (감마 1.0) 선호
노이즈 제거 강하게 적용 (AI 기반 포함) 비활성화 또는 최소
샤프닝 적극 적용 비활성화

머신비전에서 Raw를 선호하는 이유

선형성 보존 감마가 적용되면 밝기와 픽셀값의 관계가 비선형이 되어, 밝기 기반 측정(예: 결함 검출 임계값)이 부정확해진다.
데이터 무손실 ISP 처리 과정에서 클리핑(Clipping)이나 반올림에 의한 정보 손실이 발생한다. Raw 데이터는 센서가 캡처한 원본 그대로이다.
재현성 AWB, NR 등 자동 처리는 장면에 따라 결과가 달라진다. 동일 조건에서 항상 같은 결과를 얻으려면 ISP를 고정하거나 꺼야 한다.
후처리 자유도 PC에서 소프트웨어로 필요한 처리만 선택적으로 적용할 수 있어 검사 알고리즘에 최적화된 이미지를 만들 수 있다.
ISP를 켜야 할 때도 있다 컬러 매칭 검사, 인쇄물 색상 검사 등에서는 ISP의 CCM과 화이트밸런스가 필수이다. 중요한 것은 목적에 맞게 ISP의 각 단계를 선택적으로 활성화/비활성화하는 것이다.
