시차 맵(Disparity Map)이란

시차 (Disparity)

스테레오 카메라의 좌측 이미지와 우측 이미지에서 동일한 물체가 나타나는 수평 위치 차이를 시차(disparity)라고 한다. 가까운 물체는 시차가 크고, 먼 물체는 시차가 작다. 시차 맵은 모든 픽셀에 대해 이 시차값을 계산한 2D 영상이다.

그림 1. 좌우 이미지에서의 시차와 시차 맵의 관계

시차 맵은 보통 StereoBM 또는 StereoSGBM 알고리즘으로 생성된다. 출력값은 정수(16배 스케일된 고정소수점) 또는 부동소수점으로 표현되며, 매칭에 실패한 픽셀은 일반적으로 -1 또는 0으로 표시된다.

StereoBM Block Matching. 빠르지만 텍스처 없는 영역에서 노이즈 발생. 실시간 처리에 적합.
StereoSGBM Semi-Global Block Matching. 경로 기반 비용 집계로 더 부드러운 시차 맵 생성. 품질 우수.
고정소수점 시차 OpenCV의 시차 맵은 16배 스케일(CV_16S). 실제 시차 = 출력값 / 16.0.

Disparity → Depth 변환

시차 값을 실제 깊이(거리)로 변환하는 공식은 스테레오 기하학에서 유도된다. 핵심 공식은 다음과 같다:

깊이 변환 공식

Z = f × B / d

Z 카메라로부터 물체까지의 깊이(거리), 단위: mm 또는 m
f 카메라의 초점 거리(focal length), 단위: 픽셀
B 두 카메라 간 간격(baseline), 단위: mm 또는 m
d 시차(disparity), 단위: 픽셀

역수 관계 — 시차와 깊이

깊이 Z는 시차 d에 반비례한다. 이 관계는 깊이 정밀도에 직접적인 영향을 미친다: 가까운 거리에서는 시차가 크므로 정밀하고, 먼 거리에서는 시차가 작아 정밀도가 급격히 떨어진다.

그림 2. 시차와 깊이의 반비례(쌍곡선) 관계

단위 변환 주의사항

단위 일관성이 핵심 초점 거리 f는 픽셀 단위이고, 베이스라인 B는 실세계 단위(mm)이다. 이 경우 Z의 단위는 B와 동일한 mm가 된다. 캘리브레이션 결과의 단위를 반드시 확인하고, f가 mm 단위인 경우 f_px = f_mm / pixel_size로 변환해야 한다.

Q 행렬과 재투영

OpenCV의 stereoRectify 함수는 스테레오 캘리브레이션 후 Q 행렬(Disparity-to-depth mapping matrix)을 출력한다. 이 4x4 행렬을 이용하면 시차 맵의 각 픽셀을 3D 좌표로 일괄 변환할 수 있다.

Q 행렬의 구조

reprojectImageTo3D 함수

cv::reprojectImageTo3D 함수는 시차 맵과 Q 행렬을 입력으로 받아, 각 픽셀의 3D 좌표 (X, Y, Z)를 담은 3채널 영상을 출력한다.

// OpenCV: 시차 맵 → 3D 포인트
cv::Mat disparity, points3D;
cv::Mat Q;  // stereoRectify에서 얻은 Q 행렬

// 시차 맵을 float로 변환 (16배 스케일 제거)
disparity.convertTo(disparity, CV_32F, 1.0/16.0);

// 3D 재투영
cv::reprojectImageTo3D(disparity, points3D, Q, true);

// points3D: 각 픽셀에 (X, Y, Z) 좌표 저장
// handleMissingValues=true → 유효하지 않은 시차는 매우 큰 Z값으로 설정
좌표계 방향 OpenCV 스테레오 좌표계는 좌측 카메라 기준이다. X는 오른쪽, Y는 아래쪽, Z는 카메라 전방(물체 방향)이 양의 방향이다. 이 좌표계는 로봇 비전이나 PCL 등 외부 라이브러리의 좌표계와 다를 수 있으므로 변환이 필요할 수 있다.

노이즈 제거와 필터링

시차 맵에서 생성된 포인트 클라우드에는 매칭 실패, 오클루전(가려짐), 반사 등으로 인한 노이즈 포인트가 포함된다. 유용한 3D 데이터를 얻으려면 후처리 필터링이 필수적이다.

유효 시차 범위 필터링

유효 범위 제한

시차 값이 0 이하이거나 numDisparities를 초과하는 픽셀은 매칭 실패이므로 제거한다. 또한 Z 값이 물리적으로 의미 없는 범위(음수, 너무 가까움, 너무 멀음)도 필터링 대상이다.

# Python: 유효 시차 및 깊이 범위 필터링
import numpy as np

# 유효 시차 마스크
mask = (disparity > 0) & (disparity < num_disparities)

# 깊이 범위 제한 (예: 200mm ~ 5000mm)
Z = points3D[:, :, 2]
depth_mask = (Z > 200) & (Z < 5000)

# 결합 마스크 적용
valid_mask = mask & depth_mask
filtered_points = points3D[valid_mask]
filtered_colors = color_image[valid_mask]

통계적 아웃라이어 제거 (SOR)

Statistical Outlier Removal은 각 포인트의 k-최근접 이웃까지의 평균 거리를 계산하고, 전체 평균에서 표준편차의 n배를 초과하는 포인트를 제거하는 방법이다.

그림 4. 통계적 아웃라이어 제거 — 평균 이웃 거리 기준으로 이상치 제거
WLS 필터 (Weighted Least Squares) OpenCV의 ximgproc::createDisparityWLSFilter를 사용하면 시차 맵 단계에서 좌우 일관성 검사와 에지 보존 스무딩을 동시에 수행할 수 있다. 포인트 클라우드 생성 전에 적용하면 후처리 부담을 크게 줄일 수 있다.

Open3D 시각화 예제

import open3d as o3d
import numpy as np

# 포인트 클라우드 생성
pcd = o3d.geometry.PointCloud()
pcd.points = o3d.utility.Vector3dVector(filtered_points)  # Nx3
pcd.colors = o3d.utility.Vector3dVector(filtered_colors / 255.0)  # 0~1 정규화

# 통계적 아웃라이어 제거
pcd_clean, idx = pcd.remove_statistical_outlier(
    nb_neighbors=20,   # k-최근접 이웃 수
    std_ratio=2.0       # 표준편차 배수 임계값
)

# 시각화
o3d.visualization.draw_geometries([pcd_clean],
    window_name="Point Cloud",
    width=1280, height=720)

# PLY 파일로 저장
o3d.io.write_point_cloud("output.ply", pcd_clean)
CloudCompare 활용 팁 CloudCompare에서 PLY 파일을 열면 좌측 DB Tree에서 포인트 수, 바운딩 박스, 색상 정보를 즉시 확인할 수 있다. Edit > Scalar fields > Compute statistical parameters로 포인트 분포를 분석하면 노이즈 수준을 빠르게 파악할 수 있다.

정밀도 한계

스테레오 비전의 깊이 정밀도는 무한하지 않다. Z = f*B/d 공식에서 시차 d가 1픽셀 변할 때 깊이 Z가 얼마나 변하는지를 분석하면 정밀도의 한계를 이해할 수 있다.

거리에 따른 깊이 해상도

깊이 해상도 공식

시차 1픽셀 변화에 대한 깊이 변화량(깊이 해상도)은 다음과 같다:

deltaZ = Z^2 / (f * B)

거리 Z의 제곱에 비례하므로, 거리가 2배가 되면 깊이 오차는 4배로 증가한다. 이것이 스테레오 비전이 근거리 측정에 적합한 근본적인 이유이다.

거리 Z (m) 시차 d (px) deltaZ (mm) 상대 오차
0.5 120 0.4 0.08%
1.0 60 1.7 0.17%
2.0 30 6.7 0.33%
5.0 12 41.7 0.83%
10.0 6 166.7 1.67%

* f=500px, B=120mm 기준 예시

서브픽셀 시차의 중요성

정수 픽셀 시차만으로는 깊이 해상도가 제한된다. 서브픽셀(sub-pixel) 시차 추정을 통해 0.1~0.25 픽셀 수준의 정밀도를 확보하면 깊이 해상도가 4~10배 향상된다.

정수 시차 (Integer Disparity)

  • 최소 변화량: 1 픽셀
  • 2m 거리에서 deltaZ = 6.7mm
  • 계단형 깊이 아티팩트 발생
  • 처리 속도 빠름

서브픽셀 시차 (Sub-pixel Disparity)

  • 최소 변화량: 0.1~0.25 픽셀
  • 2m 거리에서 deltaZ = 0.7~1.7mm
  • 부드러운 깊이 전이
  • 포물선/곡선 피팅 필요
베이스라인과 해상도의 트레이드오프 베이스라인 B를 늘리면 깊이 정밀도가 좋아지지만, 좌우 이미지의 외관 차이(오클루전)가 증가하여 매칭 성공률이 떨어진다. 측정 거리와 정밀도 요구사항에 맞춰 적절한 베이스라인을 선택해야 한다.
실무 가이드라인 일반적으로 베이스라인은 측정 거리의 1/30 ~ 1/10 정도로 설정한다. 예를 들어 1m 거리를 측정하려면 30~100mm 베이스라인이 적절하다. 서브픽셀 시차와 적절한 베이스라인을 조합하면 대부분의 산업용 3D 측정 요구를 충족할 수 있다.
