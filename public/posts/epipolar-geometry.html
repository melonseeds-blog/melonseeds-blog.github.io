에피폴라 기하학이란

에피폴라 기하학(Epipolar Geometry)은 두 대의 카메라가 같은 장면을 서로 다른 시점에서 촬영했을 때, 두 이미지 사이의 기하학적 관계를 설명하는 이론이다. 스테레오 비전 시스템에서 3D 복원의 수학적 기반이 되며, 대응점(Correspondence) 탐색을 2D 영역에서 1D 직선으로 축소하여 계산 효율을 극적으로 높여준다.

왜 에피폴라 기하학이 중요한가?

스테레오 매칭에서 왼쪽 이미지의 한 점에 대응하는 오른쪽 이미지의 점을 찾으려면, 이론적으로 이미지 전체를 탐색해야 한다. 에피폴라 기하학을 활용하면 탐색 범위를 하나의 직선(에피폴라 라인)으로 제한할 수 있어, 탐색 비용이 O(N^2)에서 O(N)으로 줄어든다.

사전 지식 이 글을 이해하려면 카메라 캘리브레이션(내부 파라미터, 외부 파라미터)과 핀홀 카메라 모델에 대한 기본 개념이 필요하다. 동차 좌표(Homogeneous Coordinates)도 수식에서 자주 등장한다.

에피폴 / 에피폴라 라인 / 에피폴라 평면

핵심 용어 정의

에피폴 (Epipole) 한쪽 카메라의 광학 중심(Optical Center)이 반대쪽 이미지 평면에 투영된 점. 왼쪽 카메라의 중심이 오른쪽 이미지에 투영되면 e', 반대는 e로 표기한다.
에피폴라 라인 (Epipolar Line) 3D 공간의 한 점 P와 두 카메라 중심을 잇는 에피폴라 평면이 각 이미지 평면과 만드는 교선. 왼쪽 이미지의 점 p에 대응하는 오른쪽 이미지의 점 p'는 반드시 이 직선 위에 존재한다.
에피폴라 평면 (Epipolar Plane) 3D 점 P와 두 카메라의 광학 중심 O, O'가 이루는 삼각형이 정의하는 평면. 모든 에피폴라 라인은 에피폴을 지나며, 에피폴라 평면의 수에 따라 무한히 많은 에피폴라 라인이 존재한다.
베이스라인 (Baseline) 두 카메라의 광학 중심 O와 O'를 잇는 선분. 모든 에피폴라 평면은 베이스라인을 포함한다.

에피폴라 기하학 다이어그램

그림 1. 에피폴라 기하학의 핵심 구조 -- 3D 점 P, 두 카메라 중심 O/O', 에피폴, 에피폴라 라인, 에피폴라 평면
핵심 성질 모든 에피폴라 라인은 해당 이미지의 에피폴을 지난다. 즉, 에피폴은 에피폴라 라인들의 교차점(Pencil of Lines)이다. 3D 점 P의 위치가 바뀌면 에피폴라 평면이 회전하고, 이에 따라 에피폴라 라인도 변하지만 항상 에피폴을 통과한다.

기본 행렬 (Fundamental Matrix, F)

기본 행렬 F는 두 이미지 사이의 에피폴라 기하학을 픽셀 좌표계에서 표현하는 3x3 행렬이다. 카메라의 내부 파라미터(intrinsic)와 외부 파라미터(extrinsic) 정보를 모두 포함하며, 캘리브레이션 없이도 대응점 쌍만으로 추정할 수 있다.

기본 행렬의 정의
x'T F x = 0
x, x'는 왼쪽/오른쪽 이미지의 대응점 (동차 픽셀 좌표)

이 수식은 오른쪽 이미지의 점 x'가 왼쪽 이미지의 점 x에 의해 결정되는 에피폴라 라인 l' = Fx 위에 놓인다는 것을 의미한다.

F 행렬의 주요 성질

성질 설명
크기 3 x 3 행렬
랭크 rank 2 (특이 행렬, det(F) = 0)
자유도 7 (9개 원소 - 스케일 1 - rank 제약 1)
에피폴 관계 Fe = 0, FTe' = 0 (에피폴은 F의 널 공간)
에피폴라 라인 l' = Fx (오른쪽), l = FTx' (왼쪽)
추정 방법 8-Point Algorithm, RANSAC + 7-Point 등
8-Point Algorithm 최소 8쌍의 대응점으로 F를 추정하는 고전적 방법이다. 각 대응점 쌍 (x, x')로부터 하나의 선형 방정식을 만들고, 8개 이상의 방정식을 SVD로 풀어 F를 구한다. 실무에서는 노이즈에 강건한 Normalized 8-Point나 RANSAC을 함께 사용한다.

필수 행렬 (Essential Matrix, E)

필수 행렬 E는 에피폴라 기하학을 정규화된 카메라 좌표계(normalized image coordinates)에서 표현하는 3x3 행렬이다. 카메라의 내부 파라미터 K가 이미 알려진(캘리브레이션 완료) 상태에서 사용하며, 순수하게 두 카메라 간의 회전(R)과 이동(t) 정보만 담고 있다.

E와 F의 관계
E = K'T F K
K, K'는 왼쪽/오른쪽 카메라의 내부 파라미터 행렬
E = [t]x R = S R
[t]x는 이동 벡터 t의 반대칭 행렬(Skew-symmetric), R은 회전 행렬

기본 행렬 (F)

  • 픽셀 좌표계에서 동작
  • 캘리브레이션 불필요
  • 자유도: 7
  • 최소 7~8쌍 대응점 필요
  • 내부+외부 파라미터 혼합

필수 행렬 (E)

  • 정규화 좌표계에서 동작
  • 캘리브레이션 필수 (K 필요)
  • 자유도: 5
  • 최소 5쌍 대응점 필요
  • 순수 외부 파라미터 (R, t)
실무 선택 기준 캘리브레이션된 스테레오 카메라를 사용하면 E를 직접 구하는 것이 정확도가 높다. 캘리브레이션 없이 특징점 매칭만으로 기하학을 추정해야 하는 경우(예: SfM, Visual SLAM)에는 F를 먼저 구한 뒤 E로 변환한다.

에피폴라 제약 (Epipolar Constraint)

에피폴라 제약은 스테레오 비전의 가장 강력한 기하학적 제약 조건이다. 이 제약을 통해 2D 이미지 영역 전체 탐색을 1D 직선(에피폴라 라인) 위 탐색으로 축소할 수 있다.

에피폴라 제약의 의미
x'T F x = 0   또는   x̂'T E x̂ = 0
왼쪽 이미지의 점 x에 대응하는 오른쪽 이미지의 점 x'는 에피폴라 라인 l' = Fx 위에 반드시 존재한다

탐색 공간 축소 효과

항목 에피폴라 제약 미적용 에피폴라 제약 적용
탐색 공간 이미지 전체 (2D) 에피폴라 라인 (1D)
연산 복잡도 O(W x H) = O(N^2) O(W) = O(N)
1920x1080 예시 ~2,073,600 픽셀 탐색 ~1,920 픽셀 탐색
속도 향상 기준 약 1000배 빠름
오매칭 가능성 높음 (넓은 탐색 영역) 낮음 (제한된 후보)
주의: 에피폴라 제약의 한계 에피폴라 제약은 대응점이 어느 직선 위에 있는지만 알려줄 뿐, 직선 위 어디에 있는지는 알려주지 않는다. 따라서 에피폴라 라인 위에서 추가적인 매칭 알고리즘(SAD, NCC, Census Transform 등)이 여전히 필요하다.

정류 (Rectification)

정류(Rectification)는 두 카메라의 이미지를 변환하여 모든 에피폴라 라인이 수평이 되도록 만드는 과정이다. 정류 후에는 대응점 탐색이 같은 행(row)에서만 이루어지므로 구현이 매우 단순해진다.

정류 전후 비교

그림 2. 정류(Rectification) 전후 비교 -- 에피폴라 라인이 수평으로 정렬되어 탐색이 단순해진다

정류 방법

캘리브레이션 기반 정류

  • 카메라 내부/외부 파라미터 사용
  • R, t로부터 정류 회전 행렬 계산
  • cv::stereoRectify() 사용
  • 가장 정확하고 안정적
  • 사전 캘리브레이션 필수

비캘리브레이션 정류

  • F 행렬로부터 호모그래피 계산
  • 대응점 쌍만으로 정류 가능
  • cv::stereoRectifyUncalibrated()
  • 캘리브레이션 없이 사용 가능
  • 정확도가 상대적으로 낮음

OpenCV 실무 코드

OpenCV에서 에피폴라 기하학을 다루는 핵심 함수들을 정리한다. 아래 코드는 스테레오 이미지에서 기본 행렬 추정, 에피폴라 라인 시각화, 정류까지의 전체 파이프라인을 보여준다.

기본 행렬 추정 및 에피폴라 라인 시각화

import cv2
import numpy as np

# 1. 특징점 검출 및 매칭
sift = cv2.SIFT_create()
kp1, des1 = sift.detectAndCompute(img_left, None)
kp2, des2 = sift.detectAndCompute(img_right, None)

bf = cv2.BFMatcher()
matches = bf.knnMatch(des1, des2, k=2)

# Lowe's ratio test
good = []
for m, n in matches:
    if m.distance < 0.7 * n.distance:
        good.append(m)

pts1 = np.float32([kp1[m.queryIdx].pt for m in good])
pts2 = np.float32([kp2[m.trainIdx].pt for m in good])

# 2. 기본 행렬 추정 (RANSAC)
F, mask = cv2.findFundamentalMat(pts1, pts2, cv2.FM_RANSAC, 1.0, 0.99)
pts1 = pts1[mask.ravel() == 1]
pts2 = pts2[mask.ravel() == 1]

# 3. 에피폴라 라인 계산 및 시각화
lines2 = cv2.computeCorrespondEpilines(pts1.reshape(-1, 1, 2), 1, F)
lines2 = lines2.reshape(-1, 3)

for r, pt1, pt2 in zip(lines2, pts1, pts2):
    color = tuple(np.random.randint(0, 255, 3).tolist())
    x0, y0 = map(int, [0, -r[2] / r[1]])
    x1, y1 = map(int, [w, -(r[2] + r[0] * w) / r[1]])
    img_right_lines = cv2.line(img_right_lines, (x0, y0), (x1, y1), color, 1)
    img_right_lines = cv2.circle(img_right_lines, tuple(map(int, pt2)), 5, color, -1)

캘리브레이션 기반 정류

# 캘리브레이션 결과가 있는 경우 정류
R1, R2, P1, P2, Q, roi1, roi2 = cv2.stereoRectify(
    K1, D1, K2, D2,
    image_size,
    R, T,
    alpha=0  # 0: 유효 영역만, 1: 전체 보존
)

# 리맵 테이블 생성
map1x, map1y = cv2.initUndistortRectifyMap(K1, D1, R1, P1, image_size, cv2.CV_32FC1)
map2x, map2y = cv2.initUndistortRectifyMap(K2, D2, R2, P2, image_size, cv2.CV_32FC1)

# 정류 적용
rect_left  = cv2.remap(img_left,  map1x, map1y, cv2.INTER_LINEAR)
rect_right = cv2.remap(img_right, map2x, map2y, cv2.INTER_LINEAR)

# 정류 검증: 수평선 그리기
for y in range(0, rect_left.shape[0], 40):
    cv2.line(rect_left,  (0, y), (rect_left.shape[1], y), (0, 255, 0), 1)
    cv2.line(rect_right, (0, y), (rect_right.shape[1], y), (0, 255, 0), 1)

필수 행렬 추정 및 포즈 복원

# 카메라 내부 파라미터가 알려진 경우
E, mask = cv2.findEssentialMat(pts1, pts2, K, method=cv2.RANSAC, prob=0.999, threshold=1.0)

# E로부터 R, t 복원 (4가지 해 중 유효한 1개 선택)
_, R, t, mask_pose = cv2.recoverPose(E, pts1, pts2, K)

print(f"회전 행렬 R:\n{R}")
print(f"이동 벡터 t:\n{t.ravel()}")

# E와 F 변환 검증
K_inv = np.linalg.inv(K)
F_from_E = K_inv.T @ E @ K_inv
print(f"F (from E):\n{F_from_E / F_from_E[2,2]}")

OpenCV 핵심 함수 정리

함수 입력 출력 용도
findFundamentalMat 대응점 쌍 F, inlier mask 기본 행렬 추정
findEssentialMat 대응점 + K E, inlier mask 필수 행렬 추정
recoverPose E + 대응점 + K R, t E에서 포즈 복원
computeCorrespondEpilines 점 + F 에피폴라 라인 계수 에피폴라 라인 계산
stereoRectify K, D, R, T R1, R2, P1, P2, Q 캘리브레이션 기반 정류
stereoRectifyUncalibrated 대응점 + F H1, H2 비캘리브레이션 정류
