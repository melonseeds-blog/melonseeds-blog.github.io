센서 선정이 중요한 이유

머신비전 검사 시스템의 성능은 광학 설계와 센서 선택에서 80% 이상이 결정된다. 아무리 뛰어난 알고리즘을 적용해도 이미지 자체의 품질이 부족하면 검출률을 높이는 데 한계가 있다.

센서가 결정하는 것들

센서는 단순히 "사진을 찍는 부품"이 아니다. 센서의 해상도, 픽셀 크기, 셔터 방식, 프레임 레이트, 감도, 노이즈 수준이 검사의 분해능, 속도, 정확도를 직접 좌우한다. 렌즈가 빛을 모아주는 역할이라면, 센서는 그 빛을 얼마나 정밀하게 전기 신호로 변환하느냐를 결정한다.

그림 1. 머신비전 시스템 성능에서 광학+센서의 비중
흔한 실수: 알고리즘으로 해결하려는 시도 조명이 잘못되었거나 센서 해상도가 부족한 상태에서 소프트웨어 보정으로 해결하려는 시도는 비용과 시간만 낭비한다. "쓰레기가 들어가면 쓰레기가 나온다(Garbage In, Garbage Out)"는 원칙이 머신비전에도 그대로 적용된다.

해상도 계산

센서 해상도를 결정하려면 먼저 FOV(Field of View)와 필요 분해능을 파악해야 한다. 이 두 값으로부터 최소 필요 해상도를 산출할 수 있다.

기본 공식

최소 해상도 산출 공식
최소 픽셀 수 (H) = FOV (mm) / 필요 분해능 (mm/pixel)
최소 해상도 = H_width x H_height
FOV 카메라가 한 번에 촬영하는 실제 영역의 크기 (mm). 검사 대상 + 여유 마진 포함.
필요 분해능 검출해야 할 최소 결함/특징의 크기. 일반적으로 최소 결함 크기의 1/3~1/5을 1 pixel로 설정.
나이퀴스트 조건 최소 2 pixel 이상으로 결함을 표현해야 안정적 검출이 가능. 실무에서는 3~5 pixel 권장.

계산 예시

PCB 납땜 검사 해상도 계산

조건: FOV = 50mm x 40mm, 최소 결함 크기 = 0.1mm, 결함당 3 pixel 필요

계산:

  • 필요 분해능 = 0.1mm / 3 = 0.033 mm/pixel
  • 수평 픽셀 = 50mm / 0.033 = 1,515 pixel
  • 수직 픽셀 = 40mm / 0.033 = 1,212 pixel
  • 최소 해상도: 1,515 x 1,212 → 실제 선택: 2 MP (1920 x 1200)
여유 마진 확보 계산값보다 20~30% 높은 해상도를 선택하는 것이 안전하다. 렌즈의 주변부 해상도 저하, 조립 공차, 워킹 디스턴스 변화 등을 고려해야 하기 때문이다.

픽셀 크기 선택

센서의 픽셀 크기(Pixel Pitch)는 해상도와 감도 사이의 트레이드오프를 결정하는 핵심 파라미터다. 같은 센서 크기에서 픽셀을 작게 만들면 해상도는 올라가지만 각 픽셀이 받는 광량이 줄어 감도가 떨어진다.

작은 픽셀 (2.4~3.45 um)

  • 높은 해상도 — 같은 센서 크기에서 더 많은 픽셀
  • 작은 렌즈 — 소형 C-mount 렌즈 사용 가능
  • 낮은 감도 — 광량 부족 시 노이즈 증가
  • 적합: 충분한 조명 확보 가능한 정밀 검사

큰 픽셀 (4.5~7.0 um)

  • 높은 감도 — 넓은 수광 면적, 낮은 노이즈
  • 넓은 Dynamic Range — 밝고 어두운 영역 동시 표현
  • 낮은 해상도 — 같은 크기에서 적은 픽셀 수
  • 적합: 저조도, 고속 촬영, 높은 대비 필요
픽셀 크기 대표 센서 해상도(1" 기준) 감도 주요 용도
2.74 um Sony IMX532 ~20 MP 보통 고해상도 정밀 검사
3.45 um Sony IMX253 ~12 MP 양호 범용 고해상도 검사
4.5 um Sony IMX264 ~5 MP 우수 고감도 범용 검사
5.86 um ON Semi PYTHON 5000 ~5 MP 우수 고속 라인/에어리어
센서 크기와 렌즈 호환성 픽셀이 작아지면 같은 해상도에서 센서가 작아져 소형 렌즈(C-mount)를 쓸 수 있지만, 높은 해상도를 원하면 센서가 커져서 F-mount나 M42 렌즈가 필요할 수 있다. 렌즈의 이미지 서클이 센서보다 커야 주변부 비네팅이 발생하지 않는다.

프레임 레이트와 대역폭

센서 해상도와 프레임 레이트를 곱하면 초당 생성되는 데이터량이 나오고, 이것이 카메라 인터페이스의 대역폭 한계를 초과하면 안 된다.

필요 대역폭 계산
대역폭 (MB/s) = 해상도 (pixel) x FPS x 비트 깊이 (byte) / 1,000,000

인터페이스별 대역폭 한계

인터페이스 이론 대역폭 실효 대역폭 5MP 8bit 최대 FPS 12MP 8bit 최대 FPS
GigE Vision 1 Gbps ~115 MB/s ~23 fps ~9 fps
USB3 Vision 5 Gbps ~380 MB/s ~76 fps ~31 fps
5GigE 5 Gbps ~570 MB/s ~114 fps ~47 fps
10GigE 10 Gbps ~1,100 MB/s ~220 fps ~91 fps
CoaXPress 2.0 12.5 Gbps ~1,500 MB/s ~300 fps ~125 fps
ROI로 대역폭 절약 전체 해상도가 필요하지 않은 경우 ROI(Region of Interest)를 설정하여 일부 영역만 전송하면 프레임 레이트를 크게 높일 수 있다. 예를 들어 5MP 센서에서 중앙 1MP 영역만 읽으면 GigE에서도 100fps 이상이 가능하다.

실전 계산 예시

대역폭 계산 예시

조건: 5MP 카메라 (2592 x 2048), 30fps, 8bit mono

  • 프레임당 데이터 = 2,592 x 2,048 x 1 byte = 5,308,416 byte = 5.06 MB
  • 초당 데이터 = 5.06 MB x 30 = 151.8 MB/s
  • GigE (115 MB/s): 부족 → USB3 Vision (380 MB/s): 충분

모노 vs 컬러

머신비전에서는 모노(Monochrome) 센서가 기본이다. 컬러 센서는 특별한 이유가 있을 때만 선택한다.

모노 센서

  • 높은 감도 — 컬러 필터 없이 빛을 100% 수광
  • 높은 해상도 — 1 pixel = 1 데이터, 베이어 보간 불필요
  • 빠른 처리 — 데이터 1/3, 디모자이킹 불필요
  • 정확한 측정 — 색수차 영향 없음
  • 주 용도: 치수 측정, 결함 검출, 위치 정렬

컬러 센서

  • 색상 정보 — 색으로만 구분 가능한 결함 검출
  • 낮은 실효 해상도 — 베이어 패턴으로 실제 해상도 감소
  • 낮은 감도 — RGB 필터가 빛의 약 2/3를 차단
  • 복잡한 조명 — 백색 조명 필요, 색온도 관리
  • 주 용도: 색상 분류, 라벨 인쇄 검사, 식품 검사
그림 2. 모노 센서와 컬러(베이어) 센서의 픽셀 구조 차이
실무 판단 기준 "이 검사를 흑백 이미지로 할 수 있는가?"를 먼저 자문하라. 할 수 있다면 모노를 선택하는 것이 해상도, 감도, 속도, 비용 모든 면에서 유리하다. 색상 차이가 유일한 검출 단서인 경우에만 컬러를 선택한다.

센서 제조사와 주요 모델

Sony Pregius / Pregius S

머신비전 시장의 사실상 표준(de facto standard). 글로벌 셔터 CMOS 센서로, Pregius는 Front-Side Illumination(FSI), Pregius S는 Back-Side Illumination(BSI) 기술을 적용하여 감도를 대폭 향상시켰다.

모델 세대 해상도 픽셀 크기 특징
IMX249 Pregius 1세대 2.3 MP 5.86 um 고감도, 범용 입문
IMX253 Pregius 2세대 12.3 MP 3.45 um 고해상도 산업 표준
IMX264 Pregius 2세대 5.0 MP 3.45 um 범용 고성능
IMX532 Pregius S (BSI) 20.4 MP 2.74 um 초고해상도 + 고감도
IMX542 Pregius S (BSI) 5.0 MP 2.74 um 소형 고해상도

ON Semiconductor (onsemi)

PYTHON 시리즈와 XGS 시리즈로 머신비전 시장에서 Sony에 이어 두 번째로 많이 사용된다. 특히 고속 촬영에 강점이 있다.

PYTHON 시리즈 글로벌 셔터, 5.86um 픽셀. PYTHON 300(VGA)~PYTHON 25000(25MP)까지. 고속 프레임 레이트.
XGS 시리즈 PYTHON 후속. 3.2um 픽셀로 소형화, 낮은 노이즈. XGS 2000~XGS 45000(45MP).

기타 주요 제조사

CMOSIS (ams) CMV 시리즈. 고속, 대형 포맷. 반도체/디스플레이 검사에서 사용. 현재 ams-OSRAM으로 통합.
Gpixel 중국 기반. GMAX/GSENSE 시리즈. 대형 포맷, 과학용 고감도. 가격 경쟁력.
Samsung (ISOCELL) 모바일 중심이나 산업용 진출 중. 초소형 픽셀(0.56um)의 리더. 머신비전에는 아직 제한적.

선정 체크리스트

센서를 선정할 때는 다음 단계를 순서대로 진행한다. 앞 단계의 결정이 뒤 단계의 선택지를 좁혀준다.

그림 3. 센서 선정 플로우차트 - 7단계 순서
  • 1 FOV 결정 — 검사 대상의 크기를 측정하고 10~20% 마진을 추가한다.
  • 2 해상도 계산 — FOV / (결함 크기 / 필요 pixel 수) = 최소 해상도. 여유 20% 추가.
  • 3 모노/컬러 결정 — 색상 검사가 아니면 모노. 의심스러우면 모노부터 시도.
  • 4 셔터 방식 — 이동 중 촬영이면 글로벌 셔터 필수. 정지 상태면 롤링도 가능.
  • 5 FPS 및 인터페이스 — 택트타임에서 FPS 산출 → 대역폭 계산 → 인터페이스 결정.
  • 6 센서/카메라 모델 — 해상도, 픽셀 크기, 감도, 가용성을 고려하여 구체 모델 선택.
  • 7 예산 확인 — 카메라 본체 + 렌즈 + 케이블 + (프레임 그래버) 총비용 확인.

실무 예시

PCB 납땜 검사

PCB 납땜 검사 센서 선정

요구사항: FOV 60x45mm, 최소 결함 0.05mm, 택트타임 2초

  • 분해능 = 0.05mm / 4 pixel = 0.0125 mm/pixel
  • 수평 = 60 / 0.0125 = 4,800 pixel → 최소 5K
  • FPS = 1/2초 = 0.5fps (여유 충분)
  • 선정: Sony IMX532 (20MP, 2.74um), GigE Vision, 모노
  • 이유: 고해상도 필수, 속도 여유 있어 GigE 가능, 납땜 형상은 밝기로 판별

라벨 인쇄 검사

라벨 인쇄 검사 센서 선정

요구사항: FOV 100x80mm, 색상 불량 검출, 라인 속도 30m/min

  • 분해능 = 0.2mm/pixel (인쇄 품질 기준)
  • 수평 = 100 / 0.2 = 500 pixel → VGA급 충분
  • FPS = 라인 속도 기반, 에어리어 카메라 시 ~25fps
  • 선정: Sony IMX264 (5MP, 3.45um), USB3 Vision, 컬러
  • 이유: 색상 불량 검출 필수 → 컬러 센서. 해상도 여유 확보. 백색 LED 조명 사용.

금속 표면 검사

금속 표면 스크래치 검사 센서 선정

요구사항: FOV 200mm 폭 (라인스캔), 최소 결함 0.02mm, 라인 속도 60m/min

  • 분해능 = 0.02mm / 3 pixel = 0.0067 mm/pixel
  • 라인 픽셀 = 200 / 0.0067 = 29,850 → 최소 32K 라인스캔
  • 라인 레이트 = (60,000mm/60s) / 0.0067 = 149,254 lines/s → ~150 kHz
  • 선정: 32K 라인스캔 카메라, CoaXPress 2.0, 모노
  • 이유: 초고해상도 + 초고속 → Camera Link/CoaXPress 필수. 스크래치는 밝기 대비로 검출.
센서 선정은 반복 과정이다 처음부터 완벽한 선정은 어렵다. 초기 계산으로 후보를 좁히고, 샘플 테스트를 통해 실제 이미지 품질을 확인한 뒤 최종 결정하는 것이 일반적인 프로세스다. 카메라 제조사에 데모 장비를 요청하는 것도 좋은 방법이다.
