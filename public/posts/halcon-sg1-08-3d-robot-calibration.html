이 챕터의 위치 3D Vision은 2D 검사의 연장선이 아니라 좌표계를 다루는 별도의 사고 체계입니다. operator를 외우기 전에 좌표계, 변환 방향, 단위를 명확히 정의하는 습관이 가장 중요합니다.

3D Vision 개요

3D Vision은 이미지 평면의 pixel 좌표를 넘어 실제 world coordinate에서 위치, 자세, 거리, 형상을 다루는 영역입니다.

HALCON의 3D 관련 기능은 다음 문제를 다룹니다.

  • camera calibration
  • image coordinate ↔ world coordinate 변환
  • stereo reconstruction
  • sheet-of-light / laser triangulation
  • depth from focus
  • 3D object model
  • 3D matching
  • robot hand-eye calibration
  • calibrated mosaicking

좌표계 이해

3D에서 가장 중요한 것은 좌표계입니다.

대표 좌표계:

좌표계 의미
Image coordinaterow/column pixel 좌표
Camera coordinate카메라 기준 3D 좌표
World coordinatecalibration object 또는 현장 기준 좌표
Object coordinate객체 자체 기준 좌표
Robot base coordinate로봇 base 기준
Tool coordinateTCP 기준
Calibration plate coordinatecalibration target 기준
3D 오류의 출처 실무에서 3D 오류의 대부분은 좌표계 정의, transform 방향, 단위 혼동에서 발생합니다.

Camera Model

카메라 모델은 3D world point가 image pixel에 어떻게 투영되는지 설명합니다.

고려 요소:

  • focal length
  • principal point
  • pixel size
  • radial distortion
  • tangential distortion
  • lens type
  • area scan / line scan
  • telecentric / perspective
  • tilt lens

3.1 Lens Distortion

렌즈 왜곡은 측정 오차를 만듭니다.

  • radial distortion
  • tangential distortion
  • perspective distortion
정밀 측정의 전제 정밀 측정에서는 왜곡 보정 없이 pixel 좌표로 치수를 재면 안 됩니다.

Camera Calibration

Calibration은 camera parameter와 pose를 추정하는 과정입니다.

4.1 기본 흐름

calibration data model 생성
→ camera parameter 초기화
→ calibration object 정의
→ 여러 pose에서 image 획득
→ calibration mark 관측
→ calibration 수행
→ parameter 저장
→ verification

4.2 좋은 Calibration Image 조건

  • calibration plate가 image 전체 영역을 다양하게 커버
  • 다양한 회전/기울기
  • 초점이 맞음
  • motion blur 없음
  • mark detection 안정
  • 실제 작업 거리와 유사
  • 충분한 image 수
  • corner/edge 영역 포함

4.3 나쁜 Calibration

  • 모든 이미지가 비슷한 pose
  • plate가 image 중앙에만 있음
  • working distance와 다름
  • plate가 휘었거나 오염됨
  • 초점이 흐림
  • 조명이 불균일
  • mark detection error가 큼

Image to World 변환

Calibration 후 image 좌표를 world coordinate로 변환할 수 있습니다.

중요한 조건:

  • 측정 대상이 calibration plane에 있어야 함
  • Z 높이 차이가 있으면 오차 발생
  • camera pose와 plane pose가 정확해야 함

5.1 평면 측정

단일 카메라로도 특정 평면 위의 metric measurement는 가능합니다.

calibrated camera
+ known measurement plane
→ pixel coordinate를 plane coordinate로 변환

이 방식은 평판 부품, PCB, glass, film 검사에 많이 쓰입니다.

Rectification

Rectification은 이미지를 world coordinate 기준으로 보정/재투영하는 과정입니다.

사용 목적:

  • perspective 제거
  • scale 균일화
  • 측정 단순화
  • 후속 2D processing 안정화
주의 interpolation에 의한 blur 가능, rectified image resolution 선택 필요, 원본 좌표와 변환 좌표 mapping 관리 필요.

Stereo Vision

Stereo는 두 카메라의 시차를 이용해 depth를 계산합니다.

7.1 기본 흐름

stereo camera calibration
→ stereo image acquisition
→ rectification
→ correspondence search
→ disparity
→ 3D reconstruction

7.2 성능 영향 요소

  • baseline
  • focal length
  • working distance
  • texture
  • illumination
  • synchronization
  • calibration accuracy
  • rectification accuracy

7.3 한계

  • texture 없는 표면에서 correspondence 어려움
  • 반사체/투명체에 취약
  • occlusion 영역 발생
  • depth noise가 거리 증가에 따라 커짐

Sheet of Light / Laser Triangulation

Laser line을 대상에 투영하고 카메라로 line profile을 관측해 3D 형상을 얻습니다.

8.1 장점

  • 높이 profile 정밀
  • 산업 측정에 강함
  • moving object와 결합 가능

8.2 필요 요소

  • laser plane calibration
  • camera calibration
  • motion/encoder 정보
  • profile extraction
  • intensity/score image 관리

8.3 실패 원인

  • laser reflection saturation
  • dark/absorbing surface
  • specular reflection
  • profile discontinuity
  • calibration plane 오차
  • motion encoder 불일치

Depth from Focus

초점 위치 변화에 따른 sharpness를 이용해 depth를 추정합니다.

적합한 경우:

  • 미세 구조
  • 낮은 depth range
  • stationary object
  • focus stack 가능

한계:

  • acquisition 시간이 김
  • vibration에 취약
  • texture가 필요
  • 조명 안정성 필요

Hand-Eye Calibration

Hand-eye calibration은 카메라와 로봇 좌표계 사이의 transform을 구합니다.

12.1 입력

  • robot tool poses
  • calibration object poses observed by camera
  • camera calibration parameter
  • configuration type

12.2 좋은 Hand-Eye Calibration 조건

  • 다양한 robot pose
  • 회전 변화 충분
  • translation 변화 충분
  • calibration target 안정 고정
  • robot TCP 정확
  • camera calibration 정확
  • outlier pose 제거

12.3 실패 원인

증상 원인
일정 offsetTCP/tool 설정 오류
방향에 따라 오차hand-eye transform 오류
Z 오차 큼plane/height 설정 오류
pose마다 오차 다름calibration image/robot pose mismatch
pick 실패object pose와 grasp pose 정의 불일치

핵심 정리

  • 3D에서는 operator보다 좌표계 정의가 더 중요합니다.
  • Calibration은 한 번 수행하는 이벤트가 아니라 검증/유지관리 대상입니다.
  • 단일 카메라도 평면 조건이 명확하면 metric measurement가 가능합니다.
  • Stereo는 texture와 calibration 품질에 크게 의존합니다.
  • Robot Vision은 camera calibration, hand-eye calibration, TCP 정의가 모두 맞아야 합니다.
  • 3D 오류는 반드시 transform chain을 표로 정리해 추적해야 합니다.
실무 팁 3D 시스템 디버깅 시에는 transform chain (camera → world → robot base → tool → object)을 표로 그려 단계별 단위·방향을 점검하면 원인 추적 시간이 크게 줄어듭니다.
