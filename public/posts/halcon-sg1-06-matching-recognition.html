Matching의 역할

Matching은 이미지에서 기준 객체 또는 pattern을 찾아 위치, 각도, scale, pose 등을 얻는 방법이다. HALCON에서 matching은 단순히 "객체를 찾는 기능"이 아니라, 후속 ROI 정렬과 검사 좌표계 생성의 핵심으로 사용된다.

대표 흐름:

reference image 준비
→ model ROI 선택
→ model 생성
→ search image에서 model 검색
→ row/column/angle/scale/score 획득
→ ROI/contour/measure object transform
→ 후속 검사

Matching 접근 종류

2.1 Correlation-Based Matching / NCC

gray value pattern의 상관성을 기반으로 찾다.

장점:

  • 비교적 빠름
  • texture가 안정적인 대상에 유리
  • 구현이 단순

한계:

  • 조명 변화에 취약
  • 회전/scale 변화에 제한
  • occlusion에 약함
  • 배경 변화에 영향

적합한 경우:

  • 조명 고정
  • 대상 gray pattern이 안정적
  • 회전 변화 작음
  • 속도가 중요

2.2 Shape-Based Matching

edge/contour 기반으로 객체 형상을 찾다. 산업 머신비전에서 가장 자주 쓰이는 matching 방식 중 하나이다.

장점:

  • 조명 밝기 변화에 비교적 강함
  • 회전/scale 대응 가능
  • texture보다 형상 안정성이 중요한 부품에 적합
  • 후속 alignment에 좋음

한계:

  • edge가 약하면 불안정
  • model에 배경 edge가 들어가면 오검출
  • parameter 설정에 따라 속도 저하
  • 과도한 angle/scale 범위는 search cost 증가

2.3 Local Deformable Matching

객체가 국소적으로 변형될 수 있을 때 사용한다.

적합한 경우:

  • 인쇄물 휨
  • 유연한 재질
  • label 변형
  • 부분적인 local distortion
주의 shape-based보다 계산량이 크고, 변형 허용 범위가 크면 false match가 증가한다. 후속 측정에 변형 field를 어떻게 반영할지 고려해야 한다.

2.4 Perspective Deformable Matching

평면 객체가 원근 변형을 받을 때 사용한다.

적합한 경우:

  • 카메라가 정면이 아님
  • 평면 label/mark가 기울어짐
  • homography 기반 정렬 필요

2.5 Descriptor-Based Matching

feature descriptor 기반 matching이다. texture feature나 local feature가 풍부한 대상에서 유리하다.

적합한 경우:

  • perspective 변화
  • scale 변화
  • textured object
  • point correspondence 필요

한계:

  • texture가 없는 금속/단순 형상에는 약할 수 있음
  • 반복 패턴에서 ambiguity 발생 가능

Template / Model 준비

Matching 성공의 절반은 model 준비에서 결정된다. 파라미터 튜닝 전에 model ROI 품질부터 점검해야 한다.

3.1 Model ROI

reference image에서 객체만 포함하도록 ROI를 선택해야 한다.

나쁜 model ROI:

  • 배경 edge 포함
  • 움직이는 부품 포함
  • 반사 highlight 포함
  • 제품마다 변하는 문자/label 포함
  • 불량 sample로 model 생성

좋은 model ROI:

  • 반복적으로 안정적인 형상 포함
  • 배경 제외
  • 충분한 edge/texture 포함
  • object coordinate 기준이 명확함
  • 후속 ROI transform 기준으로 사용 가능

3.2 Synthetic Model

실제 이미지 대신 CAD/도형 기반 contour로 model을 만들 수 있다.

장점:

  • 배경 noise 없음
  • 이상적인 형상
  • 제품 이미지가 없어도 가능

주의:

  • 실제 edge appearance와 차이
  • chamfer/rounding/blur 반영 필요
  • scale/camera projection 차이 고려

Search Parameter

4.1 Angle Range

객체 회전 범위를 지정한다.

  • 범위가 좁을수록 빠르고 안정적
  • 범위가 넓을수록 계산량 증가 및 false match 가능성 증가

실무 원칙:

기구적으로 가능한 회전 범위만 열기

4.2 Scale Range

scale 변화가 필요한 경우만 열어야 한다.

scale 변화 원인:

  • working distance 변화
  • 제품 크기 편차
  • zoom/focus 변경
  • perspective 효과
정밀 검사에서의 주의 scale을 자유롭게 두면 후속 측정 기준이 흔들릴 수 있다.

4.3 Score

Score는 match 품질 지표이다.

주의할 점:

  • score가 높다고 항상 올바른 match는 아님
  • score threshold가 너무 높으면 양품 miss
  • 너무 낮으면 false match
  • occlusion을 허용하려면 score 기준 재설정 필요

실무에서는 양품/불량/오검출 sample score 분포를 저장해 threshold를 잡아야 한다.

4.4 Greediness

Search를 얼마나 공격적으로 가지치기할지 결정한다.

  • 높음: 빠름, miss 가능성 증가
  • 낮음: 느림, robust

속도 이슈가 있을 때 greediness를 올리기 전에 먼저 다음을 확인한다.

  1. search ROI 제한
  2. angle range 제한
  3. scale range 제한
  4. pyramid level 조정
  5. model ROI 품질 개선

4.5 Pyramid / NumLevels

Image pyramid를 사용해 coarse-to-fine search를 수행한다.

장점:

  • 속도 개선
  • 큰 객체 search 효율 향상

주의:

  • 작은 feature가 높은 pyramid level에서 사라질 수 있음
  • 너무 많은 level은 작은 model에 불리
  • model size와 feature thickness를 고려해야 함

Matching 결과 활용

Matching 결과는 보통 다음 값이다.

  • row
  • column
  • angle
  • scale
  • score
  • homography 또는 pose

이 결과를 직접 표시하는 것보다 중요한 것은 후속 검사 좌표계를 정렬하는 것이다.

5.1 ROI Transform

reference ROI
→ match pose로 affine transform
→ current ROI
→ inspection

5.2 Contour Transform

모델 contour를 현재 위치에 표시하거나 편차 검사에 사용한다.

5.3 Measurement ROI Transform

정밀 측정에서 가장 중요하다.

matching으로 기준 위치 확보
→ measure rectangle 이동/회전
→ edge 측정

Multiple Matches

여러 객체를 동시에 찾는 경우:

  • NumMatches 설정
  • minimum distance / overlap 제어
  • score sorting
  • duplicate suppression
  • object identity 관리
반복 패턴 주의 반복 패턴에서는 서로 비슷한 위치가 많기 때문에 단순 score sorting만으로는 잘못된 순서를 선택할 수 있다. 예상 위치, index, grid 구조를 함께 써야 한다.

Occlusion / Clutter 대응

7.1 Occlusion

부분 가림이 있으면 score가 낮아진다.

대응:

  • score threshold 조정
  • model feature 분산
  • 중요하지 않은 영역 제외
  • multiple model 사용
  • deformable/local matching 고려

7.2 Clutter

주변 edge가 많으면 오검출 가능성이 증가한다.

대응:

  • search ROI 제한
  • model ROI 정제
  • polarity/contrast 조건 활용
  • 후속 geometric validation

Matching 실패 디버깅

8.1 Model 확인

  • model contour가 의도한 object만 포함하는가?
  • 배경 edge가 포함되었는가?
  • 너무 작은 feature만 포함했는가?
  • reference image가 blur되었는가?

8.2 Search Image 확인

  • edge contrast가 충분한가?
  • 조명 방향이 바뀌었는가?
  • object가 model과 실제로 다른가?
  • occlusion이 있는가?

8.3 Parameter 확인

  • angle range가 충분한가?
  • scale range가 필요한가?
  • min score가 너무 높은가?
  • num levels가 과한가?
  • search ROI가 object를 포함하는가?

핵심 정리

Matching 설계 핵심
  • Matching은 단독 판정보다 ROI/alignment 기준 생성에 더 중요하다.
  • Shape-based matching은 산업 현장에서 가장 범용적인 기준 위치 검출 방법이다.
  • Model ROI 품질이 성능과 robust에 직접 영향을 준다.
  • Search 범위는 기구적으로 가능한 범위만 열어야 한다.
  • Score는 절대 기준이 아니라 sample 분포 기반으로 해석해야 한다.
  • Matching 결과는 후속 ROI, measure, contour를 transform하는 데 사용해야 한다.
