산업용 이더넷이란?

산업용 이더넷(Industrial Ethernet)은 표준 이더넷(IEEE 802.3)을 기반으로 실시간 통신과 결정론적 데이터 전달을 보장하도록 확장한 통신 기술이다. 공장 자동화, 모션 제어, 프로세스 제어 등 산업 현장에서 PLC, 서보 드라이브, I/O 모듈, 로봇 등이 실시간으로 데이터를 교환하는 데 사용된다.

일반 이더넷 vs 산업용 이더넷

일반 이더넷은 "최선 노력(Best Effort)" 방식으로, 패킷 충돌이나 지연이 허용된다. 하지만 산업 현장에서는 수백 마이크로초 이내에 정확한 타이밍으로 데이터가 도착해야 한다. 서보 모터의 위치 명령이 1ms만 늦어도 불량이 발생하거나 장비가 손상될 수 있기 때문이다.

실시간성의 등급

Soft Real-Time 수~수십 ms 수준. 데이터 수집, HMI 통신 등. 표준 TCP/IP로도 가능.
Hard Real-Time 1ms 이하, 지터(Jitter) 수 us 이내. PLC I/O 제어, 모션 제어에 필수.
Isochronous 수백 ns 수준의 동기화. 다축 서보 모터의 동기 모션 제어 등.
그림 1. 표준 이더넷과 3대 산업용 이더넷 프로토콜의 스택 비교
왜 표준 TCP/IP로는 안 되는가? TCP/IP는 재전송, 라우팅, 윈도우 크기 조절 등으로 인해 수십~수백 ms의 불확정 지연이 발생한다. 산업용 이더넷 프로토콜은 TCP/IP 스택을 우회하거나 최적화하여 결정론적(deterministic) 통신을 달성한다.

EtherCAT

EtherCAT (Ethernet for Control Automation Technology)

독일 Beckhoff Automation이 2003년에 개발한 산업용 이더넷 프로토콜. "Processing on the Fly" 방식으로 하나의 이더넷 프레임이 모든 슬레이브 노드를 순환하며 데이터를 읽고 쓰는 혁신적인 구조로, 초고속 사이클 타임과 높은 대역폭 효율을 달성한다. 현재 EtherCAT Technology Group(ETG)이 관리하며, 전 세계 7,000개 이상의 회원사를 보유하고 있다.

동작 원리: Processing on the Fly

일반적인 이더넷 통신에서는 각 노드가 프레임을 수신, 해석, 응답하는 과정을 거치지만, EtherCAT에서는 프레임이 각 슬레이브의 하드웨어(ESC, EtherCAT Slave Controller)를 통과하면서 하드웨어 수준에서 데이터를 삽입/추출한다. 이 과정은 수 나노초밖에 걸리지 않아, 마치 하나의 거대한 공유 메모리처럼 동작한다.

그림 2. EtherCAT의 패킷 순환 구조 -- 하나의 프레임이 모든 슬레이브를 통과

토폴로지

EtherCAT은 라인(Line), 트리(Tree), 스타(Star) 토폴로지를 모두 지원한다. 가장 일반적인 것은 라인 토폴로지로, 각 슬레이브가 2개의 이더넷 포트를 가지고 데이지 체인으로 연결된다. 별도의 스위치가 필요 없어 배선이 간단하고 비용이 절감된다.

핵심 특징

  • 사이클 타임 -- 최소 12.5us (1000개 분산 I/O)
  • 대역폭 효율 -- 90% 이상 (패킷 오버헤드 최소)
  • 분산 클럭 -- DC (Distributed Clock)로 슬레이브 간 <1us 동기화
  • 핫 커넥트 -- 운전 중 슬레이브 추가/제거 가능

주요 적용 분야

  • 반도체 장비 -- 초정밀 다축 모션 제어
  • 패키징 머신 -- 고속 동기 제어
  • CNC/로봇 -- 짧은 사이클 필수
  • 풍력 발전 -- 분산 I/O 제어
EtherCAT의 강점: 표준 NIC 사용 EtherCAT 마스터는 별도의 전용 하드웨어가 필요 없이 표준 이더넷 NIC만으로 구현할 수 있다. 마스터 소프트웨어(TwinCAT, SOEM 등)가 프레임을 생성/해석하고, 슬레이브 측에서만 ESC(EtherCAT Slave Controller) 칩이 필요하다.

PROFINET

PROFINET

Siemens를 중심으로 PROFIBUS International(PI)에서 개발한 산업용 이더넷 표준. 기존 PROFIBUS(필드버스)의 후속 기술로, PROFIBUS 자산과의 호환성을 유지하면서 이더넷 기반의 실시간 통신을 제공한다. IEC 61158/61784 국제 표준에 등재되어 있으며, 공장 자동화 분야에서 가장 높은 시장 점유율을 차지하고 있다.

RT / IRT 등급

PROFINET은 요구 성능에 따라 3가지 통신 등급을 제공한다.

등급 사이클 타임 방식 적용 분야
NRT (Non Real-Time) ~100ms 표준 TCP/UDP 파라미터 설정, 진단, HMI
RT (Real-Time) 1~10ms VLAN Priority 기반 I/O 제어, 일반 자동화
IRT (Isochronous RT) 31.25us~1ms 전용 ASIC, 시분할 모션 제어, 고속 동기
RT와 IRT의 차이 RT는 소프트웨어로 이더넷 프레임의 우선순위(VLAN Priority)를 조정하여 실시간성을 확보한다. 표준 스위치를 사용할 수 있어 비용이 낮다. IRT는 전용 ASIC이 시간 슬롯을 예약하여 통신하므로, 네트워크 부하와 무관하게 일정한 사이클 타임을 보장하지만 전용 스위치가 필요하다.

PROFIBUS 호환

PROFINET은 PROFIBUS 프록시(Proxy) 기능을 통해 기존 PROFIBUS 장치를 PROFINET 네트워크에 통합할 수 있다. PROFINET IO 컨트롤러에 PROFIBUS 마스터 모듈을 연결하면, PROFIBUS 슬레이브 장치가 PROFINET 장치처럼 동작한다. 이를 통해 기존 설비의 점진적 마이그레이션이 가능하다.

장점

  • Siemens 생태계 -- TIA Portal 완벽 통합
  • 유연한 등급 -- NRT/RT/IRT 혼용 가능
  • 표준 IT 기술 -- DHCP, DNS, SNMP 지원
  • 미디어 리던던시 -- MRP로 링 토폴로지 지원
  • PROFIBUS 호환 -- 기존 자산 활용

단점

  • IRT 비용 -- 전용 ASIC/스위치 필요
  • Siemens 의존 -- 타 PLC에서 지원 제한적
  • RT 대역폭 -- 스위치 성능에 영향받음
  • 설정 복잡도 -- GSD 파일 관리, 네트워크 설계 필요

EtherNet/IP

EtherNet/IP (Ethernet Industrial Protocol)

ODVA(Open DeviceNet Vendors Association)에서 관리하는 산업용 이더넷 프로토콜. CIP(Common Industrial Protocol)를 애플리케이션 계층으로 사용하며, 전송 계층은 표준 TCP/UDP/IP를 수정 없이 그대로 사용한다. Rockwell Automation(Allen-Bradley)이 주도적으로 채택하고 있으며, 북미 시장에서 높은 점유율을 보유하고 있다.

CIP (Common Industrial Protocol)

CIP는 DeviceNet, ControlNet, EtherNet/IP가 공유하는 공통 애플리케이션 계층 프로토콜이다. 장치를 객체(Object)로 모델링하며, 동일한 장치 프로파일을 모든 네트워크에서 사용할 수 있다.

Implicit Messaging UDP 기반, I/O 데이터의 주기적 교환. Producer-Consumer 모델로 멀티캐스트 지원.
Explicit Messaging TCP 기반, 설정/진단 데이터의 비주기적 교환. Client-Server 모델.
CIP Safety SIL 3 안전 통신. 기존 네트워크 인프라에서 안전 데이터 전송.
CIP Motion 모션 제어 확장. IEEE 1588 시간 동기화로 다축 동기 제어.

표준 인프라 활용

EtherNet/IP의 최대 강점은 표준 이더넷 인프라를 수정 없이 사용할 수 있다는 점이다. 일반 관리형 스위치, 표준 라우터, 방화벽을 그대로 사용할 수 있으며, IT 네트워크와의 통합이 용이하다.

장점

  • 표준 인프라 -- 스위치, 라우터 그대로 사용
  • IT 친화적 -- TCP/IP 수정 없이 사용
  • CIP 통합 -- DeviceNet/ControlNet과 프로파일 공유
  • DLR 지원 -- Device Level Ring으로 네트워크 이중화

단점

  • 실시간 성능 -- TCP/IP 오버헤드로 EtherCAT/PROFINET IRT 대비 느림
  • 지터 -- 네트워크 부하에 따라 지연 변동
  • Rockwell 의존 -- 타 벤더 지원이 상대적으로 제한적
  • 고속 모션 -- CIP Motion 지원 장치가 아직 제한적
EtherNet/IP의 "IP"는 Internet Protocol이 아니다 EtherNet/IP의 "IP"는 Industrial Protocol의 약자다. TCP/IP의 IP(Internet Protocol)와 혼동하기 쉽지만, EtherNet/IP는 "이더넷 위에서 동작하는 산업용 프로토콜"이라는 의미이다. 물론 실제로 TCP/IP를 사용하기도 하므로 이중적 의미가 있다.

머신비전과의 연동

산업용 이더넷 프로토콜은 머신비전 시스템에서 직접 이미지를 전송하는 것이 아니라, PLC와 비전 컨트롤러(PC) 사이의 제어/결과 통신에 사용된다. 카메라 영상은 GigE Vision, USB3 Vision 등 전용 프로토콜로 전송하고, 검사 결과와 트리거 신호는 산업용 이더넷으로 PLC에 전달하는 구조이다.

트리거 / 결과 전달

그림 4. 머신비전 시스템과 산업용 이더넷의 연동 구조

주고받는 데이터

트리거 신호 PLC가 제품 도착을 감지하면 비전 PC에 촬영 명령 전송. 하드웨어 I/O 트리거보다 지연이 크지만 배선이 간단.
검사 결과 OK/NG 판정, 측정값, 불량 코드 등을 PLC에 전달. PLC는 이를 기반으로 리젝터 동작.
레시피 관리 PLC에서 모델 변경 시 비전 PC에 레시피(검사 파라미터 세트) 전환 명령 전송.
상태 모니터링 비전 PC의 상태(Ready, Busy, Error), 검사 카운트, 통계 정보를 PLC/HMI에 표시.
TwinCAT Vision (Beckhoff) Beckhoff의 TwinCAT 3에는 비전 기능이 통합되어 있어, EtherCAT 마스터 PC에서 GigE Vision 카메라 제어와 이미지 처리를 동일한 실시간 환경에서 수행할 수 있다. PLC 로직과 비전 검사가 하나의 사이클 안에서 동기적으로 실행되므로, 별도의 통신 지연이 없다.

선택 기준

산업용 이더넷 프로토콜의 선택은 단순히 성능 비교만으로 결정할 수 없다. 기존 설비와의 호환성, PLC 벤더, 팀의 기술 역량, 지역별 서비스 인프라 등을 종합적으로 고려해야 한다.

멀티 벤더 환경

EtherCAT 또는 EtherNet/IP 권장. 7,000+ ETG 회원사, ODVA 개방형 구조. 특정 PLC에 종속되지 않는 시스템.

프로토콜 전환 비용에 주의 기존 라인에서 프로토콜을 변경하면 PLC 프로그램, I/O 모듈, 케이블, 엔지니어 교육까지 모두 바꿔야 한다. 신규 라인이 아닌 한 기존 프로토콜을 유지하는 것이 일반적이며, 점진적 마이그레이션 전략을 권장한다.
TSN (Time-Sensitive Networking)의 부상 IEEE 802.1 TSN은 표준 이더넷에 실시간성을 부여하는 기술로, 향후 산업용 이더넷 프로토콜들이 TSN 위에서 통합될 가능성이 있다. PROFINET over TSN, EtherNet/IP over TSN 등이 이미 개발 중이며, OPC UA + TSN 조합도 주목받고 있다. 다만 상용화까지는 아직 시간이 필요하다.
