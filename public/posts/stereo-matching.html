스테레오 정합이란

Stereo Matching

두 대의 카메라(또는 한 카메라의 두 시점)에서 촬영한 좌/우 영상을 비교하여, 같은 3D 점이 양쪽 영상에서 각각 어디에 맺히는지 대응점(Correspondence)을 찾는 과정이다. 대응점 간의 수평 위치 차이를 시차(Disparity)라 하며, 시차 값이 클수록 카메라에 가까운 물체를 의미한다.

스테레오 정합은 3D 복원의 핵심 단계다. 에피폴라 기하(Epipolar Geometry)에 의해 정류(Rectification)된 영상에서는 대응점 탐색이 같은 행(Scanline) 위의 1차원 문제로 단순화된다. 시차 값 d와 깊이 Z의 관계는 다음과 같다.

시차-깊이 관계 Z = f * B / d 여기서 f는 초점거리(px), B는 기선 길이(Baseline, m), d는 시차(px). 시차가 클수록 깊이가 작다(가까움).
그림 1. 스테레오 정합의 기본 원리 - 좌/우 영상의 시차로 깊이를 계산

스테레오 정합 파이프라인

1. 비용 계산 각 픽셀에 대해 가능한 시차 범위(0~d_max)의 매칭 비용(Cost)을 계산. SAD, SSD, Census 등.
2. 비용 집계 로컬 윈도우 또는 경로(Path)를 따라 비용을 합산하여 노이즈 영향을 줄임.
3. 시차 선택 각 픽셀에서 비용이 최소인 시차 값을 선택 (Winner-Takes-All, WTA).
4. 후처리 좌우 일관성 검사(LR Check), 서브픽셀 보간, 미디언 필터 등으로 품질 개선.

Block Matching (BM)

Block Matching 알고리즘

가장 단순하고 빠른 로컬(Local) 정합 방법. 좌 영상의 각 픽셀 주변에 고정 크기 윈도우(Block)를 잡고, 우 영상의 같은 행(에피폴라 라인)에서 가장 유사한 블록을 찾아 시차를 결정한다. OpenCV의 StereoBM이 대표적 구현이다.

그림 2. Block Matching - 좌 영상의 블록을 우 영상에서 슬라이딩 탐색

비용 함수 종류

밝기 기반

  • SAD (Sum of Absolute Differences) - 절대차 합
  • SSD (Sum of Squared Differences) - 제곱차 합
  • NCC (Normalized Cross-Correlation) - 정규화 상관

구조 기반

  • Census Transform - 주변 픽셀과의 대소관계 비트열
  • Rank Transform - 주변보다 작은 픽셀 수
  • BRIEF - 바이너리 패턴 기술자
Census Transform이 강건한 이유 밝기 절대값이 아닌 상대적 순서만 비교하므로, 좌우 카메라 간 노출 차이나 조명 변화에 강건하다. SGM에서도 Census 기반 비용이 가장 널리 쓰인다.

BM의 한계

텍스처 부족 영역 벽면, 하늘 등 균일한 영역에서 블록 간 구분이 안 되어 오정합 발생.
경계 번짐 윈도우 크기가 크면 물체 경계에서 시차가 번져 정밀도 저하.
가림 영역(Occlusion) 한쪽 영상에서만 보이는 영역은 대응점을 찾을 수 없음.

SGM (Semi-Global Matching)

Semi-Global Matching

Hirschmuller(2005)가 제안한 알고리즘으로, 로컬 방법의 속도와 글로벌 방법의 품질을 절충한다. 픽셀별 매칭 비용에 여러 방향(보통 8~16개)에서의 경로 비용을 합산하여, 인접 픽셀 간 시차 변화에 스무드니스 패널티를 부여한다. OpenCV의 StereoSGBM이 대표 구현이다.

그림 3. SGM - 8개 방향에서 경로 비용을 누적하여 픽셀 p의 최적 시차 결정

스무드니스 패널티 (P1, P2)

SGM의 핵심은 인접 픽셀 간 시차 변화에 패널티를 부여하는 것이다.

P1 (소형 패널티)

시차가 1 픽셀만 변할 때 적용. 값이 작아 경사면 등 부드러운 깊이 변화를 허용한다. 일반적으로 P1 = 8 * numChannels * blockSize^2 수준.

P2 (대형 패널티)

시차가 2 이상 변할 때 적용. P1보다 훨씬 크게 설정하여 급격한 시차 점프를 억제한다. 단, 물체 경계에서는 적응적으로 줄여야 한다. P2 > P1 필수.

적응적 P2 원래 SGM 논문에서는 P2 = P2' / |I_L(p) - I_L(p-r)|처럼 영상 그래디언트에 반비례하게 P2를 조정한다. 물체 경계(밝기 변화가 큰 곳)에서는 패널티를 줄여 시차 불연속을 허용하고, 평탄한 영역에서는 크게 유지해 스무딩한다.

SGM의 강점

속도/품질 균형 O(W * H * D * R) 복잡도. 글로벌 최적화 대비 10~100배 빠르면서 품질은 유사.
병렬화 용이 각 경로(r)가 독립적이므로 SIMD, GPU, FPGA 구현이 자연스럽다.
산업 표준 자율주행(LiDAR 보완), 드론 매핑, 산업용 3D 검사 등 실무에서 가장 많이 사용.

글로벌 방법

글로벌 최적화 방법

영상 전체에 걸쳐 에너지 함수를 최소화하는 방식. 데이터 항(Data Term)과 스무드니스 항(Smoothness Term)으로 구성된 에너지 함수 E(d) = E_data(d) + E_smooth(d)를 전역적으로 최적화한다. 품질은 최고이나 계산 비용이 매우 높다.

Graph Cut (GC)

  • 영상을 그래프로 모델링, 최소 컷으로 최적 시차 할당
  • alpha-expansion 방식으로 반복 최적화
  • MRF 에너지의 근사 최적해 보장
  • 고해상도에서 메모리/시간 매우 큼

Belief Propagation (BP)

  • 인접 노드 간 메시지 전달로 확률적 최적화
  • Loopy BP: MRF에서 반복적 메시지 갱신
  • TRW-S (Tree-Reweighted) 변형이 수렴성 개선
  • GC보다 빠르지만 여전히 실시간 어려움
실무에서 글로벌 방법이 잘 안 쓰이는 이유 VGA(640x480) 영상에서도 수 초~수십 초가 소요되어 실시간 처리가 불가능하다. 학술 벤치마크에서는 최고 성능을 보이지만, 산업 현장에서는 SGM이나 딥러닝 기반 방법이 선호된다.

시차 맵 품질 문제

어떤 알고리즘을 사용하든 스테레오 정합에는 본질적으로 어려운 영역이 존재한다.

그림 4. 스테레오 정합이 어려운 4가지 대표적 영역

후처리 기법

LR Check 좌→우, 우→좌 시차 맵을 각각 계산 후 차이가 임계값(보통 1px) 초과인 픽셀을 가림 영역으로 마킹.
서브픽셀 보간 비용 함수의 최소값 주변 3점에 포물선 피팅하여 0.1px 이하의 정밀도 달성.
미디언 필터 스파이크 형태의 오정합(Salt-and-pepper noise)을 제거. 3x3 또는 5x5.
WLS 필터 Weighted Least Squares 필터로 물체 경계를 보존하면서 시차 맵을 스무딩. OpenCV ximgproc::DisparityWLSFilter.
스펙클 제거 작은 고립 영역(speckle)을 무효화. filterSpeckles()로 연결 요소 크기 기반 필터링.

OpenCV 구현

StereoBM (Block Matching)

import cv2
import numpy as np

# 영상 로드 (그레이스케일)
imgL = cv2.imread('left.png', cv2.IMREAD_GRAYSCALE)
imgR = cv2.imread('right.png', cv2.IMREAD_GRAYSCALE)

# StereoBM 생성
stereo = cv2.StereoBM_create(
    numDisparities=128,   # 시차 탐색 범위 (16의 배수)
    blockSize=15           # 블록 크기 (홀수, 5~51)
)

# 시차 맵 계산 (16배 스케일된 정수)
disparity = stereo.compute(imgL, imgR)
disparity_float = disparity.astype(np.float32) / 16.0

# 시각화
disp_vis = cv2.normalize(disparity_float, None, 0, 255, cv2.NORM_MINMAX)
cv2.imshow('Disparity BM', disp_vis.astype(np.uint8))
cv2.waitKey(0)

StereoSGBM (Semi-Global Block Matching)

# StereoSGBM 파라미터 설정
numDisparities = 128
blockSize = 5

stereo = cv2.StereoSGBM_create(
    minDisparity=0,
    numDisparities=numDisparities,
    blockSize=blockSize,
    P1=8 * 3 * blockSize ** 2,      # 소형 패널티
    P2=32 * 3 * blockSize ** 2,     # 대형 패널티 (P2 > P1)
    disp12MaxDiff=1,               # LR 일관성 검사
    uniquenessRatio=10,            # 최적/차선 비용 차이 비율
    speckleWindowSize=100,         # 스펙클 필터 윈도우
    speckleRange=32,               # 스펙클 시차 범위
    mode=cv2.STEREO_SGBM_MODE_SGBM_3WAY  # 최적화 모드
)

# 시차 맵 계산
disparity = stereo.compute(imgL, imgR)
disparity_float = disparity.astype(np.float32) / 16.0

WLS 필터로 후처리

from cv2 import ximgproc

# 좌/우 매처 생성
left_matcher = cv2.StereoSGBM_create(...) # 위와 동일 파라미터
right_matcher = ximgproc.createRightMatcher(left_matcher)

# 좌/우 시차 맵 계산
dispL = left_matcher.compute(imgL, imgR)
dispR = right_matcher.compute(imgR, imgL)

# WLS 필터 적용
wls_filter = ximgproc.createDisparityWLSFilter(left_matcher)
wls_filter.setLambda(8000)      # 스무딩 강도
wls_filter.setSigmaColor(1.5)  # 에지 보존 정도

# 필터링된 시차 맵
filtered_disp = wls_filter.filter(dispL, imgL, disparity_map_right=dispR)
filtered_disp = filtered_disp.astype(np.float32) / 16.0
파라미터 튜닝 가이드 numDisparities는 장면의 깊이 범위에 맞게, blockSize는 텍스처에 따라 조절한다. 텍스처가 풍부하면 작게(5~7), 부족하면 크게(15~21). P1/P2는 blockSize에 비례하여 설정하되, P2는 항상 P1보다 커야 한다.
