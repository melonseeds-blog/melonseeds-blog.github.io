ISTQB FL이란?

ISTQB(International Software Testing Qualifications Board)는 소프트웨어 테스팅 분야의 국제 자격 인증 기관이다. Foundation Level(FL)은 ISTQB 자격증 체계의 입문 단계로, 소프트웨어 테스팅의 기본 개념과 원리를 다룬다.

테스팅에 관심이 있는 개발자, QA 엔지니어, 프로젝트 관리자 등 누구나 응시할 수 있으며, 소프트웨어 품질 보증의 기초를 체계적으로 학습할 수 있는 자격증이다.

왜 ISTQB FL인가? 개발자라도 테스팅 기본기를 갖추면 코드 품질이 올라간다. 테스트 설계 기법, 결함 관리, 테스트 프로세스를 체계적으로 이해하면 실무에서 큰 도움이 된다.

학습 계획

ISTQB FL 실라버스(Syllabus)를 기준으로 총 6개 챕터를 순서대로 학습한다. 각 챕터의 학습 목표(Learning Objectives)를 중심으로 핵심 개념을 정리하고, 모의고사로 마무리하는 전략이다.

공부 노트 - 테스팅의 7가지 원리

1. 테스팅은 결함의 존재를 보여줄 수 있지만, 결함이 없음을 증명할 수는 없다.
2. 완벽한 테스팅(Exhaustive Testing)은 불가능하다.
3. 조기 테스팅(Early Testing)으로 시간과 비용을 절약할 수 있다.
4. 결함은 집중된다(Defect Clustering).
5. 살충제 패러독스(Pesticide Paradox) - 같은 테스트를 반복하면 새로운 결함을 찾지 못한다.
6. 테스팅은 정황(Context)에 의존적이다.
7. 오류 부재의 궤변(Absence-of-errors Fallacy) - 결함을 모두 고쳐도 사용자 요구를 충족하지 못하면 의미 없다.

핵심 용어 정리

ISTQB FL 시험에서 자주 출제되는 핵심 용어들이다. 정확한 정의를 숙지하는 것이 합격의 핵심이다.

Verification
검증 - 제품이 명세/설계대로 올바르게 만들어졌는지 확인하는 활동
Validation
확인 - 제품이 사용자 요구사항을 충족하는지 확인하는 활동
Test Case
테스트 케이스 - 특정 조건에서 기대 결과를 검증하기 위한 입력값, 실행 조건, 기대 결과의 집합
Test Suite
테스트 스위트 - 특정 목적을 위해 그룹화된 테스트 케이스의 집합
Defect
결함 - 요구사항 또는 명세를 충족하지 못하는 코드나 문서의 불완전한 부분
Regression Testing
회귀 테스팅 - 변경 후 기존 기능이 영향받지 않았는지 확인하는 테스팅
Verification vs Validation 구분법 Verification은 "제품을 올바르게 만들고 있는가?(Are we building the product right?)", Validation은 "올바른 제품을 만들고 있는가?(Are we building the right product?)"로 구분하면 쉽다.
