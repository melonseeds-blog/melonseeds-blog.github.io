1. 필수 단축키 모음

VS2022에서 마우스 사용을 최소화하면 코딩 속도가 눈에 띄게 빨라집니다. 아래 단축키들은 C++ 개발 시 매일 사용하게 되는 것들입니다. 처음에는 의식적으로 연습하고, 1~2주만 지나면 손이 자동으로 움직입니다.

탐색

대규모 프로젝트에서 원하는 파일이나 심볼을 빠르게 찾아가는 것이 생산성의 핵심입니다.

Ctrl+T
Go to All — 파일, 클래스, 함수 등 모든 것을 한 번에 검색. 가장 많이 쓰는 단축키입니다.
F12
Go to Definition — 함수나 클래스의 정의 위치로 바로 이동합니다.
Ctrl+Shift+F
Find in Files — 솔루션 전체에서 텍스트를 검색합니다. 정규식도 지원됩니다.
Alt+F12
Peek Definition — 현재 파일을 떠나지 않고 인라인으로 정의를 확인합니다.
Ctrl+-
Navigate Back — 이전 위치로 되돌아갑니다. F12로 이동한 후 원래 위치로 복귀할 때 필수입니다.
Ctrl+Shift+-
Navigate Forward — 앞으로 다시 이동합니다. Back과 짝으로 사용합니다.
팁 Ctrl+T를 누른 후 접두사를 붙이면 검색 범위를 좁힐 수 있습니다. f 키워드는 파일만, t 키워드는 타입만, m 키워드는 멤버만 검색합니다.

편집

코드를 빠르게 수정하고 정리하는 단축키입니다. 특히 주석 토글과 라인 이동은 매우 자주 사용됩니다.

Ctrl+K, Ctrl+C
Comment — 선택 영역을 주석 처리합니다.
Ctrl+K, Ctrl+U
Uncomment — 선택 영역의 주석을 해제합니다.
Alt+Up/Down
Move Line — 현재 줄(또는 선택 영역)을 위/아래로 이동합니다.
Ctrl+Shift+V
Clipboard Ring — 이전에 복사한 내용들을 순환하며 붙여넣기 합니다.
Ctrl+D
Duplicate Line — 현재 줄을 복제합니다. 비슷한 코드를 빠르게 만들 때 유용합니다.
Ctrl+K, Ctrl+D
Format Document — 전체 문서의 코드를 자동 정렬합니다.

빌드/실행

빌드와 디버깅 시작은 가장 기본적인 단축키입니다.

Ctrl+Shift+B
Build Solution — 솔루션 전체를 빌드합니다.
F5
Start Debugging — 디버거를 붙여서 실행합니다.
Ctrl+F5
Start Without Debugging — 디버거 없이 실행합니다. 릴리즈 테스트 시 유용합니다.
F9
Toggle Breakpoint — 현재 줄에 브레이크포인트를 설정/해제합니다.

2. 에디터 생산성 기능

IntelliSense 최적화

C++ IntelliSense는 프로젝트 규모가 커지면 느려질 수 있습니다. 아래 설정으로 IntelliSense 속도를 개선하세요.

Tools > Options > Text Editor > C/C++ > Advanced에서 주요 설정들을 조정할 수 있습니다:

  • Auto List Members: True (자동 완성 목록 표시)
  • Disable IntelliSense Squiggles: 성능이 느릴 때 True로 설정하면 도움이 됩니다
  • IntelliSense Rescan Delay: 500ms 이상으로 설정하면 타이핑 중 지연이 줄어듭니다

.clang-format 파일을 프로젝트 루트에 두면 VS가 자동으로 인식합니다. Allman 스타일을 사용하는 설정 예시:

# .clang-format
BasedOnStyle: Microsoft
IndentWidth: 4
UseTab: Always
TabWidth: 4
BreakBeforeBraces: Allman
ColumnLimit: 120
SortIncludes: true

헤더/소스 파일 간 전환은 Ctrl+K, Ctrl+O로 즉시 가능합니다. C++ 개발에서 .h와 .cpp를 오가는 일이 매우 잦으므로 반드시 익혀두세요.

코드 스니펫

VS2022에는 다양한 내장 코드 스니펫이 포함되어 있습니다. 키워드를 입력하고 Tab을 두 번 누르면 스니펫이 확장됩니다.

  • for + Tab Tab: for 루프 자동 생성
  • if + Tab Tab: if 블록 자동 생성
  • class + Tab Tab: 클래스 템플릿 자동 생성
  • switch + Tab Tab: switch 문 자동 생성

커스텀 스니펫을 만들려면 Tools > Code Snippets Manager에서 XML 파일로 작성합니다. 자주 사용하는 코드 패턴이 있다면 스니펫으로 만들어두면 매우 편리합니다.

Surround With 기능은 Ctrl+K, Ctrl+S로 사용합니다. 선택한 코드를 if, for, try-catch 등으로 감쌀 수 있습니다.

멀티 커서와 선택

멀티 커서 기능은 반복적인 편집 작업을 한 번에 처리하는 강력한 도구입니다.

Alt+Click
원하는 위치에 커서를 추가합니다. 여러 줄을 동시에 편집할 수 있습니다.
Ctrl+Shift+L
현재 선택된 단어와 동일한 모든 항목을 선택합니다. 변수명 일괄 변경에 유용합니다.
Alt+Shift+Arrow
Box Selection — 직사각형 영역을 선택합니다. 테이블 형태의 데이터 편집에 적합합니다.
팁 멤버 변수 앞에 m_ 접두사를 일괄로 붙이거나, enum 값들의 형식을 맞출 때 멀티 커서가 특히 유용합니다. 마우스로 하나씩 바꾸던 작업을 몇 초만에 끝낼 수 있습니다.

코드 정리

코드 정리(Code Cleanup) 기능은 저장 시 자동으로 코드를 정돈합니다.

설정 방법:

  1. Analyze > Code Cleanup > Configure Code Cleanup에서 프로파일을 설정합니다
  2. 불필요한 using 제거, 정렬, 공백 정리 등을 포함시킵니다
  3. Tools > Options > Text Editor > Code Cleanup에서 저장 시 자동 실행을 활성화합니다

EditorConfig 파일을 활용하면 팀 전체가 동일한 코딩 스타일을 유지할 수 있습니다:

# .editorconfig
root = true

[*]
indent_style = tab
indent_size = 4
end_of_line = crlf
charset = utf-8-bom
trim_trailing_whitespace = true
insert_final_newline = true

[*.{h,cpp}]
cpp_indent_braces = false
cpp_indent_access_specifiers = false
cpp_new_line_before_open_brace_block = new_line

Include 정리와 정렬도 중요합니다. Tools > Options > Text Editor > C/C++ > Advanced > Sort Includes를 활성화하면 #include 구문이 자동으로 정렬됩니다.

3. 디버깅 고급 기법

VS2022의 디버거는 세계 최고 수준의 C++ 디버거입니다. 기본적인 브레이크포인트 외에도 강력한 고급 기능들이 있습니다.

조건부 브레이크포인트

특정 조건에서만 멈추도록 설정하면 반복문 안에서 원하는 상황을 빠르게 포착할 수 있습니다.

브레이크포인트를 우클릭하고 Conditions를 선택하면 다양한 조건을 설정할 수 있습니다:

  • Conditional Expression: 특정 조건이 참일 때만 정지 (예: i == 500)
  • Hit Count: N번째 히트에서만 정지 (예: 1000번째 반복)
  • Filter: 특정 스레드나 프로세스에서만 정지
// Example: debugging image processing loop
for (int y = 0; y < image.GetHeight(); ++y)
{
    for (int x = 0; x < image.GetWidth(); ++x)
    {
        auto pixel = image.GetPixel(x, y);
        // Set conditional breakpoint here: pixel > 250
        // This stops only when a near-white pixel is found
        ProcessPixel(pixel);
    }
}

Action (Tracepoint) 기능은 특히 유용합니다. 브레이크포인트 설정 시 Actions를 체크하고 로그 메시지를 입력하면, 실행을 멈추지 않고 Output 창에 로그를 남길 수 있습니다. {변수명} 형식으로 변수 값도 출력됩니다.

팁 Tracepoint에서 $FUNCTION, $CALLER, $TID 등의 매크로를 사용하면 함수명, 호출자, 스레드 ID 등을 자동으로 출력할 수 있습니다. printf 디버깅을 대체하는 훨씬 깔끔한 방법입니다.

데이터 브레이크포인트

데이터 브레이크포인트는 C++ 전용 기능으로, 특정 메모리 주소의 값이 변경될 때 실행을 멈춥니다.

Debug > New Breakpoint > Data Breakpoint에서 설정하거나, Watch 창에서 변수를 우클릭하여 설정합니다.

  • 메모리 손상(corruption) 추적에 매우 유용합니다
  • 어디서 값이 변경되는지 모를 때 사용합니다
  • 포인터가 가리키는 메모리의 변경도 감지 가능합니다
// Data breakpoint usage scenario
class ImageBuffer
{
    int m_width;   // Set data breakpoint on this
    int m_height;
    uint8_t* m_data;
};

// If m_width unexpectedly changes to 0,
// set a data breakpoint on &buffer.m_width
// The debugger will stop at the exact line that modifies it
주의 데이터 브레이크포인트는 하드웨어 디버그 레지스터를 사용하므로, 동시에 4개까지만 설정할 수 있습니다. 또한 디버깅 세션 중에만 설정 가능하며, 스택 변수는 해당 스코프에서만 유효합니다.

메모리/레지스터 뷰

이미지 처리 개발에서는 메모리 레벨의 데이터 확인이 종종 필요합니다.

Debug > Windows > Memory에서 메모리 창을 열 수 있습니다 (최대 4개). 주소 입력란에 변수 이름이나 주소를 입력하면 해당 메모리 영역을 확인할 수 있습니다.

Watch 창에서 유용한 표현식들:

  • buffer, 100: 배열의 처음 100개 요소 표시
  • (int*)ptr, 10: 포인터를 int 배열로 해석하여 10개 표시
  • &object: 객체의 메모리 주소 확인

Natvis 파일을 만들면 복잡한 데이터 구조를 디버거에서 보기 좋게 표시할 수 있습니다:

// Image.natvis - Custom debugger visualization
// Place in project root or
//   %USERPROFILE%\Documents\Visual Studio 2022\Visualizers

// <?xml version="1.0" encoding="utf-8"?>
// <AutoVisualizer xmlns="...">
//   <Type Name="ImagingLib::Core::Image">
//     <DisplayString>
//       Image {m_width}x{m_height} ({m_format})
//     </DisplayString>
//     <Expand>
//       <Item Name="Width">m_width</Item>
//       <Item Name="Height">m_height</Item>
//       <Item Name="Format">m_format</Item>
//       <Item Name="Data Size">m_dataSize</Item>
//     </Expand>
//   </Type>
// </AutoVisualizer>

진단 도구

디버깅 중 Diagnostic Tools 창은 자동으로 표시되며, 실시간 CPU와 메모리 사용량을 모니터링합니다.

  • CPU Usage: 디버깅 중 CPU 사용률 그래프와 핫 경로 확인
  • Memory Usage: 힙 메모리 사용량 추적, 스냅샷 비교 가능
  • Events: 브레이크포인트 히트, 예외 발생 등의 이벤트 타임라인

PerfTips는 디버깅 중 코드 한 줄씩 실행할 때 각 줄의 실행 시간을 코드 옆에 표시합니다. 어떤 줄이 느린지 바로 확인할 수 있어 성능 문제를 빠르게 파악할 수 있습니다.

4. vcpkg 패키지 관리

vcpkg는 C++ 라이브러리 의존성을 관리하는 Microsoft의 공식 패키지 매니저입니다. Manifest 모드를 사용하면 프로젝트별로 의존성을 깔끔하게 관리할 수 있습니다.

Manifest 모드 설정

프로젝트 루트에 vcpkg.json 파일을 생성하면 Manifest 모드가 활성화됩니다.

{
    "name": "imaging-library",
    "version": "0.1.0",
    "dependencies": [
        {
            "name": "opencv4",
            "version>=": "4.10.0",
            "features": ["contrib"]
        },
        "spdlog",
        "gtest",
        "fmt"
    ],
    "builtin-baseline": "a1a1cbc975ed44943ae5c2308ebe352ce22eafe0"
}

VS2022 통합 설정:

  1. Tools > Options > vcpkg에서 vcpkg 자동 통합을 활성화합니다
  2. CMake 프로젝트의 경우 CMAKE_TOOLCHAIN_FILE을 vcpkg toolchain으로 지정합니다
  3. MSBuild 프로젝트는 vcpkg integrate install 명령으로 전역 통합합니다

Triplet 선택은 빌드 환경에 따라 다릅니다:

Triplet설명사용 시점
x64-windows64비트, 동적 링크개발/디버깅 시 (기본값)
x64-windows-static64비트, 정적 링크배포 바이너리 빌드 시
x64-windows-static-md64비트, 정적 링크 + 동적 CRTCRT DLL 공유 필요 시

자주 쓰는 패키지

C++ 산업용 이미징 개발에서 자주 사용하는 패키지들입니다:

패키지용도비고
opencv4이미지 처리 핵심 라이브러리features: contrib, cuda (선택)
spdlog고성능 로깅헤더 온리 / 컴파일 모드 선택 가능
gtest단위 테스트 프레임워크GoogleTest + GoogleMock 포함
fmt문자열 포맷팅C++20 std::format의 원조
nlohmann-jsonJSON 파싱직관적인 API, 헤더 온리

버전 제약 조건 설정:

{
    "dependencies": [
        {
            "name": "opencv4",
            "version>=": "4.10.0"
        }
    ],
    "overrides": [
        {
            "name": "opencv4",
            "version": "4.10.0"
        }
    ]
}

트러블슈팅

vcpkg 사용 중 흔히 발생하는 문제와 해결 방법입니다:

증상원인해결법
헤더를 찾을 수 없음Triplet 불일치프로젝트 설정과 triplet 일치 확인
링크 에러 (LNK2019)static/dynamic 혼합Runtime Library 설정 확인 (/MT vs /MD)
빌드가 너무 오래 걸림바이너리 캐시 미설정Binary caching 활성화
버전 충돌baseline 불일치builtin-baseline 업데이트

바이너리 캐싱으로 빌드 시간을 단축하세요:

# Set environment variable for local binary cache
set VCPKG_DEFAULT_BINARY_CACHE=D:\vcpkg-cache

# NuGet server-based team shared cache
vcpkg install --binarysource="nuget,https://my-server/nuget/v3/index.json,readwrite"
팁 커스텀 triplet이 필요하면 triplets/ 폴더에 파일을 만듭니다. 예를 들어 OpenCV에서 CUDA만 활성화하고 나머지 기능은 비활성화하는 triplet을 만들 수 있습니다. vcpkg.json의 "vcpkg-configuration"에서 오버레이 triplet 경로를 지정하세요.

5. 성능 분석 도구

VS2022에는 강력한 성능 프로파일러가 내장되어 있습니다. 별도 도구를 설치할 필요 없이 Debug > Performance Profiler (Alt+F2)에서 바로 사용 가능합니다.

CPU 프로파일러

CPU Usage 도구는 코드의 어느 부분이 가장 많은 CPU 시간을 소비하는지 보여줍니다.

사용 방법:

  1. Debug > Performance Profiler (Alt+F2)를 엽니다
  2. CPU Usage를 체크하고 Start를 클릭합니다
  3. 테스트 시나리오를 실행한 후 Stop Collection을 클릭합니다
  4. 결과에서 Hot Path를 확인하여 병목 함수를 찾습니다

Call Tree 뷰에서는 함수 호출 계층별 CPU 시간을 확인할 수 있고, Caller/Callee 뷰에서는 특정 함수의 호출자와 피호출자를 함께 볼 수 있습니다.

// Example: Profiling image filter performance
ErrorCode ApplyGaussianBlur(Image& image, int kernelSize)
{
    // CPU profiler will show time spent in each section
    ErrorCode err = validateInput(image, kernelSize);
    if (err != ErrorCode::Success)
        return err;

    cv::Mat mat = image.ToMat();     // Data conversion cost
    cv::GaussianBlur(mat, mat,
        cv::Size(kernelSize, kernelSize), 0);  // Actual processing
    image.FromMat(mat);              // Data copy back

    return ErrorCode::Success;
}
팁 Release 빌드에서 프로파일링하세요. Debug 빌드는 최적화가 꺼져 있어서 실제 성능과 크게 다릅니다. Project Properties > C/C++ > Optimization에서 /O2로 설정하고, Linker > Debugging > Generate Debug Info를 /DEBUG로 유지하면 Release에서도 심볼 정보를 볼 수 있습니다.

메모리 프로파일러

이미지 처리는 대용량 메모리를 사용하므로 메모리 프로파일링이 특히 중요합니다.

Memory Usage 도구 사용법:

  1. Performance Profiler에서 Memory Usage를 선택합니다
  2. 실행 중 Take Snapshot으로 스냅샷을 찍습니다
  3. 두 스냅샷을 비교하여 메모리 증가 원인을 파악합니다

힙 스냅샷 비교에서 확인할 것들:

  • Allocations diff: 두 시점 사이의 할당 횟수 차이
  • Size diff: 메모리 크기 변화량
  • Allocation call stack: 메모리를 할당한 코드 위치

ETW 이벤트 추적

ETW(Event Tracing for Windows)는 시스템 레벨의 성능 데이터를 수집합니다.

Performance Profiler에서 .NET Events와 File I/O 도구를 활용하면 다음을 분석할 수 있습니다:

  • File I/O: 파일 읽기/쓰기 빈도와 소요 시간 (이미지 로딩 최적화에 유용)
  • GPU Usage: CUDA 연산 사용 시 GPU 활용도 확인
  • Events Viewer: 커스텀 ETW 이벤트를 타임라인으로 시각화

6. Git 통합 활용

VS2022의 Git 통합 기능은 매우 강력해서 별도의 Git 클라이언트가 거의 필요 없습니다. IDE에서 벗어나지 않고 대부분의 Git 작업을 처리할 수 있습니다.

Git Changes 창

View > Git Changes (Ctrl+Shift+G)로 열 수 있으며, 변경 사항 관리의 중심이 됩니다.

  • Stage/Unstage: 파일 옆의 + / - 버튼으로 스테이징 관리
  • Inline Diff: 파일을 더블클릭하면 변경 전/후를 나란히 비교
  • Commit + Push: 커밋 메시지 입력 후 드롭다운에서 Commit All and Push 선택
  • Amend: 이전 커밋을 수정할 때 Amend 체크박스 사용

커밋 메시지 작성 시 프로젝트 규칙을 따르는 것이 중요합니다:

# Our commit message format
[Core] Add GaussianBlur wrapper
[Measurement] Add edge-based distance calculation
[Viewer] Add blur block parameter panel
[Test] Add ImageProcessor blur test cases

브랜치 관리

하단 상태 표시줄의 브랜치 이름을 클릭하면 브랜치 관리 메뉴가 나타납니다.

  • 브랜치 전환: 로컬/원격 브랜치 목록에서 빠르게 전환
  • 새 브랜치 생성: 현재 위치에서 새 feature 브랜치 생성
  • Merge/Rebase: Git > Manage Branches에서 브랜치 간 병합

3-Way Merge Editor는 충돌 해결의 핵심 도구입니다. 충돌이 발생하면 VS가 자동으로 3-way 병합 편집기를 열어서 좌측(내 변경), 우측(상대 변경), 하단(결과)을 동시에 보여줍니다. 체크박스로 원하는 변경을 선택하면 됩니다.

Git Blame과 히스토리

코드의 변경 이력을 추적하는 것은 디버깅과 코드 이해에 필수적입니다.

  • Annotate (Blame): 에디터 좌측 거터를 우클릭 > Git > Annotate로 각 줄의 마지막 수정자와 커밋을 확인합니다
  • File History: Solution Explorer에서 파일 우클릭 > Git > View History로 전체 변경 이력을 타임라인으로 확인합니다
  • Line History: 특정 코드 블록을 선택하고 우클릭 > Git > View Line History로 해당 줄의 변경 이력만 볼 수 있습니다
팁 GitLab Self-Hosted 서버를 사용하는 경우에도 VS2022의 Git 통합은 완벽하게 동작합니다. Git > Settings > Git Repository Settings에서 Remote URL을 설정하면 push/pull/fetch가 IDE 내에서 가능합니다.

7. 추천 확장 프로그램

VS2022의 기본 기능만으로도 충분히 강력하지만, 아래 확장 프로그램들을 추가하면 개발 경험이 한 단계 더 향상됩니다.

확장 프로그램설명추천 이유
Visual Assist C++ 코드 탐색, 리팩토링, 자동완성 강화 기본 IntelliSense보다 빠르고 정확한 C++ 자동완성. 심볼 검색과 리팩토링 기능이 우수합니다.
CodeMaid 코드 정리, 포맷팅, 복잡도 시각화 저장 시 자동 코드 정리, 파일 내 메서드 계층 구조 시각화 기능이 편리합니다.
Productivity Power Tools 에디터 편의 기능 모음 가이드라인 표시, 솔루션 에러 시각화, 탭 관리 등 소소하지만 유용한 기능들의 모음입니다.
GitHub Copilot AI 기반 코드 자동완성 반복적인 코드 작성을 크게 줄여줍니다. OpenCV 래핑 코드 작성 시 특히 유용합니다.
Markdown Editor 마크다운 편집기와 미리보기 README, 문서 작성 시 실시간 미리보기가 가능합니다.
Output Enhancer 출력 창 색상 강조 빌드 에러/경고를 색상으로 구분하여 긴 빌드 로그에서 문제를 빠르게 찾을 수 있습니다.
BuildVision 빌드 상태 시각화 도구 대규모 솔루션의 프로젝트별 빌드 상태, 소요 시간, 에러를 한눈에 보여줍니다.
주의: 확장 프로그램 과다 설치 금지 확장 프로그램을 너무 많이 설치하면 VS 시작 시간과 메모리 사용량이 크게 증가합니다. 실제로 매일 사용하는 것만 설치하고, 나머지는 비활성화하세요. 특히 C++ 프로젝트에서는 IntelliSense와 충돌하는 확장이 있을 수 있으니 하나씩 추가하면서 안정성을 확인하는 것이 좋습니다.

8. 추천 설정

환경 설정

VS2022를 처음 설치하면 아래 설정들을 먼저 변경하는 것을 권장합니다:

  • 테마: Tools > Options > Environment > General에서 Dark 테마를 선택합니다. 장시간 코딩 시 눈의 피로가 크게 줄어듭니다.
  • 폰트: Tools > Options > Environment > Fonts and Colors에서 Cascadia Code 또는 D2Coding을 추천합니다. Cascadia Code는 리가처(ligature) 지원으로 !=, >= 등이 하나의 기호로 표시됩니다.
  • 줄 번호: Tools > Options > Text Editor > All Languages > General에서 Line numbers를 항상 켜두세요.
  • 자동 줄바꿈: Word wrap은 일반적으로 끄는 것을 추천합니다. 코드가 길어지면 리팩토링이 필요하다는 신호입니다.

C++ 프로젝트 설정

새 C++ 프로젝트를 생성할 때 반드시 확인해야 할 설정들입니다.

설정 항목위치권장값
C++ 표준C/C++ > Language/std:c++17
경고 수준C/C++ > General/W4
경고를 에러로C/C++ > General/WX (권장)
Address SanitizerC/C++ > GeneralDebug 빌드에서 활성화
소스 인코딩C/C++ > Command Line/utf-8

특히 /utf-8 플래그는 한글이 포함된 소스 파일에서 반드시 필요합니다:

// Without /utf-8 flag, this might cause encoding issues
// Project Properties > C/C++ > Command Line > Additional Options: /utf-8

#include <string>

const std::string kErrorMessage = "Image not found.";
// /utf-8 ensures correct encoding for both source and execution character sets

Address Sanitizer는 메모리 관련 버그를 조기에 발견하는 데 매우 유용합니다:

// Address Sanitizer catches these at runtime:
// - Buffer overflow
// - Use after free
// - Stack buffer overflow
// - Memory leaks

// Enable: Project Properties > C/C++ > General
//         > Enable Address Sanitizer = Yes
// Only for Debug builds (adds ~2x runtime overhead)
팁 /W4와 /WX(경고를 에러로 처리)를 함께 사용하면 잠재적인 버그를 컴파일 단계에서 잡을 수 있습니다. 처음에는 경고가 많아 귀찮을 수 있지만, 깨끗한 코드를 유지하는 가장 효과적인 방법입니다.

성능 최적화

VS2022가 느리게 느껴진다면 아래 설정들을 확인하세요:

  • 사용하지 않는 확장 비활성화: Extensions > Manage Extensions에서 불필요한 확장을 끕니다. 시작 시간이 크게 단축됩니다.
  • 솔루션 필터 (.slnf): 대규모 솔루션에서 현재 작업 중인 프로젝트만 로드하여 메모리와 IntelliSense 성능을 개선합니다. 
    {
    "solution": {
        "path": "ImagingLibrary.sln",
        "projects": [
            "core\\Core.vcxproj",
            "tests\\core_tests\\CoreTests.vcxproj"
        ]
    }
}
  • 병렬 빌드 설정: Tools > Options > Projects and Solutions > Build and Run에서 Maximum number of parallel project builds를 CPU 코어 수에 맞게 설정합니다. 또한 프로젝트 속성에서 /MP (Multi-processor Compilation) 플래그를 활성화합니다.
  • SSD 사용 권장: 프로젝트 파일, vcpkg 캐시, 심볼 캐시를 모두 SSD에 배치하세요. 빌드 시간과 IntelliSense 반응 속도가 눈에 띄게 개선됩니다.
마무리 VS2022의 모든 기능을 한 번에 익힐 필요는 없습니다. 매일 사용하는 단축키 5개부터 시작하고, 디버깅과 프로파일링 도구를 하나씩 실습해보세요. 작은 습관의 변화가 쌓이면 몇 달 후에는 확실히 다른 속도로 개발하게 됩니다.
