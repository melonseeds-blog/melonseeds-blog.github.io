1. 개요 - Claude Desktop이란?

Claude Desktop vs claude.ai 웹

Claude Desktop은 Anthropic이 공식적으로 제공하는 데스크톱 애플리케이션입니다. 웹 브라우저에서 접속하는 claude.ai와 동일한 AI 모델을 사용하지만, 네이티브 앱으로서 더 깊은 시스템 통합과 확장 기능을 제공합니다.

브라우저 탭 하나에 의존하는 웹 버전과 달리, 데스크톱 앱은 독립 실행 프로세스로 동작하여 시스템 자원을 더 효율적으로 활용할 수 있습니다. 특히 파일 시스템 접근, MCP 서버 연동, 키보드 단축키 등 웹에서는 불가능한 기능들이 큰 차별점입니다.

데스크톱 앱의 장점

  • 네이티브 성능 - 브라우저 오버헤드 없이 빠르게 동작합니다
  • 파일 접근 - 로컬 파일을 드래그 앤 드롭으로 바로 첨부할 수 있습니다
  • MCP 통합 - 외부 도구와 서비스를 연결하여 기능을 확장합니다
  • 단축키 - 전역 단축키로 어디서든 빠르게 Claude를 호출할 수 있습니다
  • 독립 실행 - 브라우저를 닫아도 대화가 유지됩니다

지원 플랫폼

현재 Claude Desktop은 macOS와 Windows를 공식 지원합니다. Linux는 아직 공식 지원되지 않지만, Claude Code(CLI)를 통해 터미널 기반으로 사용할 수 있습니다.

Claude Desktop vs Claude Web vs Claude Code

Claude Desktop

  • 네이티브 데스크톱 애플리케이션
  • MCP 서버 연동 지원
  • Projects 기능으로 컨텍스트 관리
  • 파일 드래그 앤 드롭
  • 전역 단축키(Alt+Space)
  • GUI 기반의 직관적 인터페이스

Claude Web (claude.ai)

  • 브라우저에서 바로 접속
  • 설치 불필요, 어디서든 사용
  • Projects 기능 동일 지원
  • Artifacts(코드, 다이어그램) 지원
  • MCP 연동 불가
  • 모바일 브라우저에서도 사용 가능

Claude Code (CLI)

  • 터미널 기반 코딩 어시스턴트
  • 파일 읽기/쓰기, Git 자동화
  • 코드베이스 전체 맥락 파악
  • 빌드, 테스트 자동 실행
  • 개발자 특화 워크플로우
  • Linux/Mac/Windows(WSL) 지원

어떤 걸 써야 할까?

  • 일반 업무: Desktop 또는 Web
  • 코딩 작업: Claude Code
  • 문서 분석: Desktop(파일 첨부 편리)
  • 이동 중: Web(모바일)
  • 외부 연동: Desktop(MCP)
  • 팀 협업: Web 또는 Desktop

2. 설치 및 초기 설정

다운로드

Claude Desktop은 claude.ai/download에서 무료로 다운로드할 수 있습니다. Anthropic 계정이 있으면 무료 플랜으로도 바로 사용 가능합니다.

설치 과정

Windows:

  1. claude.ai/download에서 Windows 버전 설치 파일(.exe)을 다운로드합니다
  2. 다운로드된 설치 파일을 실행하면 자동으로 설치가 진행됩니다
  3. 설치가 완료되면 자동으로 Claude Desktop이 실행됩니다
  4. Anthropic 계정으로 로그인하면 사용 준비가 완료됩니다

macOS:

  1. claude.ai/download에서 macOS 버전(.dmg)을 다운로드합니다
  2. DMG 파일을 열고 Claude 아이콘을 Applications 폴더로 드래그합니다
  3. Applications에서 Claude를 실행합니다
  4. 처음 실행 시 macOS 보안 경고가 나타나면 "열기"를 클릭합니다

첫 로그인 및 설정

앱을 처음 실행하면 Anthropic 계정 로그인 화면이 나타납니다. Google, 이메일 등으로 로그인할 수 있으며, 계정이 없다면 무료로 생성할 수 있습니다. 로그인 후 바로 대화를 시작할 수 있습니다.

주요 단축키

기능WindowsmacOS
빠른 실행 (전역)Alt + SpaceOption + Space
새 대화Ctrl + NCmd + N
대화 검색Ctrl + KCmd + K
파일 첨부Ctrl + UCmd + U
설정 열기Ctrl + ,Cmd + ,
팁 전역 단축키를 활용하면 다른 프로그램에서 작업 중에도 즉시 Claude를 호출할 수 있습니다. 작업 흐름을 끊지 않고 질문하거나 도움을 받을 수 있어 생산성이 크게 향상됩니다.

3. 인터페이스 가이드

채팅 인터페이스

Claude Desktop의 메인 화면은 간결한 채팅 인터페이스로 구성됩니다. 왼쪽 사이드바에는 대화 히스토리와 Projects 목록이 표시되며, 중앙 영역이 대화 공간입니다. 하단 입력창에서 메시지를 작성하고 전송합니다.

파일 첨부

Claude Desktop에서 파일을 첨부하는 방법은 여러 가지입니다.

  • 드래그 앤 드롭 - 파일을 대화창으로 직접 끌어다 놓으면 됩니다
  • 클립보드 붙여넣기 - 스크린샷이나 복사한 이미지를 Ctrl+V로 바로 붙여넣을 수 있습니다
  • 첨부 버튼 - 입력창의 클립 아이콘을 클릭하여 파일 탐색기에서 선택합니다

지원하는 파일 형식은 이미지(PNG, JPG, GIF, WebP), PDF, 텍스트 파일, 코드 파일 등입니다. 한 번에 여러 파일을 동시에 첨부할 수도 있습니다.

이미지 및 PDF 분석

Claude는 멀티모달 AI로서 이미지와 문서를 직접 분석할 수 있습니다. 업로드된 이미지의 내용을 설명하거나, 차트/그래프를 해석하거나, PDF 문서의 내용을 요약하는 등의 작업이 가능합니다.

참고 PDF의 경우 텍스트 기반 PDF는 물론 스캔된 이미지 PDF도 분석할 수 있습니다. 다만, 매우 긴 문서의 경우 토큰 제한에 의해 전체를 한 번에 처리하지 못할 수 있으므로 섹션별로 나누어 분석하는 것이 효과적입니다.

Artifacts 시스템

Claude가 생성하는 콘텐츠 중 코드, 문서, 다이어그램 등은 Artifacts라는 별도 패널에 표시됩니다. Artifacts는 대화 옆에 나란히 표시되어 코드를 실행하거나 미리보기를 확인할 수 있습니다.

  • 코드 - 구문 강조가 적용된 코드 블록, 복사 버튼 제공
  • 문서 - Markdown 기반 문서 렌더링
  • 다이어그램 - Mermaid, SVG 등의 시각화
  • 웹 페이지 - HTML/CSS/JS를 실시간 미리보기

모델 선택

대화창 하단에서 사용할 모델을 선택할 수 있습니다. 각 모델의 특성에 따라 적절히 선택하세요.

모델특징추천 용도
Opus최고 성능, 복잡한 추론심층 분석, 복잡한 코딩, 연구
Sonnet성능과 속도의 균형일반 업무, 코드 작성, 문서 작업
Haiku빠른 응답, 가벼운 작업간단한 질문, 요약, 분류

확장 사고(Extended Thinking) 모드

Pro 요금제에서 사용할 수 있는 확장 사고 모드는 Claude가 답변하기 전에 더 깊은 추론 과정을 거치도록 합니다. 복잡한 수학 문제, 코드 디버깅, 전략적 분석 등에서 더 정확한 결과를 얻을 수 있습니다.

팁 확장 사고 모드는 응답 시간이 더 걸리지만, 복잡한 문제에서는 한 번에 정확한 답을 얻을 수 있어 오히려 전체 시간을 절약할 수 있습니다.

4. Projects 기능

Projects란?

Projects는 관련된 대화, 지침, 참고 문서를 하나의 공간에 모아 관리하는 기능입니다. 프로젝트별로 Claude에게 지속적인 컨텍스트를 제공할 수 있어, 매번 배경 설명을 반복할 필요가 없습니다.

예를 들어, "산업용 이미징 라이브러리 개발" 프로젝트를 만들면 코딩 규칙, 아키텍처 문서, API 레퍼런스 등을 한곳에 모아두고 해당 프로젝트의 모든 대화에서 자동으로 참조할 수 있습니다.

프로젝트 생성 및 구성

  1. 사이드바에서 "새 프로젝트" 버튼을 클릭합니다
  2. 프로젝트 이름과 설명을 입력합니다
  3. 프로젝트 지침(Project Instructions)을 작성합니다
  4. 필요한 참고 문서를 Knowledge에 업로드합니다
  5. 프로젝트 내에서 새 대화를 시작합니다

프로젝트 지침 작성

프로젝트 지침은 해당 프로젝트의 모든 대화에 자동으로 적용되는 시스템 프롬프트입니다. 효과적인 지침을 작성하면 Claude가 일관된 스타일과 규칙을 따르도록 할 수 있습니다. 

# Project Instructions Example

## Role
You are a senior C++ developer for industrial imaging software.

## Coding Rules
- Use C++17 standard
- Allman brace style
- All processing functions return ErrorCode
- Smart pointers only (no raw new/delete)

## Response Format
- Explanations in Korean
- Code and comments in English
- Keep it concise

Knowledge Base

Knowledge에 업로드한 문서는 프로젝트의 모든 대화에서 Claude가 참조할 수 있는 지식 기반이 됩니다. API 문서, 스타일 가이드, 사양서, 코드 샘플 등을 업로드하면 Claude가 해당 내용을 기반으로 더 정확한 답변을 제공합니다.

  • 지원 형식: PDF, TXT, CSV, 코드 파일 등
  • 프로젝트당 최대 용량이 제한되어 있으니 핵심 문서를 선별하세요
  • 문서를 업데이트하면 이후 대화에 바로 반영됩니다

팀 공유

Team이나 Enterprise 플랜에서는 프로젝트를 팀원과 공유할 수 있습니다. 공유된 프로젝트의 지침과 Knowledge는 팀 전체에 동일하게 적용되어 일관된 AI 활용 환경을 구축할 수 있습니다.

프로젝트 구성 베스트 프랙티스

  • 목적별 분리 - "프론트엔드 개발", "백엔드 개발", "문서 작성" 등 목적별로 프로젝트를 분리합니다
  • 지침은 구체적으로 - 모호한 지침보다 구체적인 규칙이 더 효과적입니다
  • 핵심 문서만 업로드 - 너무 많은 문서는 오히려 혼란을 줄 수 있습니다
  • 정기적 업데이트 - 프로젝트 진행에 따라 지침과 문서를 갱신합니다
  • 대화 정리 - 오래된 대화는 아카이브하여 프로젝트를 깔끔하게 유지합니다

5. MCP 서버 연동

MCP란?

MCP(Model Context Protocol)는 Anthropic이 개발한 개방형 프로토콜로, AI 모델이 외부 도구 및 데이터 소스와 표준화된 방식으로 통신할 수 있게 합니다. USB-C 포트처럼 하나의 표준으로 다양한 도구를 연결하는 개념입니다.

Claude Desktop은 MCP 클라이언트 역할을 하여, 설정된 MCP 서버들과 통신합니다. 이를 통해 파일 시스템 접근, 데이터베이스 조회, 웹 브라우징, GitHub 연동 등 다양한 외부 기능을 Claude에게 제공할 수 있습니다.

설정 파일 구성

MCP 서버는 claude_desktop_config.json 파일에서 설정합니다. 이 파일의 위치는 운영체제에 따라 다릅니다.

OS설정 파일 경로
Windows%APPDATA%\Claude\claude_desktop_config.json
macOS~/Library/Application Support/Claude/claude_desktop_config.json

기본 설정 파일 구조는 다음과 같습니다.

{
  "mcpServers": {
    "server-name": {
      "command": "executable",
      "args": ["arg1", "arg2"],
      "env": {
        "ENV_VAR": "value"
      }
    }
  }
}

주요 MCP 서버

1. 파일 시스템 접근

로컬 파일 시스템에 읽기/쓰기 접근을 허용합니다. 특정 디렉토리만 허용하여 보안을 유지할 수 있습니다.

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\username\\Documents",
        "C:\\Projects"
      ]
    }
  }
}

2. GitHub 연동

GitHub 리포지토리의 이슈, PR, 코드를 직접 조회하고 관리할 수 있습니다.

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    }
  }
}

3. 웹 검색 및 브라우징

Brave Search API를 활용하여 웹 검색 기능을 Claude에게 제공합니다.

{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "your-api-key"
      }
    }
  }
}

4. 데이터베이스 연결

SQLite, PostgreSQL 등 데이터베이스에 읽기 전용으로 접근하여 데이터를 조회하고 분석할 수 있습니다.

{
  "mcpServers": {
    "sqlite": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-sqlite",
        "--db-path", "C:\\data\\mydb.sqlite"
      ]
    }
  }
}

전체 설정 예시

여러 MCP 서버를 동시에 설정하는 전체 예시입니다.

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Projects",
        "C:\\Documents"
      ]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    },
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "your-api-key"
      }
    }
  }
}

보안 고려사항

보안 주의 MCP 서버는 로컬 시스템에 접근할 수 있으므로 다음 사항을 반드시 준수하세요.
  • 최소 권한 원칙 - 파일 시스템 접근 시 필요한 디렉토리만 허용합니다
  • API 키 관리 - 설정 파일에 직접 기입하는 API 키는 안전하게 관리합니다
  • 신뢰할 수 있는 서버만 사용 - 공식 MCP 서버 또는 검증된 커뮤니티 서버만 설치합니다
  • 읽기 전용 우선 - 가능하면 읽기 전용 접근부터 시작하고, 필요 시에만 쓰기 권한을 추가합니다
  • 정기 점검 - 사용하지 않는 MCP 서버는 설정에서 제거합니다

7. 실전 활용 사례

문서 분석 및 요약

PDF 보고서, 기술 문서, 논문 등을 첨부하여 핵심 내용을 추출하거나 요약할 수 있습니다. 특히 영문 기술 문서를 한국어로 요약해달라고 요청하면 빠르게 내용을 파악할 수 있습니다.

  • 기술 문서를 첨부하고 "이 문서의 핵심 내용을 정리해줘"
  • 여러 문서를 비교 분석: "이 두 스펙 문서의 차이점을 정리해줘"
  • 긴 문서에서 특정 정보 검색: "이 계약서에서 해지 조건을 찾아줘"

코드 리뷰

코드 파일을 첨부하여 리뷰를 요청하면 버그 가능성, 개선 사항, 보안 취약점 등을 체계적으로 검토받을 수 있습니다.

  • "이 코드에서 잠재적 버그를 찾아줘"
  • "이 코드의 성능을 개선할 수 있는 부분이 있어?"
  • "이 함수를 C++17 스타일로 리팩터링해줘"

리서치 및 조사

웹 검색 MCP를 연동하면 최신 정보를 기반으로 리서치를 수행할 수 있습니다. 기술 동향 조사, 경쟁사 분석, 시장 조사 등에 활용할 수 있습니다.

회의록 및 액션 아이템

회의 내용을 텍스트로 입력하거나 녹취 파일을 첨부하면 Claude가 회의록을 정리하고 액션 아이템을 도출해줍니다.

  • 회의 내용 구조화 (참석자, 주요 논의, 결정 사항, 액션 아이템)
  • 후속 조치 담당자 및 기한 정리
  • 회의 결과를 이메일 형식으로 변환

기술 문서 작성

API 문서, 사용자 가이드, 프로젝트 제안서 등 다양한 기술 문서를 작성할 수 있습니다. Projects의 Knowledge에 기존 문서를 업로드하면 일관된 스타일로 새 문서를 생성합니다.

데이터 분석

CSV나 Excel 파일을 첨부하면 데이터 분석, 트렌드 파악, 이상치 탐지 등을 수행할 수 있습니다. Artifacts를 통해 분석 결과를 차트나 표로 시각화해줍니다.

  • "이 판매 데이터에서 월별 트렌드를 분석해줘"
  • "이 센서 로그에서 이상치를 찾아줘"
  • "이 데이터를 기반으로 대시보드용 차트를 만들어줘"

컨텍스트 관리 전략

Claude의 대화 컨텍스트는 무한하지 않습니다. 효율적인 컨텍스트 관리로 더 좋은 답변을 얻을 수 있습니다.

  • 주제별 대화 분리 - 하나의 대화에 여러 주제를 섞지 않습니다
  • 요약 요청 - 긴 대화가 이어지면 중간에 요약을 요청하여 핵심을 정리합니다
  • 새 대화 시작 - 주제가 바뀌면 새 대화를 시작하되, Projects 안에서 시작하면 기본 컨텍스트가 유지됩니다
  • 핵심 정보 먼저 - 가장 중요한 정보와 요청을 메시지 앞부분에 배치합니다

  2. 업무 중 - 전역 단축키로 빠르게 질문. 문서 분석, 코드 리뷰 등 수시로 활용
