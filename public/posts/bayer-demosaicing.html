Bayer 패턴이란

이미지 센서의 각 픽셀은 본래 밝기(광량)만 측정할 수 있다. 색상 정보를 얻으려면 각 픽셀 위에 특정 색만 통과시키는 컬러 필터 배열(CFA, Color Filter Array)을 올려야 한다. 1976년 Kodak의 Bryce Bayer가 발명한 Bayer 패턴은 가장 널리 쓰이는 CFA 방식이다.

핵심 원리 — 1픽셀 = 1색

Bayer 패턴에서 각 픽셀은 R, G, B 중 하나의 색상 정보만 기록한다. 나머지 두 색상은 주변 픽셀 값으로부터 추정(보간)해야 한다. 이 보간 과정이 바로 디모자이킹(Demosaicing)이다.

왜 Green이 2개인가?

Bayer 패턴은 2x2 단위 셀에서 R 1개, G 2개, B 1개로 구성된다. Green 필터가 2배 많은 이유는 다음과 같다.

인간 시각 특성 사람의 눈은 녹색 파장(~555nm)에 가장 민감하다. 녹색 해상도를 높이면 전체 이미지의 선명도가 향상된다.
휘도(Luminance) 기여 밝기 정보의 약 70%가 녹색 채널에서 온다. Green 샘플이 많으면 노이즈가 줄고 디테일이 살아난다.
보간 정확도 Green 샘플이 촘촘하면 나머지 R, B 채널의 보간도 Green을 기준으로 더 정확하게 수행할 수 있다.

CFA 배치 변형

2x2 단위 셀에서 R, G, B 필터의 시작 위치에 따라 4가지 변형이 존재한다. 센서 제조사마다 다르므로, SDK에서 올바른 패턴을 설정해야 정확한 색상이 나온다.

패턴 이름 첫 번째 행 두 번째 행 대표 센서
RGGB R G G B Sony IMX 시리즈 다수
GRBG G R B G ON Semi PYTHON 시리즈
GBRG G B R G 일부 CMOSIS 센서
BGGR B G G R 일부 Aptina 센서
패턴을 잘못 설정하면? Bayer 패턴 타입이 실제 센서와 다르면 R과 B가 뒤바뀌거나, 이미지 전체에 색 줄무늬(color cast)가 나타난다. SDK에서 PixelFormat을 설정할 때 센서 데이터시트의 CFA 배치를 반드시 확인하자.

Bayer 패턴 구조

아래는 RGGB 배치의 4x4 Bayer 패턴 다이어그램이다. 각 픽셀은 하나의 색상 필터만 가지며, 이 Raw 데이터로부터 풀컬러 이미지를 복원하는 것이 디모자이킹의 목표이다.

그림 1. RGGB Bayer 패턴의 4x4 격자 구조 — Green이 전체의 50%를 차지한다
Raw 이미지란? Bayer 패턴 센서에서 출력되는 가공되지 않은 데이터를 Raw 이미지라고 한다. 각 픽셀에 하나의 색상 값(8~16bit)만 들어 있으며, 이를 그대로 보면 모자이크처럼 보인다. 디모자이킹을 거쳐야 우리가 보는 풀컬러 RGB 이미지가 된다.

디모자이킹(Demosaicing)

디모자이킹(Demosaicing, 또는 Debayering)은 Bayer Raw 이미지에서 각 픽셀의 빠진 2개 색상 값을 보간(interpolation)하여 풀컬러 RGB 이미지를 생성하는 과정이다.

디모자이킹의 본질

예를 들어 Red 필터가 있는 픽셀은 R 값만 알고 있다. 이 픽셀의 G, B 값을 인접 픽셀들의 G, B 값으로부터 추정해야 한다. 모든 픽셀에 대해 이 과정을 수행하면 최종 RGB 이미지가 완성된다.

그림 2. Bayer Raw에서 풀컬러 RGB로의 디모자이킹 과정

보간 알고리즘 비교

디모자이킹 알고리즘은 속도와 품질 사이의 트레이드오프가 있다. 단순한 알고리즘은 빠르지만 아티팩트가 많고, 고급 알고리즘은 품질이 좋지만 연산량이 크다.

알고리즘 원리 품질 속도 용도
Nearest Neighbor 가장 가까운 같은 색 픽셀 복사 낮음 매우 빠름 미리보기, 실시간 모니터링
Bilinear 상하좌우 같은 색 픽셀의 평균 보통 빠름 일반 산업용, 실시간 처리
MHC (Malvar-He-Cutler) 5x5 커널, 그래디언트 보정 포함 높음 보통 고품질 산업 검사
AHD (Adaptive Homogeneity-Directed) 수평/수직 보간 후 균질성 비교 매우 높음 느림 사진, 의료 영상
DCB (DCB Demosaicing) 반복 색차 보정, 에지 보존 매우 높음 느림 RAW 현상 소프트웨어

Bilinear 보간 원리

가장 많이 쓰이는 Bilinear 보간은 인접한 같은 색 픽셀의 평균을 취한다. 예를 들어 Green 위치에서 빠진 Red 값을 구할 때, 좌우(또는 상하)에 있는 R 픽셀의 평균을 사용한다.

그림 3. Bilinear 보간 — 인접 같은 색 픽셀의 평균으로 빠진 값을 추정

MHC(Malvar-He-Cutler) 알고리즘

Bilinear의 개선 버전으로, Microsoft Research에서 2004년에 발표했다. 단순 평균 대신 5x5 커널을 사용하고, 그래디언트(기울기) 정보를 반영하여 에지 부근에서의 보간 정확도를 높인다. 연산량은 Bilinear보다 약간 많지만, 하드웨어 구현이 가능한 수준이라 많은 카메라 SDK에서 기본 디모자이킹으로 채택한다.

실무에서의 선택 기준 머신비전에서는 속도가 중요하면 Bilinear, 색 정확도가 중요하면 MHC를 선택한다. AHD나 DCB는 오프라인 분석이나 사진 후처리에 적합하다. 대부분의 산업용 카메라 SDK는 Bilinear과 MHC를 모두 지원한다.

디모자이킹 아티팩트

디모자이킹은 본질적으로 없는 정보를 추정하는 과정이므로, 완벽할 수 없다. 특히 고주파 패턴이나 급격한 에지 부근에서 다양한 아티팩트가 발생한다.

지퍼링 (Zippering)

에지를 따라 톱니 모양의 색상 줄무늬가 나타나는 현상. 수평/수직 에지에서 보간 방향이 잘못 선택되면 발생한다.

  • 원인: 에지 방향과 수직인 방향으로 보간
  • 대책: 에지 감지 기반 알고리즘(MHC, AHD) 사용

모아레 (Moire)

미세한 반복 패턴(직물, 격자)에서 원래 없던 물결무늬가 나타나는 현상. Bayer 격자의 샘플링 주파수보다 높은 공간 주파수가 원인이다.

  • 원인: 나이퀴스트 주파수 초과 (앨리어싱)
  • 대책: 광학 로우패스 필터(OLPF) 사용

컬러 프린징 (Color Fringing)

밝고 어두운 영역의 경계에서 원래 없는 색상 번짐이 나타나는 현상. False color라고도 한다.

  • 원인: 인접 채널 간 상관관계 무시한 보간
  • 대책: 색차(chrominance) 기반 후처리

해상도 저하

보간 자체가 일종의 로우패스 필터 역할을 하여, 원본 센서 해상도보다 실효 해상도가 낮아진다.

  • 원인: 보간에 의한 평활화(smoothing)
  • 대책: 샤프닝 후처리, 고급 알고리즘 사용
OLPF (Optical Low-Pass Filter) 일부 카메라는 센서 앞에 광학 로우패스 필터를 배치하여 모아레와 앨리어싱을 억제한다. 단, OLPF는 전체적인 선명도를 약간 저하시키므로, 머신비전 카메라에서는 OLPF를 제거한 모델이 많다.

모노 vs 컬러 센서

머신비전에서는 모노(Monochrome) 센서가 컬러 센서보다 선호되는 경우가 많다. 그 핵심 이유는 Bayer 패턴의 근본적 한계에 있다.

모노 센서의 핵심 장점

모노 센서에는 Bayer 패턴(컬러 필터)이 없다. 따라서 각 픽셀이 입사광의 전체 파장을 받아들여, 동일 센서 대비 더 높은 감도, 해상도, 속도를 제공한다.

항목 모노 센서 컬러(Bayer) 센서
감도(Sensitivity) 높음 — 필터 없이 전체 광량 수집 낮음 — CFA가 약 2/3 광량 차단
공간 해상도 센서 원래 해상도 그대로 디모자이킹으로 실효 해상도 저하
전송 속도 1채널 데이터 — 3배 작은 데이터량 RGB 3채널 또는 Raw 후 변환 필요
아티팩트 디모자이킹 불필요 — 아티팩트 없음 지퍼링, 모아레, 컬러 프린징 가능
색상 정보 없음 — 밝기만 측정 있음 — RGB 풀컬러
주요 용도 치수 측정, 결함 검출, OCR 색상 분류, 외관 검사, 식품 검사

모노 선택 시나리오

  • 치수/형상 측정 — 밝기 대비만 필요
  • 표면 결함(스크래치, 크랙) 검출
  • OCR/바코드 — 흑백 대비 충분
  • 고속 라인스캔 — 데이터량 최소화
  • NIR/UV 특수 조명 — 단파장 필터 사용

컬러 선택 시나리오

  • 색상 기반 분류 — 과일 숙성도, 불량 색
  • 외관 검사 — 도장, 인쇄물 색상 확인
  • 식품 검사 — 이물질 색상 구분
  • 의약품 — 정제/캡슐 색상 확인
  • PCB 검사 — 납땜 색상으로 품질 판별
모노 + 컬러 조명 조합 모노 센서에 특정 파장의 조명(예: Red LED)을 사용하면, 해당 파장에서의 반사 특성을 고감도로 촬영할 수 있다. 이 조합은 컬러 센서보다 특정 검사에서 더 높은 대비(contrast)를 제공한다.

실무 팁

SDK에서 Bayer 포맷 설정

카메라 SDK에서 PixelFormat 레지스터를 설정할 때, Bayer Raw 출력과 RGB 출력 중 선택할 수 있다. GenICam 표준 픽셀 포맷 이름은 다음과 같다.

BayerRG8 RGGB 패턴, 8비트 Raw 출력. 디모자이킹은 호스트 PC에서 수행.
BayerGR8 GRBG 패턴, 8비트 Raw 출력.
BayerGB8 GBRG 패턴, 8비트 Raw 출력.
BayerBG8 BGGR 패턴, 8비트 Raw 출력.
RGB8 카메라 내부에서 디모자이킹 완료된 8비트 RGB 출력. 대역폭 3배 사용.
BayerRG12 RGGB 패턴, 12비트 Raw. 더 넓은 다이내믹 레인지 제공.

Raw vs RGB 출력 선택

항목 Raw(Bayer) 출력 RGB 출력
데이터 크기 1채널 (작음) 3채널 (3배)
전송 대역폭 적게 사용 — 높은 프레임률 가능 많이 사용 — 프레임률 감소
디모자이킹 위치 호스트 PC (소프트웨어) 카메라 내부 (FPGA)
디모자이킹 품질 PC에서 고급 알고리즘 적용 가능 카메라 FPGA의 고정 알고리즘
CPU 부하 높음 — PC에서 변환 필요 낮음 — 바로 사용 가능
권장 상황 고속 촬영, 대역폭 제한, 후처리 필요 간단한 구성, 실시간 디스플레이
GigE Vision에서 RGB 출력 주의 GigE Vision(1Gbps)에서 RGB8 포맷을 사용하면 데이터가 3배로 늘어나 프레임률이 크게 떨어진다. 예를 들어 5MP 카메라의 경우 BayerRG8은 약 23fps가 가능하지만, RGB8은 약 7fps로 떨어진다. 고속이 필요하면 반드시 Raw 출력 후 PC에서 디모자이킹하자.

화이트 밸런스와 디모자이킹

디모자이킹 전후로 화이트 밸런스(White Balance) 보정이 필요하다. 조명의 색온도에 따라 R, G, B 채널의 게인을 조절하여 흰색이 실제로 흰색으로 보이도록 한다.

화이트 밸런스 적용 순서 일반적으로 화이트 밸런스는 디모자이킹 전(Raw 단계)에 적용하는 것이 좋다. Raw 데이터에서 각 채널의 게인을 조절한 후 디모자이킹하면, 보간 과정에서 채널 간 불균형으로 인한 컬러 아티팩트를 줄일 수 있다.
