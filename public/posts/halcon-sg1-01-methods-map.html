이 장의 역할

HALCON Solution Guide I의 첫 장은 HALCON의 개별 기능을 "어떤 문제에 어떤 방법을 써야 하는가"라는 관점에서 분류합니다. 중상급 사용자에게 중요한 것은 각 항목을 독립 기능으로 외우는 것이 아니라, 실제 검사 문제를 어떤 처리 블록으로 분해할지 판단하는 것입니다.

머신비전 문제 유형별 HALCON 접근

2.1 Color Inspection — 색상 검사

색상 검사는 단순히 RGB 값을 비교하는 작업이 아닙니다. 실제로는 조명, 카메라 white balance, 표면 반사, 노출 변화에 의해 색상 분포가 크게 달라집니다.

일반적인 처리 흐름

Color image acquisition
→ color space 변환
→ ROI 제한
→ 색상 채널 또는 색상 거리 계산
→ threshold / classification
→ area 또는 defect feature 판정

주요 기술 포인트

  • RGB는 조명 세기 변화에 취약합니다.
  • HSV는 hue 기반 구분에 유리하지만 low saturation 영역에서 불안정합니다.
  • Lab 계열 색공간은 색차 계산에 유리합니다.
  • 단일 threshold보다 reference color와의 distance 기반 판단이 안정적인 경우가 많습니다.
  • 색상 검사는 반드시 조명 repeatability를 먼저 확보해야 합니다.

실무 실패 원인

증상 가능 원인
양품 색상이 매번 달라짐 조명 색온도, 노출, white balance
특정 위치만 색상 오검출 조명 균일도, 렌즈 shading
광택 표면에서 불안정 specular reflection
threshold margin 부족 색공간 선택 또는 조명 문제

2.2 Completeness Check — 완전성/누락 검사

완전성 검사는 부품이 있어야 할 위치에 존재하는지, 조립이 완성되었는지 판단하는 문제입니다.

대표 접근

  1. 고정 ROI + blob feature
  2. 기준 위치 matching + 동적 ROI
  3. 차영상 또는 reference 비교
  4. edge/contour 존재 여부
  5. classification 또는 deep learning

중상급 관점의 핵심

고정 지그가 매우 안정적이면 고정 ROI로 충분할 수 있습니다. 하지만 실제 라인에서는 위치 편차, 회전, 높이 변화가 있으므로 다음 구조가 더 안정적입니다.

기준 형상 matching
→ part coordinate 생성
→ 각 부품 expected ROI 생성
→ ROI별 존재 여부 검사
→ missing / misalignment / wrong part 판정

판정 설계

누락 검사는 단순히 "면적이 작으면 NG"가 아닙니다. 다음 feature를 함께 보는 것이 좋습니다.

  • area
  • center position
  • orientation
  • compactness / circularity
  • edge count
  • contrast
  • expected region overlap ratio

2.3 Identification — 식별

식별은 객체의 ID를 읽거나 구분하는 작업입니다.

하위 유형

  • 1D barcode
  • 2D data code
  • OCR/OCV
  • pattern-based identification
  • classification-based identification

핵심 포인트

식별 문제는 알고리즘보다 image acquisition 품질이 중요합니다.

  • code module 크기
  • blur
  • quiet zone
  • contrast
  • print quality
  • perspective distortion
  • specular reflection
  • focus
디버깅 우선순위 OCR/Code는 decode 실패 후 파라미터를 만지는 것보다, 먼저 영상의 물리 조건을 확인해야 합니다.

2.4 Measuring and Comparison 2D — 2D 측정/비교

2D 측정은 이미지 평면에서 객체의 위치, 길이, 각도, 면적 등을 추출합니다.

주요 방법

방법 입력 장점 한계
Region processing Region 빠르고 직관적 경계 정밀도 제한
Edge / XLD contour Image → XLD subpixel 정밀도 contrast 의존
1D measuring 측정 ROI 빠르고 정밀 edge pair 설계 필요
2D metrology model 기반 반복 측정 구조화 설정 복잡
Geometric fitting point/contour 치수/형상 분석 이상점 영향

판단 기준

  • 면적/개수/대략 위치: Region
  • 폭/간격/edge 위치: 1D Measure
  • 원/선/사각형 등 형상 parameter: Metrology 또는 XLD fitting
  • 복잡한 윤곽 편차: XLD contour 비교

2.5 Measuring and Comparison 3D — 3D 측정/비교

3D 측정은 Z 정보 또는 world coordinate를 다룹니다.

주요 조건

  • camera calibration
  • stereo / sheet-of-light / depth sensor / focus stack 등 depth 획득 방식
  • 좌표계 정의
  • point cloud 또는 object model 처리
  • 3D pose와 2D image coordinate 연결

실무 포인트

3D에서 가장 위험한 문제는 "좌표계 착각"입니다.

  • camera coordinate
  • world coordinate
  • robot base coordinate
  • calibration plate coordinate
  • object coordinate
좌표계 문서화의 중요성 이 좌표계 간 transform을 명확히 문서화하지 않으면 후속 로봇/측정 시스템에서 오차 원인을 찾기 어렵습니다.

2.6 Object Recognition 2D / Position Recognition 2D

객체 인식과 위치 인식은 비슷해 보이지만 목적이 다릅니다.

구분 목적 예
Object Recognition 무엇인지 구분 여러 부품 중 어떤 부품인가
Position Recognition 어디 있는지 찾기 하나의 부품 위치/각도 찾기

HALCON에서는 matching이 두 목적 모두에 사용됩니다.

Matching 선택 기준

  • 객체 외곽/edge가 안정적: shape-based matching
  • gray texture가 안정적: NCC/correlation
  • perspective 변화 있음: perspective deformable/descriptor
  • 변형 있음: local deformable matching
  • 3D pose 필요: 3D matching 또는 calibrated matching

2.7 Print Inspection / Surface Inspection / Texture Inspection

이 영역은 양품 reference와의 편차를 다루는 경우가 많습니다.

Print Inspection

  • 문자/패턴 누락
  • 번짐
  • 위치 틀어짐
  • contrast 부족
  • OCR/OCV와 연계

Surface Inspection

  • scratch
  • contamination
  • dent
  • stain
  • texture defect

Texture Inspection

반복 패턴 또는 미세 질감 차이를 이용합니다. 전통적으로는 filter bank, co-occurrence, frequency feature 등을 사용하고, 최근에는 deep learning anomaly detection이 많이 사용됩니다.

핵심 설계

표면/텍스처 검사는 "정상 변동"과 "불량 변동"의 분리가 어렵습니다. 따라서 단순 threshold보다 다음이 중요합니다.

  • 조명 각도 설계
  • multi-light acquisition
  • flat-field correction
  • local background normalization
  • defect size 기준
  • false reject 허용 범위

2.8 Robot Vision

Robot Vision은 HALCON의 vision 결과를 robot coordinate와 연결하는 영역입니다.

핵심 구성

Camera calibration
→ Hand-eye calibration
→ Object pose estimation
→ Robot coordinate transform
→ Pick/place or alignment

실무 위험 포인트

  • camera가 moving인지 stationary인지
  • eye-in-hand인지 eye-to-hand인지
  • calibration target pose가 충분히 다양한지
  • robot TCP 오차가 calibration에 섞이지 않았는지
  • Z 방향 오차를 어떻게 검증할지

HALCON 방법 선택을 위한 실무 판단표

문제 1차 후보 2차 후보 주의점
부품 유무 Blob + ROI Matching + ROI 위치 편차 있으면 alignment 필요
위치/각도 Shape matching NCC/Descriptor template 품질 중요
길이/폭 1D measure XLD/Metrology calibration 필수
면적/개수 Region processing DL segmentation threshold 안정성
표면 흠집 Filtering + blob DL anomaly 조명 설계 중요
QR/DataMatrix 2D code reader preprocessing module 품질
문자 인식 OCR DL OCR segmentation 품질
3D 치수 calibrated 3D stereo/laser 좌표계 관리
로봇 피킹 pose estimation 3D matching hand-eye calibration

이 장에서 가져가야 할 핵심

핵심 요약
  • HALCON은 단일 "정답 operator"가 아니라 문제 유형별 toolset입니다.
  • 대부분 실무 시스템은 matching, ROI, segmentation, measuring을 조합합니다.
  • 중상급 설계에서는 algorithm보다 coordinate, ROI, acquisition, calibration을 먼저 봐야 합니다.
  • 문서의 각 방법은 독립 기능이 아니라 전체 pipeline의 모듈입니다.
