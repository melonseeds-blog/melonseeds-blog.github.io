1. 소개 - Claude Code란?

Claude Code의 정체

Claude Code는 Anthropic이 공식 출시한 에이전틱(agentic) CLI 코딩 도구입니다. 단순히 코드 자동완성을 제안하는 수준이 아니라, 여러분의 전체 코드베이스를 읽고, 파일을 생성/수정하고, 터미널 명령을 실행하고, Git 커밋까지 자율적으로 수행하는 코딩 에이전트입니다.

GitHub Copilot이나 Cursor가 편집기 안에서 코드 제안을 해주는 도구라면, Claude Code는 터미널에서 여러분의 지시를 받아 독립적으로 작업을 완수하는 도구입니다. 파일 구조를 파악하고, 관련 코드를 검색하고, 수정 사항을 반영하고, 테스트를 돌리는 전체 워크플로우를 하나의 대화 안에서 처리합니다.

기존 도구와의 차이점

항목 GitHub Copilot Cursor Claude Code
동작 방식 인라인 코드 제안 에디터 내 채팅 + 코드 제안 터미널 에이전트 (자율 실행)
파일 접근 현재 열린 파일 위주 프로젝트 인덱싱 전체 파일시스템 직접 접근
명령 실행 불가 제한적 셸 명령 직접 실행
Git 연동 PR 리뷰 정도 기본적 커밋, 브랜치, PR 생성까지
멀티 파일 수정 하나씩 가능하나 제한적 자유롭게 다수 파일 동시 수정
커스터마이징 제한적 .cursorrules CLAUDE.md + Hooks + 설정

사용 가능한 플랫폼

Claude Code는 다양한 환경에서 사용할 수 있습니다.

  • CLI (터미널) - 가장 기본적인 형태. npm으로 설치하여 터미널에서 직접 사용
  • VS Code 확장 - VS Code 안에서 Claude Code 패널을 열어 사용
  • JetBrains 확장 - IntelliJ, PyCharm 등 JetBrains IDE에서 사용
  • Claude Desktop - Anthropic의 데스크톱 앱에서 Claude Code 기능 통합
  • GitHub Actions - CI/CD 파이프라인에서 자동화된 코드 리뷰, 이슈 대응에 활용

현재 Claude Code는 Claude Opus 4.6 모델을 기본 탑재하고 있으며, 상황에 따라 Sonnet 4.6이나 Haiku 4.5로 전환할 수도 있습니다.

TIP Claude Code는 Max 구독(월 $100 또는 $200)으로 사용하거나, Anthropic API 키를 직접 연결하여 사용량 기반으로 과금받을 수 있습니다. 업무용으로 많이 쓴다면 API 키 방식이 유연합니다.

2. 설치 및 설정

npm으로 설치하기

Claude Code는 Node.js 18 이상 환경에서 npm으로 전역 설치합니다.

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

# Start Claude Code in your project directory
cd /path/to/your/project
claude

처음 실행하면 브라우저가 열리면서 Anthropic 계정 인증을 진행합니다. API 키를 직접 사용하려면 환경 변수를 설정합니다.

# Set API key directly
export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx"

# Or add to your shell profile (~/.bashrc, ~/.zshrc)
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx"' >> ~/.bashrc

권한 관리 (Permission Modes)

Claude Code는 파일 수정, 셸 명령 실행 등 강력한 기능을 제공하기 때문에 권한 관리가 중요합니다. 매번 허락을 구하는 것이 번거롭다면, settings.json에서 자동 승인 패턴을 설정할 수 있습니다. 

{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Glob",
      "Grep",
      "Bash(git status)",
      "Bash(git diff)",
      "Bash(git log)",
      "Bash(npm run build)",
      "Bash(npm test)"
    ],
    "deny": [
      "Bash(rm -rf)"
    ]
  }
}
주의 Bash 도구에 대한 전체 자동 승인("Bash"만 쓰는 경우)은 위험할 수 있습니다. 가능하면 Bash(git *)처럼 특정 패턴만 허용하는 것이 안전합니다.

설정 파일 구조

Claude Code의 설정은 여러 계층으로 나뉩니다.

위치 범위 용도
~/.claude/settings.json 전역 (모든 프로젝트) 기본 권한, 전역 설정
.claude/settings.json 프로젝트별 프로젝트 전용 설정, Hooks
~/.claude/CLAUDE.md 전역 지시사항 모든 프로젝트에 적용되는 규칙
./CLAUDE.md 프로젝트 루트 해당 프로젝트 전용 규칙

모델 선택

Claude Code에서 사용할 모델을 선택할 수 있습니다. 각 모델의 특성을 이해하고 적절히 사용하면 비용을 절약할 수 있습니다.

모델 특징 추천 용도
Opus 4.6 최고 성능, 복잡한 추론 아키텍처 설계, 복잡한 리팩토링, 디버깅
Sonnet 4.6 빠르고 효율적 일반 코딩, 파일 수정, 테스트 작성
Haiku 4.5 경량, 저비용 간단한 질문, 코드 검색, 서브에이전트 
# Switch model during session
/model sonnet

# Or set default model via config
claude config set model claude-sonnet-4-6-20250514

3. CLAUDE.md 프로젝트 설정

CLAUDE.md란?

CLAUDE.md는 Claude Code에게 프로젝트의 컨텍스트와 규칙을 전달하는 프로젝트 설명서입니다. 이 파일이 있으면 Claude가 매 세션 시작 시 자동으로 읽어서, 프로젝트의 기술 스택, 코딩 컨벤션, 디렉터리 구조, 빌드 명령어 등을 이해한 상태에서 작업을 시작합니다.

이것은 단순한 주석이 아니라 Claude의 행동을 직접 제어하는 지시문이므로, 잘 작성하면 코드 품질이 크게 향상됩니다.

배치 위치와 우선순위

위치 적용 범위 설명
~/.claude/CLAUDE.md 전역 모든 프로젝트에 공통 적용. 응답 언어, 인코딩 규칙 등
./CLAUDE.md (프로젝트 루트) 프로젝트 해당 프로젝트에만 적용. Git에 커밋하여 팀과 공유 가능
~/.claude/projects/{path}/CLAUDE.md 개인 프로젝트별 특정 프로젝트에 대한 개인 설정 (Git에 포함되지 않음)
하위 디렉터리의 CLAUDE.md 디렉터리별 해당 디렉터리 파일 작업 시에만 자동 로드

실전 CLAUDE.md 예시

아래는 실제 프로젝트에서 사용하는 CLAUDE.md의 구조 예시입니다.

# My Project

## Technology Stack
- Language: C++17
- IDE: Visual Studio 2022
- Build: CMake + vcpkg
- Test: GoogleTest

## Coding Convention
- Naming: PascalCase for classes, camelCase for variables
- Brace style: Allman
- Header guard: #pragma once
- File encoding: UTF-8 BOM

## Build Commands
- Build: cmake --build build --config Release
- Test: ctest --test-dir build -C Release
- Format: clang-format -i src/**/*.cpp

## Directory Structure
- src/ : source files
- include/ : public headers
- tests/ : GoogleTest files

## Rules for Claude
- Always return ErrorCode for action functions
- Use smart pointers, never raw new/delete
- Write comments in English
- Explain in Korean
- Do not modify more than 3 files at once
작성 팁 CLAUDE.md는 간결할수록 좋습니다. Claude는 매 대화 시작 시 이 파일을 전체 읽으므로, 너무 길면 컨텍스트 윈도우를 낭비합니다. 핵심 규칙과 자주 참조하는 정보만 담으세요. 경험적으로 500줄 이내가 적당합니다.

효과적인 CLAUDE.md 운영법

  • 빌드/테스트 명령어를 반드시 포함 - Claude가 코드 수정 후 자체적으로 빌드와 테스트를 돌릴 수 있습니다
  • 코딩 컨벤션을 구체적으로 - "깔끔하게 짜라"보다 "Allman 브레이스, PascalCase 클래스명"이 훨씬 효과적
  • 금지 사항을 명시 - "절대 raw new/delete 쓰지 마라", "매직 넘버 금지" 등 명확한 규칙
  • 정기적으로 업데이트 - 프로젝트가 발전하면 CLAUDE.md도 함께 갱신합니다
  • /memory 명령 활용 - 대화 중 "이것 기억해"라고 하면 자동으로 MEMORY.md에 저장됩니다

4. 핵심 도구들

도구 총람

Claude Code는 다양한 내장 도구(tool)를 통해 실제 작업을 수행합니다. 여러분이 "이 파일 수정해줘"라고 하면, Claude는 내부적으로 적절한 도구를 선택하여 실행합니다. 각 도구의 특성을 이해하면 더 효과적인 지시를 내릴 수 있습니다.

도구 기능 사용 예시
Read 파일 내용 읽기 소스 코드 확인, 설정 파일 파악
Edit 파일 부분 수정 (diff 방식) 함수 수정, 버그 픽스, 코드 추가
Write 파일 전체 생성/덮어쓰기 새 파일 생성, 설정 파일 작성
Bash 셸 명령 실행 빌드, 테스트, Git 명령, npm 스크립트
Glob 파일 패턴 검색 **/*.cpp 패턴으로 파일 목록 조회
Grep 파일 내용 검색 (ripgrep) 함수 정의 찾기, 특정 문자열 검색
Agent 서브 에이전트 생성 병렬 작업 분배, 탐색 위임
TodoWrite 작업 목록 관리 복잡한 태스크의 진행 상태 추적
WebFetch 웹 페이지 가져오기 문서 참조, API 응답 확인
WebSearch 웹 검색 최신 라이브러리 버전 확인, 에러 해결법 검색

파일 작업: Read / Edit / Write

파일 작업은 Claude Code의 핵심입니다. Read로 현재 코드를 파악하고, Edit으로 특정 부분만 수정하거나, Write로 새 파일을 생성합니다.

중요한 특성:

  • Edit은 문자열 매칭 기반 - 수정할 부분의 원본 텍스트가 정확히 일치해야 합니다
  • Write는 파일 전체를 덮어쓰므로, 기존 파일 수정 시에는 Edit이 더 안전합니다
  • Read는 이미지(PNG, JPG)와 PDF도 읽을 수 있습니다
알아두기 Claude Code에게 "이 파일 수정해"라고 하면, 자동으로 Read로 먼저 읽고 Edit으로 수정합니다. 명시적으로 도구 이름을 지정할 필요는 없지만, 도구의 동작 원리를 알면 디버깅 시 유용합니다.

셸 실행: Bash

Bash 도구는 터미널 명령을 직접 실행합니다. 빌드, 테스트, Git 작업, 패키지 설치 등 거의 모든 셸 작업을 수행할 수 있습니다. 

# Claude Code can run these via Bash tool:
npm run build          # Build project
npm test               # Run tests
git status             # Check Git state
python script.py       # Execute scripts
docker compose up -d   # Start containers

주의: Bash 실행 사이에 작업 디렉터리(cwd)는 초기화됩니다. 따라서 Claude Code는 항상 절대 경로를 사용하거나, 한 번의 명령에 &&로 체인합니다.

검색: Glob / Grep

대규모 코드베이스에서 원하는 파일이나 코드를 빠르게 찾는 것은 매우 중요합니다.

  • Glob - 파일명 패턴으로 검색. **/*.h, src/**/*.cpp 등
  • Grep - 파일 내용을 정규식으로 검색. ripgrep 기반으로 매우 빠름
# Examples of what Claude does internally:
# Glob: find all header files
# pattern: "**/*.h"

# Grep: find function definition
# pattern: "ErrorCode\s+ApplyBlur"
# type: "cpp"
효과적인 지시법 "ApplyBlur 함수가 어디에 정의되어 있는지 찾아줘"라고 하면 Claude가 알아서 Grep을 사용합니다. 하지만 "cpp 파일에서 ErrorCode를 반환하는 모든 함수를 찾아줘"처럼 구체적으로 말하면 더 정확한 검색이 가능합니다.

5. Hooks 시스템

Hooks란?

Hooks는 Claude Code가 특정 도구를 실행하기 전이나 후에 자동으로 셸 명령을 실행하는 시스템입니다. Git의 pre-commit hook과 비슷한 개념이지만, Claude Code의 모든 도구 실행에 적용할 수 있습니다.

예를 들어:

  • 파일 저장 후 자동으로 clang-format 실행
  • 커밋 전에 린트 체크
  • 특정 파일 수정 시 관련 테스트 자동 실행
  • 작업 완료 시 데스크톱 알림

Hook 종류

Hook 타입 실행 시점 용도
PreToolUse 도구 실행 전 유효성 검사, 실행 차단, 입력 검증
PostToolUse 도구 실행 후 자동 포맷, 린트, 테스트 실행
Notification 알림 발생 시 데스크톱 알림, 사운드 재생
UserPromptSubmit 사용자 입력 전송 시 컨텍스트 자동 추가, 입력 전처리

Hook 설정 예시

Hooks는 프로젝트의 .claude/settings.json에 설정합니다. 

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "clang-format -i $CLAUDE_FILE_PATH"
          }
        ]
      },
      {
        "matcher": "Bash(git commit)",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint"
          }
        ]
      }
    ],
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "notify-send 'Claude Code' '$CLAUDE_NOTIFICATION'"
          }
        ]
      }
    ]
  }
}

Hook에서 사용 가능한 환경 변수

환경 변수 설명
$CLAUDE_TOOL_NAME 실행된 도구 이름 (예: Edit, Bash)
$CLAUDE_TOOL_INPUT 도구에 전달된 입력 (JSON)
$CLAUDE_FILE_PATH 작업 대상 파일 경로
$CLAUDE_NOTIFICATION 알림 메시지 내용
실전 활용 C++ 프로젝트에서 파일 수정 후 자동으로 clang-format을 돌리면, Claude가 생성한 코드가 항상 프로젝트의 포맷 규칙을 따르게 됩니다. CLAUDE.md에 코딩 컨벤션을 쓰는 것과 병행하면 코드 품질이 크게 향상됩니다.

6. 멀티 에이전트 패턴

서브 에이전트란?

Claude Code는 복잡한 작업을 처리할 때 서브 에이전트(subagent)를 생성하여 작업을 분배할 수 있습니다. 메인 에이전트가 전체 작업을 조율하고, 서브 에이전트가 개별 작업을 병렬로 처리하는 구조입니다.

서브 에이전트는 독립적인 컨텍스트에서 실행되므로, 메인 에이전트의 컨텍스트 윈도우를 절약하면서 대규모 작업을 처리할 수 있습니다.

에이전트 유형

유형 역할 특징
범용 에이전트 파일 수정, 코드 생성 모든 도구 사용 가능
Explore 에이전트 코드 탐색, 정보 수집 읽기 전용 도구만 사용
Plan 에이전트 계획 수립 코드를 읽고 분석한 뒤 실행 계획 제안

Worktree를 이용한 안전한 병렬 작업

여러 서브 에이전트가 동시에 같은 파일을 수정하면 충돌이 발생합니다. 이를 방지하기 위해 Git Worktree를 활용한 격리된 작업 환경을 사용합니다. 

# Claude Code can create isolated worktrees
# Each subagent works in its own worktree
# Changes are merged back to the main branch

# Internal flow:
# 1. Main agent creates worktree
# 2. Subagent works in isolated directory
# 3. Changes are committed in worktree
# 4. Main agent merges results

실전 예시: 블로그 글 병렬 생성

이 블로그(Melon Seeds)에서 실제로 사용하는 패턴입니다. 여러 블로그 포스트를 동시에 작성해야 할 때, 메인 에이전트가 각 포스트를 서브 에이전트에게 위임합니다. 

# User prompt:
# "3D Vision, Stereo Matching, Point Cloud 글 3개를 동시에 작성해줘"

# Claude Code internally:
# 1. Spawns 3 subagents in parallel
# 2. Each subagent creates one blog post
# 3. Main agent coordinates and verifies results
# 4. All 3 posts are created simultaneously
알아두기 서브 에이전트는 Sonnet 모델을 사용하여 비용을 절약합니다. 메인 에이전트(Opus)가 전략을 세우고, 서브 에이전트(Sonnet)가 실행하는 구조가 가장 효율적입니다.

7. 실전 워크플로우 예시

예시 1: 대규모 리팩토링

프로젝트 전체에서 함수 이름을 변경하거나, 패턴을 일괄 수정해야 하는 경우입니다. 

# User: "프로젝트 전체에서 getCwd를 getCurrentWorkingDirectory로 변경해줘"

# Claude Code flow:
# 1. Grep으로 getCwd 사용처 전체 검색
# 2. TodoWrite로 수정할 파일 목록 정리
# 3. 각 파일에 대해 Edit으로 수정
# 4. git diff로 변경사항 확인
# 5. 빌드 + 테스트로 검증

핵심은 Claude가 검색 -> 계획 -> 수정 -> 검증의 사이클을 자동으로 수행한다는 점입니다. 수십 개 파일에 걸친 리팩토링도 한 번의 대화로 완료할 수 있습니다.

예시 2: 블로그 포스트 자동화

이 블로그 자체가 Claude Code로 작성되고 있습니다. 실제 워크플로우를 소개합니다. 

# Step 1: Create post file
# "claude-code-practical.html 블로그 글을 작성해줘"

# Step 2: Claude reads existing templates
# - git-basics.html 등 기존 글의 구조 파악
# - CLAUDE.md에서 블로그 규칙 확인

# Step 3: Generate content
# - HTML 템플릿 + 한국어 콘텐츠 생성
# - Prism.js 코드 하이라이팅 포함
# - UTF-8 BOM 인코딩으로 저장

# Step 4: Update sidebar/index
# - posts.js에 새 글 데이터 추가
# - 카테고리별 정렬 확인

예시 3: 코드 리뷰 및 버그 수정

PR 리뷰나 버그 수정도 Claude Code의 강력한 활용 사례입니다. 

# Code review
# User: "이 PR을 리뷰해줘"
# Claude: git diff, 코드 분석, 개선 사항 제안

# Bug fix
# User: "ImageProcessor에서 null 체크 없이 크래시가 발생해"
# Claude Code flow:
# 1. Grep으로 ImageProcessor 관련 코드 검색
# 2. Read로 해당 파일 분석
# 3. 버그 원인 파악 (null 체크 누락)
# 4. Edit으로 수정
# 5. 테스트 코드 추가
# 6. 빌드 + 테스트 실행으로 검증

예시 4: 테스트 코드 생성

기존 코드에 대한 테스트를 자동으로 생성하는 것도 매우 유용합니다. 

# User: "ImageProcessor 클래스에 대한 GoogleTest를 작성해줘"

# Claude Code flow:
# 1. Read ImageProcessor.h / .cpp
# 2. Analyze public API (methods, parameters)
# 3. Generate test cases:
#    - Normal input test
#    - Edge case test (null image, zero kernel)
#    - Error handling test
# 4. Write test file
# 5. Run tests to verify they pass
테스트 생성 팁 "해피 케이스만 테스트하지 말고, 엣지 케이스와 에러 케이스도 포함해줘"라고 지시하면 더 견고한 테스트를 생성합니다. CLAUDE.md에 테스트 가이드라인을 넣어두면 매번 지시할 필요가 없습니다.

8. 팁과 주의사항

권한 관리 전략

매번 "Allow?"에 y를 누르는 것이 귀찮다면, 자주 쓰는 도구는 자동 승인으로 설정하세요. 하지만 Bash에 대한 무제한 허용은 피하는 것이 좋습니다. 

{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Glob",
      "Grep",
      "Bash(git *)",
      "Bash(npm run *)",
      "Bash(cmake *)",
      "Bash(python -c *)",
      "Bash(ls *)",
      "Bash(cat *)",
      "Bash(mkdir *)",
      "WebSearch"
    ]
  }
}

컨텍스트 윈도우 관리

Claude Code의 컨텍스트 윈도우는 약 200K 토큰입니다. 긴 대화를 하다 보면 컨텍스트가 가득 차서 성능이 저하될 수 있습니다.

  • /compact - 대화 내용을 요약하여 컨텍스트를 확보합니다. 가장 자주 쓰는 명령입니다.
  • 자동 압축 - 컨텍스트가 95% 이상 차면 자동으로 압축이 실행됩니다
  • 새 세션 시작 - 작업 성격이 완전히 바뀌면 새 세션을 시작하는 것이 효율적입니다
  • 서브 에이전트 활용 - 큰 작업은 서브 에이전트에게 위임하여 메인 컨텍스트를 절약합니다
주의 컨텍스트가 가득 찬 상태에서 계속 작업하면 Claude가 이전 대화 내용을 잊어서 같은 실수를 반복하거나, 이미 수정한 코드를 다시 수정하려 할 수 있습니다. /compact를 습관적으로 사용하세요.

비용 인식

Claude Code는 API 사용량에 따라 과금됩니다. 모델별 비용 차이가 크므로 전략적으로 사용하세요.

모델 입력 비용 (1M 토큰) 출력 비용 (1M 토큰) 권장 용도
Opus 4.6 $15 $75 복잡한 아키텍처, 디버깅
Sonnet 4.6 $3 $15 일반 코딩, 리팩토링
Haiku 4.5 $0.80 $4 간단한 질문, 검색
비용 절약 팁 일반적인 코딩 작업은 /model sonnet으로 전환하여 처리하고, 복잡한 설계나 디버깅이 필요할 때만 Opus를 사용하세요. Max 구독($200/월)을 쓰면 Opus도 무제한 사용 가능합니다.

유용한 슬래시 명령어

명령어 기능
/compact 대화 내용 압축하여 컨텍스트 확보
/model 사용 모델 전환
/memory 기억할 내용을 MEMORY.md에 저장
/config 설정 확인 및 변경
/clear 대화 히스토리 초기화
/help 도움말 표시
/commit 변경사항 커밋 (메시지 자동 생성)
/review-pr PR 코드 리뷰

Claude Code를 쓰지 말아야 할 때

Claude Code가 만능은 아닙니다. 다음과 같은 상황에서는 직접 작업하는 것이 더 나을 수 있습니다.

  • 보안에 민감한 코드 - 인증 토큰, API 키, 암호화 로직 등은 직접 작성
  • 성능 크리티컬한 알고리즘 - 최적화가 핵심인 코드는 전문가의 직접 튜닝이 필요
  • 회사 기밀 코드 - API를 통해 Anthropic 서버로 코드가 전송되므로, 보안 정책을 확인하세요
  • 복잡한 UI/UX 작업 - 시각적 디자인은 WYSIWYG 도구가 더 적합
  • 한 줄짜리 수정 - 간단한 수정은 직접 편집기에서 하는 것이 더 빠릅니다

최고의 활용법 정리

  1. CLAUDE.md를 정성껏 작성하세요 - 이것 하나로 코드 품질이 결정됩니다
  2. 구체적으로 지시하세요 - "좋은 코드 짜줘"보다 "Allman 스타일, ErrorCode 반환, GoogleTest 포함"
  3. 검증 단계를 포함하세요 - "수정하고 빌드해서 에러 없는지 확인해"
  4. 점진적으로 작업하세요 - 한 번에 10개 파일 수정보다, 3개씩 나눠서
  5. /compact를 자주 쓰세요 - 컨텍스트 관리가 생산성의 핵심입니다
  6. 모델을 전략적으로 선택하세요 - 작업 복잡도에 맞는 모델을 사용
  7. Hooks로 자동화하세요 - 반복 작업은 Hook으로 처리
  8. 결과를 항상 확인하세요 - AI가 생성한 코드를 무조건 신뢰하지 마세요
