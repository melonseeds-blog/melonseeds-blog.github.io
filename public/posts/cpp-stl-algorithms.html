알고리즘 분류

<algorithm> 헤더는 100여 개의 함수를 제공한다. 카테고리로 묶어 두면 필요한 함수를 빠르게 떠올릴 수 있다.

카테고리역할대표 함수
비변경(non-modifying)읽기만 함find, count, all_of, equal
변경(modifying)요소를 수정/이동copy, transform, replace, fill
정렬(sorting)순서 재배치sort, stable_sort, partial_sort, nth_element
탐색(searching)정렬된 시퀀스 이진 탐색lower_bound, upper_bound, binary_search
집합(set)정렬 시퀀스 집합 연산set_intersection, set_union, set_difference
수치(numeric)<numeric>에 별도 위치accumulate, reduce, inner_product, iota

자주 쓰는 조합 패턴

erase-remove idiom

STL의 remove_if는 이름과 달리 실제로 요소를 지우지 않는다. 조건에 맞지 않는 요소를 앞으로 옮기고, "새로운 끝" 반복자만 반환한다. 컨테이너 크기는 그대로다. 따라서 물리적으로 지우려면 erase를 함께 호출해야 한다.

#include <vector>
#include <algorithm>

std::vector<int> v = {1, 2, 3, 4, 5, 6};

// 짝수만 제거
v.erase(
    std::remove_if(v.begin(), v.end(),
        [](int x) { return x % 2 == 0; }),
    v.end()
);
// v = {1, 3, 5}
remove만 호출하면 버그다 std::remove_if(v.begin(), v.end(), pred);만 쓰면 컨테이너 크기는 그대로고 뒤쪽에 쓰레기 값이 남는다. 반드시 erase로 마무리해야 한다.

C++20부터는 std::erase_if가 추가되어 한 줄로 끝낼 수 있다.

// C++20
std::erase_if(v, [](int x) { return x % 2 == 0; });

sort + unique — 중복 제거

std::unique는 인접한 중복만 제거한다. 따라서 먼저 sort로 같은 값을 모아 둔 뒤 호출해야 한다. 마무리는 erase-remove와 동일하다.

std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3};

std::sort(v.begin(), v.end());
v.erase(std::unique(v.begin(), v.end()), v.end());
// v = {1, 2, 3, 4, 5, 6, 9}

transform + back_inserter

한 컨테이너의 모든 요소를 변환해 새 컨테이너로 옮길 때 쓴다.

std::vector<int> src = {1, 2, 3, 4};
std::vector<int> dst;
dst.reserve(src.size());  // 재할당 방지

std::transform(src.begin(), src.end(),
               std::back_inserter(dst),
               [](int x) { return x * x; });
// dst = {1, 4, 9, 16}
reserve를 잊지 말자 back_inserter는 매 삽입마다 push_back을 호출한다. 미리 reserve하지 않으면 재할당이 여러 번 일어난다.

accumulate — 합/곱/문자열 연결

#include <numeric>

std::vector<int> v = {1, 2, 3, 4};

int sum  = std::accumulate(v.begin(), v.end(), 0);          // 10
int prod = std::accumulate(v.begin(), v.end(), 1,
                           std::multiplies<int>());          // 24

// 문자열 연결
std::vector<std::string> words = {"hello", "stl", "world"};
std::string joined = std::accumulate(
    words.begin(), words.end(), std::string(),
    [](const std::string& a, const std::string& b) {
        return a.empty() ? b : a + ", " + b;
    });
// "hello, stl, world"
accumulate vs reduce (C++17) accumulate는 왼쪽부터 순차 적용을 보장한다. std::reduce는 결합 법칙을 가정하고 순서를 자유롭게 바꿀 수 있어 병렬화가 가능하다. 부동소수 합처럼 순서에 따라 결과가 달라지는 경우는 결과가 달라질 수 있다.

copy_if — 조건부 복사

std::vector<int> src = {1, 2, 3, 4, 5, 6};
std::vector<int> even;

std::copy_if(src.begin(), src.end(),
             std::back_inserter(even),
             [](int x) { return x % 2 == 0; });
// even = {2, 4, 6}

partition / stable_partition

술어를 만족하는 요소를 앞으로, 나머지를 뒤로 보낸다. 분할 지점 반복자를 반환한다.

std::vector<int> v = {1, 2, 3, 4, 5, 6};

auto mid = std::partition(v.begin(), v.end(),
    [](int x) { return x % 2 == 0; });
// 예: v = {6, 2, 4, 3, 5, 1}, mid는 3을 가리킴
// 짝수는 [begin, mid), 홀수는 [mid, end)

// 안정성이 필요하면 stable_partition
// 짝수끼리, 홀수끼리 원래 순서를 유지한다.

lower_bound / upper_bound / equal_range

정렬된 시퀀스에서만 정상 동작하는 이진 탐색이다. O(log n).

함수의미
lower_bound(v)v 이상인 첫 위치
upper_bound(v)v 초과인 첫 위치
equal_range(v){lower_bound, upper_bound} pair
std::vector<int> v = {1, 2, 4, 4, 4, 5, 9};

auto lb = std::lower_bound(v.begin(), v.end(), 4);  // index 2
auto ub = std::upper_bound(v.begin(), v.end(), 4);  // index 5
int count4 = ub - lb;                                // 3

auto [first, last] = std::equal_range(v.begin(), v.end(), 4);
// 4가 등장하는 구간 [first, last)

set_intersection / set_union / set_difference

두 입력이 정렬되어 있어야 한다. 컨테이너가 std::set일 필요는 없고, 정렬된 vector도 가능하다.

std::vector<int> a = {1, 2, 3, 4, 5};
std::vector<int> b = {3, 4, 5, 6, 7};
std::vector<int> out;

std::set_intersection(a.begin(), a.end(),
                      b.begin(), b.end(),
                      std::back_inserter(out));
// out = {3, 4, 5}

out.clear();
std::set_difference(a.begin(), a.end(),
                    b.begin(), b.end(),
                    std::back_inserter(out));
// out = {1, 2}  (a에 있고 b에 없는 것)

술어 함수 작성 팁

알고리즘에 넘기는 조건 함수를 술어(predicate)라 부른다. 세 가지 형태를 비교한다.

형태특징언제
함수 객체 (functor)상태 보유 가능, 인라인화 잘 됨재사용, 상태 필요
람다현장에서 짧게 작성, 캡처 가능대부분의 경우 (권장)
std::function타입 소거, 호출 시 간접 비용다형 저장이 필요할 때만
// 함수 객체
struct GreaterThan
{
    int threshold;
    bool operator()(int x) const { return x > threshold; }
};
std::count_if(v.begin(), v.end(), GreaterThan{10});

// 람다 — 가장 흔한 선택
std::count_if(v.begin(), v.end(),
              [t = 10](int x) { return x > t; });

// std::function — 호출 비용이 있어 핫 루프에는 비권장
std::function<bool(int)> pred = [](int x) { return x > 10; };
std::count_if(v.begin(), v.end(), pred);
술어는 부작용이 없어야 한다 술어는 같은 입력에 대해 항상 같은 결과를 내야 한다. 카운터를 증가시키거나 외부 상태를 변경하면 알고리즘이 어떤 순서로 호출하는지에 따라 결과가 달라진다.

back_inserter / front_inserter / inserter

출력 반복자(output iterator) 어댑터다. 알고리즘이 결과를 쓸 때 컨테이너의 어느 위치에 어떻게 삽입할지를 정한다.

어댑터호출하는 함수지원 컨테이너
back_inserter(c)push_backvector, deque, list, string
front_inserter(c)push_frontdeque, list, forward_list
inserter(c, it)insert(it, ...)거의 모든 컨테이너 (set, map 포함)
std::vector<int> src = {1, 2, 3};

std::list<int> lst;
std::copy(src.begin(), src.end(), std::front_inserter(lst));
// lst = {3, 2, 1}  (앞에 하나씩 끼워 넣어 뒤집힘)

std::set<int> s;
std::copy(src.begin(), src.end(), std::inserter(s, s.end()));
// s = {1, 2, 3}

for_each vs range-based for

대부분의 경우 range-based for가 더 간결하고 가독성이 좋다. std::for_each를 굳이 쓸 만한 상황은 제한적이다.

// range-based for — 99%의 경우 이걸로 충분하다
for (auto& x : v)
{
    process(x);
}

// for_each — 일부 구간에만 적용하거나 병렬 정책을 쓸 때
std::for_each(v.begin() + 2, v.begin() + 5,
              [](int& x) { x *= 2; });

// 병렬 적용
std::for_each(std::execution::par, v.begin(), v.end(),
              [](int& x) { x = heavy_compute(x); });

min_element / max_element / minmax_element

반복자를 반환한다. 값이 필요하면 역참조해야 한다. 비교자를 넘겨 커스텀 기준으로 찾을 수 있다.

std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6};

auto itMin = std::min_element(v.begin(), v.end());  // → 1
auto itMax = std::max_element(v.begin(), v.end());  // → 9

// 한 번에 둘 다 — 비교 횟수가 절반에 가깝다
auto [itLo, itHi] = std::minmax_element(v.begin(), v.end());

// 절대값 기준 최댓값
struct Point { int x, y; };
std::vector<Point> pts = { {1, 2}, {-5, 3}, {2, 4} };
auto farthest = std::max_element(pts.begin(), pts.end(),
    [](const Point& a, const Point& b) {
        return (a.x*a.x + a.y*a.y) < (b.x*b.x + b.y*b.y);
    });

count / count_if / all_of / any_of / none_of

술어를 적용해 컨테이너의 상태를 한 줄로 검사한다. 명시적 루프보다 의도가 분명하다.

std::vector<int> v = {1, 2, 3, 4, 5};

int n3 = std::count(v.begin(), v.end(), 3);                // 1
int nEven = std::count_if(v.begin(), v.end(),
                          [](int x) { return x % 2 == 0; }); // 2

bool allPos  = std::all_of (v.begin(), v.end(),
                            [](int x) { return x > 0; });   // true
bool anyEven = std::any_of (v.begin(), v.end(),
                            [](int x) { return x % 2 == 0; }); // true
bool noneNeg = std::none_of(v.begin(), v.end(),
                            [](int x) { return x < 0; });   // true
빈 컨테이너에서의 동작 all_of와 none_of는 빈 범위에서 true를 반환한다(공허한 참, vacuous truth). any_of는 false다. 의도와 다른 결과를 만나면 빈 범위 처리부터 의심해 보자.

C++17 병렬 알고리즘

대부분의 알고리즘은 첫 인자로 실행 정책을 받을 수 있게 확장됐다. <execution>을 포함하면 한 줄 추가로 병렬화된다.

정책의미
std::execution::seq순차 (기본)
std::execution::par병렬 (스레드 분배)
std::execution::par_unseq병렬 + 벡터화 (SIMD)
#include <execution>
#include <algorithm>

std::vector<int> v(1'000'000);
// ... 채우기 ...

// 한 줄 변경으로 병렬 정렬
std::sort(std::execution::par, v.begin(), v.end());

// 병렬 변환
std::transform(std::execution::par_unseq,
               v.begin(), v.end(), v.begin(),
               [](int x) { return x * x + 1; });
한 줄로 멀티스레드 직접 std::thread나 작업 큐를 만들 필요가 없다. 단, 술어가 데이터 경합 없이 동작해야 하고(부작용 없음), 작업량이 충분히 커야 스레드 분배 비용을 회수한다. 작은 컨테이너에는 오히려 느리다.

C++20 ranges 미리보기

C++20부터 std::ranges 네임스페이스에 동일한 알고리즘이 다시 정의됐다. 핵심 차이는 두 가지다.

  • begin/end 한 쌍 대신 컨테이너 자체를 넘긴다 — 호출이 짧아진다.
  • projection 인자 지원 — 멤버 추출이 깔끔해진다.
#include <ranges>
#include <algorithm>

struct Person { std::string name; int age; };
std::vector<Person> people = { /* ... */ };

// C++17 — 비교자 안에서 멤버 접근
std::sort(people.begin(), people.end(),
    [](const Person& a, const Person& b) { return a.age < b.age; });

// C++20 ranges — projection으로 멤버를 직접 지정
std::ranges::sort(people, {}, &Person::age);

// 파이프라인과 결합 (views)
auto adults = people
    | std::views::filter([](const Person& p) { return p.age >= 18; })
    | std::views::transform(&Person::name);
for (const auto& name : adults) { /* ... */ }

흔한 함정

remove만 호출하고 erase 안 하기

가장 흔한 실수다. 컨테이너 크기는 그대로고 뒤쪽에 미정의 값이 남는다. 항상 erase 또는 C++20 erase_if로 마무리하자.

strict weak ordering 위반

sort의 비교자는 엄격 약 순서(strict weak ordering)를 만족해야 한다. 핵심 규칙은 "cmp(a, a)는 항상 false"다. <=를 쓰면 cmp(a, a)가 true가 되어 정렬이 미정의 동작에 빠진다.

// 잘못된 예 — UB
std::sort(v.begin(), v.end(),
    [](int a, int b) { return a <= b; });  // 위반!

// 올바른 예
std::sort(v.begin(), v.end(),
    [](int a, int b) { return a < b; });
비교자는 strict less, 절대 less-or-equal이 아니다 구현체에 따라 무한 루프, 크래시, 메모리 손상까지 일어날 수 있다. 디버그 모드에서는 발견되지만 릴리스 빌드에서 묻혀 가는 경우가 많다.

정렬 안 된 컨테이너에 binary_search

binary_search, lower_bound, upper_bound는 정렬을 전제로 한다. 정렬되지 않은 컨테이너에 쓰면 잘못된 결과를 조용히 반환한다. 컴파일러는 경고하지 않는다.

reserve 없이 transform + back_inserter

출력 컨테이너가 비어 있는 채로 back_inserter를 쓰면 매 push_back마다 재할당이 일어날 수 있다. 입력 크기를 알면 reserve를 먼저 호출해 한 번에 메모리를 잡아 두자.

std::vector<int> dst;
dst.reserve(src.size());  // 이 한 줄로 재할당이 사라진다
std::transform(src.begin(), src.end(),
               std::back_inserter(dst),
               [](int x) { return x * 2; });
