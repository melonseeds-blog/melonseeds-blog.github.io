C++17 개요

C++17은 C++11/14의 흐름을 이어 문법 단순화와 표준 라이브러리 확장에 초점을 맞춘 표준이다. 혁신적 기능보다는 일상 코드의 가독성과 안전성을 높이는 변화가 많다. 데이터 흐름이 명확해지고, 컴파일 타임 메타프로그래밍이 한결 단순해졌다.

분류주요 기능
문법structured bindings, if/switch initializer, nested namespace
컴파일 타임if constexpr, fold expression, constexpr lambda
타입optional, variant, any, string_view, byte, CTAD
표준 라이브러리filesystem, 병렬 알고리즘
속성[[nodiscard]], [[maybe_unused]], [[fallthrough]]

기본 문법 개선

structured bindings — 구조화 바인딩

tuple, pair, 배열, struct를 한 번에 여러 변수로 분해한다.

std::pair<int, std::string> p{1, "one"};
auto [id, name] = p;          // id=1, name="one"

std::map<std::string, int> m{{"a", 1}, {"b", 2}};
for (const auto& [key, val] : m)
{
    std::cout << key << "=" << val << "\n";
}

struct Point { int x; int y; };
Point pt{10, 20};
auto& [x, y] = pt;            // 참조 바인딩 — 원본 수정 가능

if / switch initializer

if 안에서 변수를 선언하고 즉시 조건을 검사한다. 변수 스코프가 if 블록으로 한정되어 누수가 없다.

std::map<int, std::string> m;

if (auto it = m.find(42); it != m.end())
{
    std::cout << it->second << "\n";
}
// it는 여기서 더 이상 보이지 않는다.

switch (auto code = GetStatus(); code)
{
    case 200: /* ... */ break;
    default:  /* ... */ break;
}

nested namespace

중첩 네임스페이스를 한 줄로 선언한다.

// C++14
namespace A { namespace B { namespace C { /* ... */ } } }

// C++17
namespace A::B::C
{
    void DoSomething();
}

컴파일 타임 기능

if constexpr

컴파일 타임에 분기를 결정한다. 거짓 분기는 아예 인스턴스화되지 않으므로, 템플릿 안에서 타입별로 다른 코드를 안전하게 작성할 수 있다.

template <typename T>
auto GetValue(T t)
{
    if constexpr (std::is_pointer_v<T>)
    {
        return *t;          // T가 포인터일 때만 컴파일된다.
    }
    else
    {
        return t;
    }
}
SFINAE / tag dispatch 대체 기존엔 std::enable_if로 오버로드를 분리했지만, if constexpr로 한 함수 안에 평범한 if처럼 쓸 수 있다. 가독성이 크게 향상된다.

fold expression — 폴드 표현식

가변 인자 템플릿을 한 줄의 연산으로 펼친다.

template <typename... Args>
auto Sum(Args... args)
{
    return (args + ...);            // 우측 fold: a1 + (a2 + (a3 + ...))
}

template <typename... Args>
auto SumWithInit(Args... args)
{
    return (0 + ... + args);        // 초깃값 0
}

template <typename... Args>
bool AllTrue(Args... args)
{
    return (... && args);           // 좌측 fold AND
}

Sum(1, 2, 3, 4);                    // 10

constexpr lambda

람다 자체를 컴파일 타임에 평가할 수 있다.

constexpr auto square = [](int x) { return x * x; };
static_assert(square(5) == 25);     // 컴파일 타임 검증

타입 시스템 확장

optional / variant / any

세 가지 새로운 합타입(sum type)이 추가됐다. 자세한 사용법은 다음 글에서 다룬다.

std::optional<int> opt = 42;       // 값이 있을 수도, 없을 수도
std::variant<int, std::string> v;   // 정해진 타입 중 하나
std::any a = 3.14;                  // 어떤 타입이든 저장

std::string_view

문자열을 복사하지 않고 참조만 보유하는 비소유(non-owning) 뷰다. 함수 매개변수로 const std::string& 대신 사용하면 C 문자열, std::string, 부분 문자열을 모두 공통으로 받을 수 있다.

void Print(std::string_view sv)
{
    std::cout << sv << "\n";
}

Print("literal");                   // const char*
Print(std::string{"hello"});        // std::string
Print(std::string{"hello world"}.substr(0, 5));  // 위험!
dangling string_view 주의 string_view는 원본 문자열의 수명을 연장하지 않는다. 임시 객체로부터 만든 view를 보관하면 댕글링 참조가 된다. 멤버 변수로 저장하지 말고, 함수 호출 동안만 살아있는 매개변수로만 쓰는 편이 안전하다.

std::byte

바이트를 명시적으로 표현하는 enum class 타입이다. char나 unsigned char와 달리 산술 연산이 차단되어 의도 없이 사용되는 일을 막는다.

std::byte b{0xFF};
b = b & std::byte{0x0F};            // 비트 연산은 가능
// b + 1;  // 에러: 산술 불가

CTAD — 클래스 템플릿 인자 추론

생성자 인자만으로 템플릿 매개변수를 추론한다. make_pair, make_tuple 같은 헬퍼가 거의 불필요해졌다.

std::pair p{1, 2.0};                // pair<int, double>
std::vector v{1, 2, 3};             // vector<int>
std::lock_guard lock(m);            // lock_guard<std::mutex>

표준 라이브러리 추가

std::filesystem

드디어 표준에 파일시스템 API가 들어왔다. 경로 조작, 디렉터리 순회, 파일 정보 조회까지 한 번에 해결된다.

#include <filesystem>
namespace fs = std::filesystem;

fs::path p = "C:/data/image.bmp";
if (fs::exists(p))
{
    std::cout << "size = " << fs::file_size(p) << "\n";
}

fs::create_directory("output");

for (const auto& entry : fs::recursive_directory_iterator("C:/data"))
{
    if (entry.is_regular_file())
    {
        std::cout << entry.path() << "\n";
    }
}

병렬 알고리즘 정책

표준 알고리즘에 실행 정책을 전달해 병렬화를 요청할 수 있다. 자세한 내용은 별도의 알고리즘 활용 글에서 다룬다.

#include <execution>

std::sort(std::execution::par, v.begin(), v.end());     // 병렬 정렬
std::for_each(std::execution::par_unseq, v.begin(), v.end(),
              [](int& x) { x *= 2; });
실행 정책 종류 seq(순차), par(병렬), par_unseq(병렬 + 벡터화). 컴파일러/표준 라이브러리 구현에 따라 지원 여부가 다르므로 사용 전 확인이 필요하다.

속성과 inline variable

[[nodiscard]] / [[maybe_unused]] / [[fallthrough]]

[[nodiscard]] ErrorCode ApplyBlur(Image& img, int kernel);
// 반환값을 무시하면 컴파일러 경고

void Foo([[maybe_unused]] int debugId)
{
    // 디버그 빌드에서만 쓰는 변수 — unused 경고 억제
}

switch (state)
{
    case State::Init:
        Initialize();
        [[fallthrough]];        // 의도적 fallthrough를 명시
    case State::Run:
        Run();
        break;
}
ErrorCode 반환 함수엔 [[nodiscard]]를 붙이자 에러 코드를 무시한 채 호출하는 실수를 컴파일 타임에 잡아준다. 산업용 SDK에서 필수에 가까운 속성이다.

inline variable

변수 정의를 헤더에 둘 수 있다. ODR(One Definition Rule) 위반 없이 단일 정의가 보장되어, 헤더 전용 라이브러리에 매우 유용하다.

// constants.h
inline constexpr int kMaxImageWidth = 8192;
inline constexpr double kPi = 3.14159265358979;

// 여러 .cpp에서 include해도 중복 정의 에러 없음
