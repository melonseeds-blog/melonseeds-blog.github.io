트리거란?

트리거(Trigger)란 외부 이벤트를 기준으로 카메라의 촬영 타이밍을 제어하는 메커니즘이다. 자유 실행(Free-run) 모드에서는 카메라가 자체 클럭에 따라 연속 촬영하지만, 트리거 모드에서는 특정 신호가 들어올 때만 이미지를 획득한다.

왜 트리거가 필요한가?

생산라인에서 제품이 일정한 속도로 이동할 때, 정확한 위치에서 촬영해야 검사 정밀도가 보장된다. 트리거 없이 자유 실행하면 제품 위치와 촬영 타이밍이 어긋나 불량 오검출이 발생한다. 센서, PLC, 인코더 등 외부 장치에서 "지금 찍어라"라는 신호를 보내면, 카메라가 정확한 순간에 노출을 시작한다.

그림 1. Free-run 모드와 Trigger 모드의 차이

하드웨어 트리거 vs 소프트웨어 트리거

하드웨어 트리거 (HW Trigger)

  • 방식 — 카메라 GPIO 핀에 전기 신호 입력
  • 지연 — 수 us (마이크로초) 이내
  • 정밀도 — 매우 높음 (전기 신호 기반)
  • 소스 — 센서, PLC, 인코더, 타이머
  • 용도 — 고속 라인, 정밀 위치 동기화

소프트웨어 트리거 (SW Trigger)

  • 방식 — API 명령으로 촬영 지시
  • 지연 — 수 ms (밀리초) 수준
  • 정밀도 — 낮음 (네트워크/USB 지연 포함)
  • 소스 — PC 소프트웨어, SDK 호출
  • 용도 — 저속 검사, 수동 트리거, 테스트
항목 하드웨어 트리거 소프트웨어 트리거
지연 시간 1~20 us 1~50 ms
지터(Jitter) < 1 us 수 ms (불확정)
배선 GPIO 케이블 필요 통신 케이블만 사용
동기화 정밀도 매우 높음 낮음
구현 난이도 배선 설계 필요 API 호출만으로 간단
적합한 환경 고속 생산라인 실험실, 저속 검사
GenICam의 트리거 관련 레지스터 GenICam 표준에서는 TriggerMode, TriggerSource, TriggerActivation 등의 레지스터로 트리거를 설정한다. 예를 들어 TriggerSource = Line0이면 GPIO Line0 핀에서 하드웨어 트리거를 받겠다는 의미이다.

트리거 신호 종류

하드웨어 트리거 신호는 전압의 변화 패턴에 따라 여러 종류로 나뉜다. TriggerActivation 레지스터에서 설정하며, 센서나 PLC의 출력 특성에 맞게 선택해야 한다.

Rising Edge / Falling Edge

Level / Pulse Width

그림 3. Level 트리거와 Pulse Width 필터링
Rising Edge LOW에서 HIGH로 전환되는 순간 트리거. 가장 많이 사용되는 기본 방식.
Falling Edge HIGH에서 LOW로 전환되는 순간 트리거. NPN 센서 출력에 적합.
Level High HIGH 상태가 유지되는 동안 노출. 노출 시간을 외부에서 직접 제어할 때 사용.
Pulse Width 설정한 최소 펄스 폭 이상의 신호만 트리거로 인식. 노이즈 필터링 용도.
Any Edge 상승/하강 에지 모두에서 트리거. 인코더 분해능을 2배로 사용할 때 활용.
NPN vs PNP 센서와 트리거 방향 NPN(싱크) 센서는 감지 시 출력이 LOW로 떨어지므로 Falling Edge를 사용하고, PNP(소스) 센서는 감지 시 HIGH로 올라가므로 Rising Edge를 사용한다. 센서 출력 타입을 반드시 확인한 후 트리거 방향을 설정해야 한다.

동기화 구성

머신비전 검사에서는 카메라, 조명, 인코더가 동시에 동작해야 한다. 제품이 센서를 지나가면 PLC가 트리거를 발생시키고, 카메라는 노출을 시작하며, 조명 컨트롤러는 스트로브 조명을 점등한다. 이 과정이 마이크로초 단위로 정밀하게 맞아야 한다.

타이밍 다이어그램

그림 4. 트리거 -> 노출 -> 조명 -> 데이터 전송 타이밍 다이어그램

스트로브 조명 동기화

카메라의 ExposureActive 출력 신호를 조명 컨트롤러의 트리거 입력에 연결하면, 노출이 시작될 때 자동으로 조명이 켜진다. 이 방식을 스트로브(Strobe) 모드라 한다.

스트로브 조명의 장점 상시 점등 대비 2~5배 높은 광량으로 짧은 노출에서도 충분한 밝기를 확보할 수 있다. LED 수명도 크게 늘어나며, 열 발생도 줄어든다. 고속 라인에서는 스트로브가 사실상 필수이다.

라인스캔 인코더 동기화

라인스캔 카메라는 한 줄씩 이미지를 촬영하므로, 제품의 이동 속도에 맞춰 정확한 간격으로 라인을 획득해야 한다. 로터리 인코더가 컨베이어 벨트의 이동량을 전기 신호로 변환하여 카메라에 전달한다.

인코더 신호 (A/B/Z)

그림 6. 인코더 A/B/Z 채널 신호 파형
A 채널 기본 펄스 출력. 이동량에 비례한 펄스 수를 제공한다.
B 채널 A 채널 대비 90도 위상차. A보다 B가 앞서면 정방향, 뒤지면 역방향.
Z 채널 1회전당 1펄스 출력. 원점 복귀(Home) 및 라인스캔 이미지 시작점 동기화에 사용.

Line Rate 계산

라인스캔 카메라의 Line Rate(초당 촬영 라인 수)는 컨베이어 속도와 인코더 분해능으로 결정된다.

Line Rate 계산 공식

Line Rate = 컨베이어 속도(mm/s) / 픽셀 크기(mm/pixel)

예시: 컨베이어 속도 1000 mm/s, 원하는 분해능 0.05 mm/pixel
Line Rate = 1000 / 0.05 = 20,000 lines/s (20 kHz)

인코더 펄스로 환산하면:
인코더 분해능 = 2000 PPR (Pulse Per Revolution), 롤러 둘레 = 200 mm
1 펄스당 이동량 = 200 / 2000 = 0.1 mm/pulse
필요 분주비 = 0.1 / 0.05 = 2 (인코더 2펄스마다 1라인 촬영)

인코더 분해능과 이미지 왜곡 인코더 분해능이 부족하면 이동 방향(Y축) 이미지가 늘어나거나 줄어든다. 반대로 분해능이 과도하면 카메라의 최대 Line Rate를 초과하여 라인 누락이 발생한다. 실제 배치 전에 반드시 계산으로 검증해야 한다.

멀티 카메라 동기화

하나의 검사 스테이션에 여러 대의 카메라를 사용할 때, 모든 카메라가 동시에 촬영해야 한다. 동기화가 어긋나면 이동하는 제품의 각 면이 서로 다른 위치에서 찍혀 검사 결과가 부정확해진다.

외부 트리거 분배

가장 간단한 방법은 하나의 트리거 신호를 분배기(Trigger Distributor)로 여러 카메라에 동시에 전달하는 것이다. 전기 신호이므로 지연이 거의 없고 구현이 쉽지만, 카메라 수가 많으면 신호 품질이 저하될 수 있다.

PTP (IEEE 1588)

PTP (Precision Time Protocol)

IEEE 1588 표준으로 정의된 정밀 시각 동기화 프로토콜. 네트워크에 연결된 장치들의 시계를 서브 마이크로초(sub-us) 정밀도로 동기화한다. GigE Vision 2.0부터 지원하며, 하드웨어 배선 없이 네트워크만으로 멀티 카메라 동기화가 가능하다.

동기화 방식 정밀도 카메라 수 제한 배선 복잡도 비용
외부 트리거 분배 < 1 us 분배기 채널 수 높음 (GPIO 배선) 낮음
PTP (IEEE 1588) < 1 us 네트워크 제한 없음 낮음 (이더넷만) 중간
액션 커맨드 < 1 us (PTP 기반) 네트워크 제한 없음 낮음 (이더넷만) 중간
소프트웨어 순차 트리거 수 ms 무제한 없음 없음

GigE Vision 액션 커맨드

GigE Vision의 Action Command는 하나의 브로드캐스트 패킷으로 여러 카메라에 동시에 트리거를 전달하는 메커니즘이다. PTP로 시각이 동기화된 상태에서 Scheduled Action Command를 사용하면, 지정 시각에 모든 카메라가 정확히 같은 순간에 촬영한다.

Action Command 설정 흐름 1. 모든 카메라에 동일한 ActionDeviceKey, ActionGroupKey, ActionGroupMask를 설정한다.
2. PTP로 시각 동기화가 완료될 때까지 기다린다 (PtpStatus = Slave).
3. PC에서 ActionCommand를 브로드캐스트하면, 조건이 일치하는 모든 카메라가 동시에 트리거된다.

트러블슈팅

트리거와 동기화 시스템에서 자주 발생하는 문제와 해결 방법을 정리한다.

노출 타이밍이 안 맞는 경우

증상 이미지가 너무 밝거나 어둡다. 또는 제품이 잘려서 찍힌다.
원인 1 트리거 지연(Trigger Delay) 설정 오류. 센서 감지 시점과 제품이 FOV에 도달하는 시점 사이의 간격이 맞지 않음.
원인 2 노출 시간(Exposure Time) 과다. 고속 라인에서 긴 노출은 모션 블러를 유발.
해결 오실로스코프로 트리거 신호와 ExposureActive 신호의 타이밍을 측정하여 Trigger Delay를 조정. 노출 시간은 1/(2 x 이동속도/FOV) 이하로 설정.

조명 깜빡임 (Flicker)

증상 촬영 이미지의 밝기가 프레임마다 달라진다.
원인 1 AC 전원 조명 사용 시 50/60Hz 주파수와 노출 타이밍이 동기되지 않음.
원인 2 스트로브 조명 응답 시간 부족. LED 컨트롤러의 ON/OFF 응답이 노출 시간보다 느림.
해결 DC 전원 조명으로 교체하거나, 노출 시간을 AC 주기의 정수배(예: 50Hz -> 20ms, 10ms)로 설정. 스트로브는 응답 시간이 1us 이하인 LED 컨트롤러를 사용.

인코더 노이즈

증상 라인스캔 이미지에 불규칙한 줄무늬(라인 불균일)가 나타나거나, 이미지 길이가 일정하지 않다.
원인 1 전기 노이즈. 모터, 인버터 근처에 인코더 케이블이 지나가면서 유도 노이즈 발생.
원인 2 접지 불량. 인코더와 카메라의 접지 전위 차이로 인한 신호 왜곡.
원인 3 기계적 슬립. 인코더 롤러와 벨트 사이의 미끄러짐으로 펄스 수 불일치.
해결 쉴드 케이블 사용 및 노이즈원과 분리 배선. 차동 출력(Line Driver) 인코더 채택. 인코더 롤러에 압력 조정 스프링 장착. 카메라의 Pulse Width Filter 기능으로 짧은 노이즈 펄스 무시.
디버깅 필수 도구: 오실로스코프 트리거/동기화 문제의 대부분은 타이밍 문제이다. 오실로스코프로 트리거 입력, ExposureActive 출력, 조명 스트로브 신호를 동시에 관찰하면 문제 원인을 빠르게 파악할 수 있다. 소프트웨어 로그만으로는 us 단위 타이밍 문제를 진단하기 어렵다.
