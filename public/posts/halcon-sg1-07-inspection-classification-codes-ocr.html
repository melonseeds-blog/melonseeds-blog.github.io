Print Inspection

Print Inspection은 인쇄된 문자, 패턴, 라벨, 마킹의 품질을 확인한다.

2.1 주요 검사 항목

  • 인쇄 누락
  • 번짐
  • 끊김
  • 위치 틀어짐
  • 회전
  • contrast 부족
  • 문자 오인쇄
  • 오염/스크래치
  • code 품질 저하

2.2 접근 방식

문제 접근
위치 틀어짐 Matching / registration
인쇄 누락 reference comparison / blob
문자 읽기 OCR
code 읽기 barcode / data code reader
품질 점수 print quality verification

OCR / OCV

OCR은 문자를 읽는 것이고, OCV는 기대한 문자가 제대로 인쇄되었는지 검증하는 것이다.

3.1 OCR 기본 흐름

ROI 정렬
→ 문자 영역 segmentation
→ 문자 region 분리
→ feature extraction
→ classifier로 문자 인식
→ 문자열 후처리

3.2 OCR에서 가장 어려운 부분

알고리즘보다 segmentation 알고리즘 자체보다 문자 segmentation이 더 어렵다.

문제 예:

  • 문자끼리 붙음
  • 문자 일부 끊김
  • 배경과 contrast 낮음
  • 글자 크기 변화
  • 회전/기울어짐
  • dot matrix font
  • 인쇄 번짐
  • 반사

3.3 OCV 설계

OCV는 기대 문자열이 정해져 있으므로 OCR보다 constraint를 더 강하게 줄 수 있다.

  • 기대 문자 set 제한
  • 위치별 허용 문자 제한
  • string length 고정
  • checksum 또는 format rule 적용
  • low confidence 문자만 재검사

Classification

Classification은 feature 또는 image를 기반으로 class를 결정한다.

4.1 전통적 Feature 기반 Classification

흐름:

object/region 추출
→ feature 계산
→ classifier 학습
→ class 판정

사용 feature:

  • gray feature
  • color feature
  • shape feature
  • texture feature
  • region feature

Classifier 예:

  • Euclidean
  • Hyperbox
  • MLP
  • SVM
  • GMM
  • k-NN

4.2 Classifier 선택 관점

Classifier 특징
k-NN 단순, 데이터 기반, 느릴 수 있음
SVM feature space 구분에 강함
MLP 비선형 분류 가능
GMM 확률 분포 기반
Hyperbox 해석 쉬움

4.3 중상급 실무 포인트

알고리즘보다 데이터/feature Classifier 성능은 알고리즘 선택보다 데이터와 feature에 더 의존한다.

중요한 것:

  • class 간 feature 분리도
  • training sample 대표성
  • 조명/제품 편차 포함 여부
  • outlier 처리
  • class imbalance
  • test set 분리
  • false reject / false accept 비용

Deep Learning 기반 접근

Deep Learning은 classification, detection, segmentation, anomaly detection 등에 사용된다.

5.1 사용 적합한 경우

  • rule 기반 feature 설계가 어려움
  • defect 형태가 다양함
  • texture/외관 검사
  • 복잡한 배경
  • 사람 눈은 구분하지만 threshold로 어려움

5.2 주의점

  • 데이터 수집/라벨링 비용
  • 현장 편차 반영 필요
  • 설명 가능성 낮음
  • 작은 데이터에서 overfitting
  • 조명/광학 바뀌면 재학습 필요
  • inference time 고려

5.3 Rule 기반과 DL 조합

실무에서는 DL만 쓰기보다 rule 기반과 조합하는 경우가 많다.

matching으로 ROI 정렬
→ DL로 defect 후보 검출
→ blob/geometry로 후검증
→ rule 기반 최종 판정

2D Data Codes

HALCON은 Data Matrix ECC 200, QR Code, Micro QR, PDF417, Aztec, DotCode 등을 다룰 수 있다.

6.1 기본 흐름

create data code model
→ find data code
→ decode string
→ result / quality access

6.2 Code Reading에서 중요한 영상 조건

  • module size
  • contrast
  • quiet zone
  • focus
  • perspective distortion
  • print growth/shrink
  • damage
  • finder pattern visibility
  • illumination uniformity

6.3 Model Adaptation

비표준 code나 어려운 code를 읽기 위해 model parameter를 조정한다.

조정 예:

  • code type
  • polarity
  • module size
  • finder pattern tolerance
  • contrast
  • timeout
  • symbol shape
  • mirrored/rotated condition

6.4 Training

여러 sample을 사용해 reader parameter를 학습/조정할 수 있다. 단, training sample이 실제 변동을 대표해야 한다.

2D Code 실패 원인

증상 가능 원인
code 위치 못 찾음 quiet zone 부족, contrast 부족
찾지만 decode 실패 module damage, blur, print distortion
특정 방향만 실패 조명 반사, perspective
속도 느림 search range 과도, code type 미제한
양품인데 가끔 실패 focus/trigger/exposure 흔들림

Print Quality Verification

2D code는 단순히 읽히는지뿐 아니라 품질 grade를 평가할 수 있다.

평가 항목 예:

  • contrast
  • modulation
  • fixed pattern damage
  • axial nonuniformity
  • grid nonuniformity
  • unused error correction
  • print growth
"읽힘"과 "품질 양호"는 다른다 라인에서는 지금 읽히더라도 고객/다른 reader에서 실패할 수 있으므로 품질 평가가 중요할 수 있다.

Surface Inspection

표면 검사는 scratch, stain, dent, particle, contamination 등을 찾다.

9.1 기본 접근

illumination 설계
→ background normalization
→ defect enhancement filter
→ threshold/local threshold
→ blob feature filtering
→ defect classification

9.2 표면 검사의 핵심

핵심은 조명이다 SW로 안 되는 문제를 조명으로 해결해야 하는 경우가 많다.
  • bright field
  • dark field
  • coaxial
  • dome
  • line light
  • polarized light
  • multi-angle light

9.3 False Reject 감소

  • defect 크기 기준
  • 위치별 threshold
  • background texture model
  • repeated frame confirmation
  • morphology feature
  • DL anomaly score + rule 후처리

Texture Inspection

Texture는 반복 패턴, 섬유, 표면 질감 등에서 defect를 찾다.

전통적 방법:

  • frequency filter
  • co-occurrence matrix
  • local variance
  • Gabor-like filtering
  • gray difference
  • texture feature classification

DL 방법:

  • anomaly detection
  • patch classification
  • segmentation model
정상 sample 다양성이 핵심 정상 texture variability가 크면 defect와 정상 편차를 구분하기 어렵다.

핵심 정리

응용 검사 설계 핵심
  • OCR/Code는 acquisition 품질이 성능의 대부분을 결정한다.
  • Classification은 feature와 training data 품질이 알고리즘보다 중요하다.
  • Surface/Texture 검사는 조명 설계가 핵심이다.
  • Code는 읽힘 여부만 보지 말고 품질 grade도 고려해야 한다.
  • DL은 강력하지만 ROI, 조명, 후처리 rule과 결합해야 현장 안정성이 좋아진다.
