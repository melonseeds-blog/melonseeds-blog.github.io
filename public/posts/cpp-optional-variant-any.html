한눈에 비교

세 타입은 모두 "하나의 변수에 여러 가능성을 담는다"는 점에서 비슷하지만, 표현하는 의미가 명확히 다르다. 먼저 비교 매트릭스로 차이를 잡고 시작한다.

std::optional<T>

"값이 있을 수도 없을 수도 있다"를 타입으로 표현한다. 과거에 nullptr 반환, sentinel 값(-1, 0xFFFFFFFF) 등으로 표현하던 부재를 안전하게 대체한다.

기본 사용

#include <optional>

std::optional<int> FindIndex(const std::vector<int>& v, int target)
{
    for (size_t i = 0; i < v.size(); ++i)
    {
        if (v[i] == target)
        {
            return static_cast<int>(i);
        }
    }
    return std::nullopt;  // 부재를 명시적으로 반환
}

auto idx = FindIndex(v, 42);
if (idx)                  // 또는 idx.has_value()
{
    std::cout << *idx;    // 또는 idx.value()
}

주요 API

std::optional<std::string> opt = "hello";

opt.has_value();          // true
opt.value();              // "hello" — 비었으면 bad_optional_access 던짐
*opt;                     // "hello" — 비었으면 UB (체크 안 함)
opt->size();              // 5
opt.value_or("default");  // 비었으면 "default" 반환

opt.reset();              // 비움
opt = std::nullopt;       // 비움
opt = "world";            // 값 대입

auto opt2 = std::make_optional<Image>(800, 600);  // 제자리 생성
비어있는 optional에 *opt는 정의되지 않은 동작 체크 없이 역참조하면 UB다. 안전을 원하면 opt.value()를 써서 예외(bad_optional_access)로 감지하거나, if (opt)로 먼저 검사한다.

C++23 monadic 연산

중첩된 if 체크 대신 함수형 스타일로 체이닝할 수 있다.

// transform: 값이 있을 때만 변환, 없으면 그대로 비어있음
std::optional<int> len = FindName(id)
    .transform([](const std::string& s) { return static_cast<int>(s.size()); });

// and_then: optional을 반환하는 함수와 체이닝 (flatMap)
auto result = ParseInt(text)
    .and_then([](int n) { return n > 0 ? std::optional{n * 2} : std::nullopt; });

// or_else: 비었을 때 대체 optional 제공
auto v = LookupCache(key)
    .or_else([&] { return LoadFromDisk(key); });
참조는 직접 담을 수 없다 std::optional<T&>는 표준이 아니다 (C++26에서 추가 예정). 참조가 필요하면 std::optional<std::reference_wrapper<T>> 또는 포인터를 쓴다.

std::variant<T1, T2, ...>

나열된 타입들 중 정확히 하나를 안전하게 담는다. C 시절의 tagged union을 타입 안전하게 만든 것이다. 컴파일 타임에 가능한 타입 집합이 정해진다.

기본 사용

#include <variant>

std::variant<int, double, std::string> v;

v = 42;          // int 보유, index() == 0
v = 3.14;        // double 보유, index() == 1
v = "hello";     // string 보유, index() == 2

v.index();       // 현재 보유 alternative의 인덱스

std::holds_alternative<double>(v);   // true
std::get<std::string>(v);            // "hello"  (잘못된 타입이면 예외)
std::get<2>(v);                      // "hello"  (인덱스로도 가능)

if (auto* p = std::get_if<int>(&v)) // 포인터 반환 — 잘못된 타입이면 nullptr
{
    std::cout << *p;
}
std::get은 잘못된 타입에 접근하면 예외 보유 중이 아닌 alternative를 꺼내면 bad_variant_access가 던져진다. 분기마다 try/catch 대신 get_if 또는 visit을 쓰는 편이 깔끔하다.

std::visit + overload 패턴

모든 alternative를 빠짐없이 처리하는 가장 안전한 방법이다. 컴파일러가 누락된 분기를 잡아준다.

// overload 헬퍼 — 람다 여러 개를 하나의 함수 객체로 결합
template<class... Ts> struct overload : Ts... { using Ts::operator()...; };
template<class... Ts> overload(Ts...) -> overload<Ts...>;

std::variant<int, double, std::string> v = "hello";

std::visit(overload{
    [](int i)                  { std::cout << "int: " << i; },
    [](double d)               { std::cout << "double: " << d; },
    [](const std::string& s)   { std::cout << "string: " << s; }
}, v);
실행 결과
string: hello

std::monostate — "비어있음" 표지

variant의 첫 번째 alternative가 기본 생성 가능하지 않으면 컴파일 에러다. std::monostate를 첫 자리에 두면 "아직 어떤 값도 없음"을 표현할 수 있다.

struct NonDefault { NonDefault(int) {} };  // 기본 생성자 없음

// std::variant<NonDefault, int> v;  // 컴파일 에러
std::variant<std::monostate, NonDefault, int> v;  // OK, 기본은 monostate

if (std::holds_alternative<std::monostate>(v))
{
    std::cout << "아직 비어있음";
}

std::any

어떤 타입이든 하나를 담는다. 내부적으로 타입 소거(type erasure)를 사용해 원래 타입 정보를 런타임에 들고 있다가 캐스트할 때 일치 여부를 검사한다.

#include <any>

std::any a;
a = 42;
a = std::string("hello");
a = std::vector<int>{1, 2, 3};   // 어떤 타입이든 OK

a.has_value();       // true
a.type() == typeid(std::vector<int>);  // true

// 꺼내기 — 타입을 정확히 알아야 함
auto v = std::any_cast<std::vector<int>>(a);  // 복사
auto& r = std::any_cast<std::vector<int>&>(a); // 참조
auto* p = std::any_cast<std::vector<int>>(&a); // 포인터 — 실패 시 nullptr

a.reset();           // 비움
잘못된 타입으로 any_cast하면 예외 값/참조 형태의 캐스트는 실패 시 bad_any_cast를 던진다. 포인터 형태(any_cast<T>(&a))는 실패 시 nullptr을 반환하니 분기 코드에서 더 편하다.
저장 비용 — SBO와 힙 구현체마다 다르지만 작은 객체(보통 포인터 2~3개 크기)는 any 내부 버퍼에 저장하는 SBO(Small Buffer Optimization)를 적용하고, 큰 객체는 힙에 할당한다. 즉 자주 큰 객체를 담을수록 비용이 커진다.

실무 적용 (ImagingLib)

ErrorCode + 출력 매개변수 vs optional 반환

ImagingLib 규칙은 action 함수가 ErrorCode를 반환하고 결과는 출력 매개변수로 받는 형태다. 단순 조회는 optional로 표현하면 의도가 더 또렷해진다.

// 기존: 출력 매개변수 패턴
ErrorCode GetArea(const Region& region, double& outArea);

// 단순 조회는 optional이 더 자연스럽다
std::optional<double> TryGetArea(const Region& region);

// 호출 측
if (auto area = TryGetArea(region))
{
    Logger::Info("area = {}", *area);
}

처리 결과를 variant로 — Image / Region / XLD

블록의 출력은 종류가 정해져 있다. variant로 묶으면 다음 블록이 visit로 안전하게 분기할 수 있다.

using BlockResult = std::variant<std::monostate, Image, Region, Xld>;

BlockResult ExecuteBlock(const IProcessBlock& block);

void Render(const BlockResult& r)
{
    std::visit(overload{
        [](std::monostate)        { /* 결과 없음 */ },
        [](const Image& img)      { DrawImage(img); },
        [](const Region& rgn)     { DrawRegion(rgn); },
        [](const Xld& xld)        { DrawXld(xld); }
    }, r);
}

블록 파라미터를 any로

플러그인 블록의 파라미터는 종류가 무한하다. 키-값 맵의 값 타입을 any로 두면 어떤 블록이든 자기 파라미터를 자유롭게 정의할 수 있다.

using ParamMap = std::unordered_map<std::string, std::any>;

ErrorCode ApplyParams(const ParamMap& params)
{
    if (auto* k = std::any_cast<int>(&params.at("kernelSize")))
    {
        m_kernelSize = *k;
    }
    else
    {
        return ErrorCode::InvalidParamType;
    }
    return ErrorCode::Ok;
}

C++23 std::expected 미리 보기

std::expected<T, E>는 "성공 시 T, 실패 시 E"를 담는 타입이다. optional은 실패 이유를 알려주지 못하지만 expected는 에러 정보를 함께 전달한다. ImagingLib의 ErrorCode 패턴과 자연스럽게 맞아떨어진다.

#include <expected>

std::expected<Image, ErrorCode> LoadImage(const std::string& path)
{
    if (!std::filesystem::exists(path))
    {
        return std::unexpected(ErrorCode::FileNotFound);
    }
    return Image::FromFile(path);
}

auto result = LoadImage("a.png");
if (result)
{
    Process(*result);              // 성공값 접근
}
else
{
    Logger::Error("load failed: {}", static_cast<int>(result.error()));
}

// monadic 체이닝도 지원
auto pipeline = LoadImage(path)
    .and_then([](Image img) { return ApplyBlur(img, 5); })
    .and_then([](Image img) { return Threshold(img, 128); });
언제 쓸까 ErrorCode를 반환하면서 결과까지 함께 돌려주고 싶을 때, 출력 매개변수 대신 expected를 쓰면 호출 코드가 훨씬 간결해진다. C++23 컴파일러가 보편화되면 ImagingLib의 새 API에 우선 검토할 가치가 있다.
