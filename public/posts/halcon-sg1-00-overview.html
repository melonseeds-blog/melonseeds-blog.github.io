전체 구조

HALCON Solution Guide I은 "특정 알고리즘 하나"를 설명하는 문서가 아니라, HALCON을 사용해 머신비전 문제를 풀 때 필요한 주요 방법들을 큰 흐름으로 연결하는 안내서입니다.

크게 보면 다음 순서입니다.

Image Acquisition
→ ROI / Domain 제어
→ Segmentation / Blob
→ Edge / Contour / Measuring
→ Matching / Recognition
→ Calibration / 3D / Robot Vision
→ OCR / Codes / Inspection / Deep Learning
→ 성능 최적화와 실무 설계

이 문서를 읽는 핵심 관점

1. Operator 이름보다 "처리 단위"를 먼저 이해해야 합니다

HALCON에서 자주 혼동되는 처리 단위는 다음과 같습니다.

처리 단위 의미 주 사용처
Image 실제 gray/color/depth 데이터 필터링, edge, matching
Region 픽셀 집합, binary mask 성격 Blob, ROI, 영역 특징
XLD Contour subpixel contour 정밀 측정, shape 분석
Tuple 수치/문자열/handle 배열 결과값, 파라미터, object handle
Handle 모델/장치/측정 객체의 참조 acquisition, matching, metrology

실무에서는 Image만 잘 다루는 것으로 부족합니다. Region과 XLD를 언제 바꿔 쓰는지가 HALCON 활용 능력의 핵심입니다.

2. ROI는 단순 속도 최적화가 아니라 "검사 안정성 구조"입니다

ROI를 줄이면 속도가 빨라지는 것은 당연합니다. 하지만 더 중요한 것은 다음입니다.

  • 검사 대상 외 영역을 제거해 오검출 감소
  • 조명 불균일, 배경 패턴, 주변 부품의 영향 감소
  • matching, measuring, blob 파라미터를 좁은 조건에서 안정화
  • 이후 world coordinate 변환 및 판정 기준을 명확화

즉, 실무 설계는 보통 다음 구조가 됩니다.

기준 위치 찾기
→ 기준 좌표계 생성
→ ROI를 기준 좌표계에 맞춰 이동/회전
→ ROI 내부에서만 검사/측정

3. Segmentation은 모든 검사 품질의 병목입니다

Blob, measuring, classification, print inspection은 대부분 어떤 방식으로든 segmentation 또는 edge extraction 결과에 의존합니다.

좋은 segmentation의 조건은 다음입니다.

  • 조명 변화에 둔감
  • 배경과 대상 간 contrast 충분
  • 반복 취득 시 region topology가 크게 바뀌지 않음
  • 후처리 morphology가 과도하지 않음
  • feature threshold가 margin을 충분히 가짐
억지 보정의 위험 실패한 segmentation을 뒤에서 feature, classifier, rule로 억지 보정하면 시스템이 취약해집니다.

4. 정밀 측정은 calibration과 edge 품질이 핵심입니다

픽셀 단위 길이는 실제 물리 길이가 아닙니다. 정밀 측정에서는 다음 4가지가 함께 맞아야 합니다.

  1. 렌즈 왜곡 보정
  2. pixel-to-world 변환
  3. subpixel edge 추출
  4. 반복성 있는 조명/초점/노출
흔한 오해 특히 측정 시스템에서 자주 생기는 오해는 "subpixel operator를 쓰면 자동으로 정밀해진다"는 것입니다. 실제로는 edge profile이 안정적이지 않으면 subpixel 결과도 불안정합니다.

5. Matching은 "모델 생성 품질"이 절반 이상입니다

Shape-based matching, NCC, descriptor, deformable matching 모두 공통적으로 모델이 중요합니다.

모델 생성 시 확인해야 할 항목:

  • template ROI가 객체만 포함하는가
  • 배경 edge가 모델에 들어가지 않았는가
  • 회전/스케일 범위를 과도하게 열지 않았는가
  • model pyramid level이 실제 객체 크기에 맞는가
  • score threshold가 양품/불량/부분가림을 구분할 수 있는가
  • 찾은 pose를 후속 ROI 정렬에 사용할 수 있는가

중요한 읽기 방식

이 문서는 "operator 암기용"이 아니라 "설계 판단용"입니다. 각 장에서 아래 질문을 계속 던지며 읽으면 좋습니다.

  • 이 방법은 Image / Region / XLD 중 무엇을 입력으로 쓰는가?
  • 결과는 pixel 좌표인가, world 좌표인가?
  • 위치 변화가 있으면 어떻게 정렬할 것인가?
  • 조명 변화에 얼마나 강한가?
  • 실패했을 때 원인을 이미지에서 확인할 수 있는가?
  • 속도 병목은 acquisition, preprocessing, search, feature extraction 중 어디인가?
학습 팁 HALCON Solution Guide I을 읽을 때, 각 챕터의 operator 목록을 외우려 하기보다 "어떤 문제 상황에서 어떤 처리 블록을 선택할 것인가"를 의식적으로 정리하면 실무 설계 감각이 빠르게 생깁니다.
