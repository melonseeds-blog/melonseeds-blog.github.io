3. 효율적 공부 전략

3.1 우선순위 전략

Priority 1 — 투자 대비 효과 최고 (먼저 공부)

  • Ch4 테스트 기법 (11문항, 27.5%) — EP, BVA, 결정 테이블, 상태 전이, 구문/분기 커버리지, 경험 기반 기법
  • Ch1 테스팅의 기초 (8문항, 20%) — 7가지 원리, 테스트 목적, 오류/결함/장애 체인, 테스트 활동

이 두 챕터만 완벽히 익혀도 19문항(47.5%)을 확보할 수 있습니다.

Priority 2 — 합격을 확정짓는 챕터

  • Ch5 테스트 관리 (8문항, 20%) — 리스크 관리, 테스트 계획, 메트릭, 결함 보고서
  • Ch2 SDLC와 테스팅 (6문항, 15%) — 테스트 레벨, 확인/리그레션 테스팅, 시프트 레프트

Priority 3 — 적은 비중, 가성비 학습

  • Ch3 정적 테스팅 (4문항, 10%) — 리뷰 유형, 리뷰 프로세스, 정적 vs 동적
  • Ch6 테스트 도구 (3문항, 7.5%) — 자동화 장점/리스크 (K1 수준, 암기만 하면 됨)

3.2 K-레벨별 학습법

ISTQB FL 시험은 세 가지 인지 수준(K-레벨)으로 출제됩니다. 각 레벨에 맞는 학습법이 다릅니다.

K-레벨요구 능력학습법해당 문항
K1 (기억) 용어, 개념 암기 플래시카드, 반복 읽기, 용어집 정리 약 12~14문항
K2 (이해) 개념을 설명할 수 있음 "왜?" "어떻게?"를 자문하며 학습. 개념 간 비교 정리 약 18~20문항
K3 (적용) 실제 계산/도출 가능 EP/BVA/결정 테이블/상태 전이/커버리지 계산 반복 연습 약 8~10문항
K3 문제가 승부처 K1/K2는 실러버스를 꼼꼼히 읽으면 대부분 맞출 수 있습니다. K3 문제는 직접 계산하고 도출해야 하므로, EP/BVA/커버리지 계산을 반복 연습하는 것이 합격의 열쇠입니다.

3.3 공부 타임라인 (2주 플랜)

D-14 — 실러버스 1회독

Ch1부터 Ch6까지 전체를 한 번 읽습니다. 이해가 안 되는 부분은 표시만 하고 넘어갑니다. 전체 구조를 파악하는 것이 목표입니다.

D-10 — Ch4 집중 학습

동등 분할(EP), 경계값 분석(BVA), 결정 테이블, 상태 전이 테스팅, 구문/분기 커버리지를 계산 문제 풀이와 함께 집중 학습합니다.

D-7 — Ch5 + Ch1 집중 학습

리스크 관리, 메트릭, 결함 보고서, 테스트 피라미드(Ch5)와 테스팅의 7가지 원리, 테스트 목적(Ch1)을 정리합니다.

D-5 — 모의고사 1회 + 오답 분석

시간을 재고 65분 안에 40문항을 풀어봅니다. 틀린 문제를 분석해 취약 영역을 파악합니다.

D-3 — 모의고사 2회 + 취약 챕터 보완

두 번째 모의고사 후 오답 패턴을 분석합니다. 반복적으로 틀리는 영역을 집중 보완합니다.

D-1 — 벼락치기 요약 복습

아래 벼락치기 1페이지 요약을 반복 읽습니다. 핵심 용어를 최종 점검하고, 일찍 잠자리에 듭니다.

D-day — 시험 당일

컨디션 관리가 최우선. 아침에 벼락치기 요약을 한 번 더 읽고, 시험장에는 30분 전에 도착합니다.

4. 시험 당일 전략

4.1 시간 배분

단계시간전략
1회독 50분 확실한 문제부터 풀기. 모르는 문제는 별표 표시 후 넘기기. 1문항당 약 1분 15초 배분.
2회독 10분 별표 표시한 문제 재검토. 2개로 좁혀진 문제는 직감을 따르기.
최종 확인 5분 미답 문항 확인. 마킹 오류 체크. 모든 문항에 답 표기 확인.
절대 빈칸을 남기지 마세요! 4지선다이므로 찍어도 25% 확률입니다. 모르는 문제도 반드시 하나를 선택하세요. 빈칸은 0%, 찍기는 25%입니다.

4.2 문제 풀이 팁

선지 판별 요령

  • "항상", "반드시", "절대", "모든 경우" → 대부분 오답 (테스팅에서 절대적인 것은 거의 없음)
  • "~할 수 있다", "~에 도움이 된다", "일반적으로" → 정답 가능성 높음
  • NOT / "해당하지 않는 것" / "제외" → 밑줄 긋고 천천히 읽기. 부정 조건을 놓치면 역전됨
  • K3 계산 문제 (EP, BVA, 커버리지) → 시간이 오래 걸리므로 1회독 때 먼저 풀기
  • 2개까지 좁혀졌을 때 → 첫 직감을 따르기 (변경 시 오답 확률이 더 높음)

실러버스 키워드 패턴

  • ISTQB 시험은 실러버스 원문 표현을 그대로 출제하는 경우가 많습니다
  • 실러버스에서 굵은 글씨, 이탤릭, 목록으로 나열된 항목은 출제 확률이 높습니다
  • 각 장의 "학습 목표(LO)"에 명시된 K-레벨이 곧 문제의 난이도와 유형입니다
  • K1이면 "상기할 수 있다" → 단순 암기 문제
  • K2이면 "설명할 수 있다" / "구별할 수 있다" → 비교/이해 문제
  • K3이면 "도출할 수 있다" / "활용할 수 있다" → 계산/적용 문제

4.3 자주 나오는 함정 패턴

함정 패턴 TOP 7

  • 7가지 원리 함정: 원리의 이름과 설명을 바꿔서 출제. 예: "조기 테스팅" 설명에 "결함 집중" 내용을 섞어 놓음
  • EP/BVA 경계값 off-by-one: 경계를 포함하는지 아닌지에 따라 답이 달라짐. 2-value BVA(경계값 + 인접값)와 3-value BVA(경계값 + 양쪽 인접값)를 구분
  • 구문 커버리지 vs 분기 커버리지 혼동: 100% 분기 커버리지 → 100% 구문 커버리지 보장. 반대는 성립하지 않음!
  • 리뷰 유형 혼동: 비공식 리뷰 → 워크스루 → 기술 리뷰 → 인스펙션 (공식성 순서). 워크스루는 저자가 리더, 인스펙션은 가장 공식적
  • 확인 테스팅 vs 리그레션 테스팅: 확인 = 결함 수정 확인, 리그레션 = 변경으로 인한 부작용 확인
  • 결정 테이블 규칙 수: 전체 규칙 수 = 2n (n=조건 수). 최소화 후 규칙 수를 묻는 문제가 출제됨
  • Verification vs Validation: Verification = "올바르게 만들고 있는가" (프로세스), Validation = "올바른 것을 만들고 있는가" (제품). 매번 헷갈리는 단골 출제

5. D-day 체크리스트

  • 준비물: 신분증 (주민등록증/운전면허증/여권), 수험표 (출력 또는 캡처)
  • 도착: 시험 시작 30분 전까지 시험장 도착
  • 마인드셋: 26/40만 맞추면 됨. 14개 틀려도 합격! 긴장하지 말 것
  • 1회독: 쉬운 문제부터 빠르게 풀기. 어려운 문제는 별표 후 넘기기
  • 2회독: 별표 문제 재검토. 2개로 좁혔으면 직감 따르기
  • 마지막 5분: 미답 문항 확인. 빈칸 절대 남기지 않기!
  • NOT 문제: "해당하지 않는 것"을 물어보는 문제에 특히 주의
  • K3 계산: EP/BVA/커버리지 문제는 차분하게 그림 그려가며 풀기
시험 당일 아침 루틴 아래 벼락치기 요약을 시험 전 한 번 더 훑어보세요. 특히 7가지 원리, 테스트 레벨 4가지, 리뷰 유형 4가지, EP/BVA 공식은 마지막까지 머릿속에 넣어두는 것이 좋습니다.

6. 벼락치기 1페이지 요약

시험 직전에 빠르게 훑어볼 수 있도록 각 챕터의 핵심만 정리했습니다. 프린트해서 시험장 가는 길에 읽어보세요.

ISTQB FL 벼락치기 요약

Ch1. 테스팅의 기초 (8문항)
  • 테스트 목적 5가지: 작업 산출물 평가, 장애 유발 및 결함 식별, 커버리지 확보, 이해관계자 정보 제공, 품질 신뢰 획득
  • 7가지 원리:
    1. 결함 존재를 밝힘 (부재 증명 X)
    2. 완벽한 테스팅 불가능
    3. 조기 테스팅 = 비용 절감
    4. 결함 집중 (파레토 원리)
    5. 살충제 패러독스 (같은 테스트 반복 → 효과 감소)
    6. 테스팅은 정황에 의존적
    7. 부재의 오류 (결함 없음 ≠ 성공)
  • Error → Defect → Failure (사람의 실수 → 산출물의 결함 → 실행 시 장애)
  • Verification = 올바르게 만드는가 (프로세스) / Validation = 올바른 것을 만드는가 (제품)
  • 테스트 활동: 계획 → 모니터링&제어 → 분석 → 설계 → 구현 → 실행 → 완료
Ch2. SDLC와 테스팅 (6문항)
  • 테스트 레벨 5가지:
    1. 컴포넌트(단위) 테스팅
    2. 컴포넌트 통합 테스팅
    3. 시스템 테스팅
    4. 시스템 통합 테스팅
    5. 인수 테스팅
  • 테스트 유형 4가지: 기능 / 비기능 / 블랙박스 / 화이트박스
  • 확인 테스팅 = 결함이 제대로 수정되었는지 확인
  • 리그레션 테스팅 = 변경으로 부작용이 없는지 확인 (자동화 적합)
  • 시프트 레프트: 테스팅을 SDLC 초기에 시작 → 비용 절감
Ch3. 정적 테스팅 (4문항)
  • 리뷰 유형 4가지 (공식성 순서):
    1. 비공식 리뷰 (프로세스 없음)
    2. 워크스루 (저자가 리더)
    3. 기술 리뷰 (중재자가 리더)
    4. 인스펙션 (가장 공식적, 저자는 리더/서기 불가)
  • 리뷰 프로세스 5단계: 계획 → 리뷰 착수 → 개별 리뷰 → 논의&분석 → 수정&보고
  • 정적 vs 동적:
    • 정적 = 실행 없이 결함 직접 식별
    • 동적 = 실행하여 장애를 통해 결함 식별
  • 정적 테스팅은 비-실행 산출물(요구사항, 설계 문서)에도 적용 가능
Ch4. 테스트 기법 (11문항) ★최다출제
  • 동등 분할(EP): 데이터를 동일 처리 그룹으로 분할. 각 분할에서 1개만 테스트. 커버리지 = 실행 분할 / 총 분할
  • 경계값 분석(BVA):
    • 2-value: 경계값 + 인접 분할 값 (경계당 2개)
    • 3-value: 경계값 + 양쪽 이웃값 (경계당 3개, 더 엄격)
  • 결정 테이블: 전체 규칙 수 = 2n (n=조건 수). 최소화로 줄일 수 있음
  • 상태 전이: 커버리지 3종 — 모든 상태 < 유효 전이(0-스위치) < 모든 전이
  • 구문 커버리지 = 실행된 구문 / 전체 실행 가능 구문
  • 분기 커버리지 = 실행된 분기 / 전체 분기. 100% 분기 → 100% 구문 (역은 불성립)
  • 경험 기반 3가지: 오류 추정, 탐색적 테스팅, 체크리스트 기반 테스팅
Ch5. 테스트 관리 (8문항)
  • 제품 리스크 vs 프로젝트 리스크:
    • 제품 리스크 = 품질 특성 관련 (기능 오류, 성능 문제, 보안 취약점)
    • 프로젝트 리스크 = 관리 관련 (일정 지연, 예산 초과, 인력 부족)
  • 리스크 수준 = 발생 가능성 × 영향(피해)
  • 테스트 피라미드: Unit(많음, 빠름) > Integration > E2E(적음, 느림). 아래층일수록 테스트 수가 많고 빠름
  • 추정 기법 4가지: 비율 기반, 외삽법, 와이드밴드 델파이(플래닝 포커), 3점 추정 E=(a+4m+b)/6
  • 우선순위 3가지: 리스크 기반, 커버리지 기반, 요구사항 기반
  • 결함 보고서 필수 항목: 고유 식별자, 제목, 날짜, 테스트 환경, 재현 절차, 기대 결과, 실제 결과, 심각도, 우선순위, 상태
  • 메트릭 종류: 진행 상황, 품질, 결함, 리스크, 커버리지, 비용
Ch6. 테스트 도구 (3문항)
  • 자동화 장점 5가지:
    1. 반복 수작업 감소 → 시간 절약
    2. 일관성 및 재현성 향상
    3. 객관적인 평가(커버리지 등) 제공
    4. 테스트 정보 접근 용이
    5. 테스트 실행 시간 단축 → 결함 조기 식별
  • 자동화 리스크 5가지:
    1. 도구 효과에 대한 비현실적 기대
    2. 도입 시간/비용에 대한 부정확한 추정
    3. 수동 테스팅이 더 적합한 곳에 자동화 적용
    4. 도구에 대한 과도한 의존 (비판적 사고 무시)
    5. 도구 공급업체 종속성 (폐업, 지원 중단 등)
