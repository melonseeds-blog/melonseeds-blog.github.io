함수 객체란?

함수 객체(functor)는 operator()를 오버로드하여 함수처럼 호출할 수 있는 클래스 인스턴스다. 일반 함수처럼 obj(args) 형태로 호출되지만 객체이기 때문에 상태(멤버 변수)를 보유할 수 있다.

struct Adder
{
    int m_base;
    Adder(int base) : m_base(base) {}

    int operator()(int x) const
    {
        return m_base + x;
    }
};

Adder add10(10);
int result = add10(5);   // 15

함수 포인터와의 차이

항목함수 포인터함수 객체
상태 보유불가가능 (멤버 변수)
인라인 최적화어려움가능 (타입이 컴파일 타임에 확정)
타입 정보시그니처만고유 타입
오버헤드간접 호출 비용대부분 zero-cost
왜 STL 알고리즘이 함수 객체를 선호하는가 std::sort는 비교자 타입을 템플릿 파라미터로 받기 때문에 함수 객체의 operator()가 인라인되어 함수 포인터보다 빠르다.

STL 표준 함수 객체

<functional> 헤더에 자주 쓰는 연산이 미리 정의되어 있다.

분류함수 객체의미
산술std::plus<T>, std::minus<T>+, -
std::multiplies<T>, std::negate<T>*, 단항 -
비교std::less<T>, std::greater<T>, std::equal_to<T><, >, ==
논리std::logical_and<T>, std::logical_or<T>, std::logical_not<T>&&, ||, !
#include <functional>
#include <algorithm>
#include <vector>

std::vector<int> v = {3, 1, 4, 1, 5};
std::sort(v.begin(), v.end(), std::greater<int>());  // 내림차순

int sum = std::accumulate(v.begin(), v.end(), 0, std::plus<int>());

Transparent 함수 객체 (C++14)

템플릿 인자를 비워두면 인자 타입을 자동 추론한다. heterogeneous lookup이 가능해 효율적이다.

// C++11
std::sort(v.begin(), v.end(), std::greater<int>());

// C++14 transparent — 타입 추론
std::sort(v.begin(), v.end(), std::greater<>());

// std::set<std::string, std::less<>>는 const char* 키로도 find 가능
std::set<std::string, std::less<>> s;
s.find("hello");  // string 임시객체 생성 안 함

람다 표현식

람다는 익명 함수 객체를 그 자리에서 만드는 문법이다. 컴파일러는 람다를 고유한 익명 클래스로 변환한다.

기본 문법

// [캡처](파라미터) -> 반환타입 { 본문 }
auto square = [](int x) -> int { return x * x; };
int n = square(5);  // 25

// 반환 타입 생략 (자동 추론)
auto add = [](int a, int b) { return a + b; };

캡처 종류

캡처의미
[]아무것도 캡처 안 함
[=]외부 변수 모두 값으로 캡처
[&]외부 변수 모두 참조로 캡처
[var]특정 변수 값 캡처
[&var]특정 변수 참조 캡처
[=, &x]x만 참조, 나머지 값
[&, x]x만 값, 나머지 참조
[this]멤버 함수 안에서 객체 포인터 캡처
[*this]객체 자체를 값으로 복사 (C++17)
int x = 10, y = 20;

auto byVal = [x, y]() { return x + y; };           // 값
auto byRef = [&x, &y]() { x++; y++; };            // 참조
auto mixed = [=, &y]() { y += x; };                // x 값, y 참조

class Foo
{
    int m_value = 42;
public:
    auto getLambda() { return [this]() { return m_value; }; }
};

이동 캡처 (C++14)

C++14의 init capture로 std::unique_ptr 같은 이동 전용 타입도 캡처할 수 있다.

auto p = std::make_unique<int>(42);

// C++11에서는 unique_ptr 캡처 불가능
// C++14 init capture로 이동 캡처
auto lambda = [ptr = std::move(p)]()
{
    return *ptr;
};

mutable 람다

값으로 캡처한 변수는 람다 안에서 기본적으로 const다. mutable 키워드로 수정 가능하게 만든다.

int counter = 0;

auto next = [counter]() mutable
{
    return ++counter;  // mutable 없으면 컴파일 에러
};

next();  // 1
next();  // 2 — 람다 내부 복사본이 유지됨
// 외부 counter는 여전히 0

람다는 익명 클래스

컴파일러는 다음 람다를 대략 이렇게 변환한다.

// 람다
auto add = [base = 10](int x) { return base + x; };

// 컴파일러가 생성하는 코드 (개념적)
class __Lambda_Anon
{
    int base;
public:
    __Lambda_Anon(int b) : base(b) {}
    int operator()(int x) const { return base + x; }
};

std::function

std::function은 호출 가능한 모든 것(함수 포인터, 람다, 함수 객체, 멤버 함수)을 같은 타입으로 저장하는 타입 소거(type erasure) 래퍼다.

#include <functional>

std::function<int(int, int)> op;

op = [](int a, int b) { return a + b; };  // 람다
op = std::plus<int>();                    // 함수 객체
op = &someFunction;                        // 함수 포인터

int r = op(3, 4);
std::function의 비용 내부적으로 가상 호출과 비슷한 간접 호출이 일어나며, 캡처가 작은 버퍼(SBO)를 넘으면 힙 할당이 발생한다. 인라인이 거의 안 된다.

std::bind vs 람다

std::bind는 C++11에서 함수와 인자를 부분 적용하기 위해 도입되었다. 하지만 모던 C++에서는 거의 모든 경우 람다가 더 낫다.

#include <functional>
using namespace std::placeholders;

int multiply(int a, int b) { return a * b; }

// std::bind — _1은 첫 번째 호출 인자
auto times3 = std::bind(multiply, _1, 3);
times3(5);  // 15

// 같은 동작을 람다로
auto times3Lambda = [](int x) { return multiply(x, 3); };
times3Lambda(5);  // 15
람다를 쓰는 이유 가독성이 뛰어나고, 컴파일러가 인라인 최적화하기 쉽다. std::bind는 placeholder 의미를 추적해야 해서 코드가 어렵고 디버깅이 까다롭다. Scott Meyers도 Effective Modern C++에서 람다를 권장한다.

알고리즘 술어로 활용

STL 알고리즘은 비교자/술어를 함수 객체로 받는다. 람다가 가장 자연스러운 짝이다.

#include <algorithm>
#include <vector>

std::vector<int> v = {5, -3, 8, -1, 4};

// 절대값 기준 정렬
std::sort(v.begin(), v.end(), [](int a, int b)
{
    return std::abs(a) < std::abs(b);
});

// 첫 번째 음수 찾기
auto it = std::find_if(v.begin(), v.end(), [](int x)
{
    return x < 0;
});

// 양수 개수
int positives = std::count_if(v.begin(), v.end(), [](int x)
{
    return x > 0;
});

// 짝수 제거 (erase-remove 관용구)
v.erase(std::remove_if(v.begin(), v.end(), [](int x)
{
    return x % 2 == 0;
}), v.end());
비교자는 strict weak ordering을 만족해야 한다 a <= b처럼 같음을 포함하면 동치 관계가 깨진다. sort는 등가성 검사 시 !comp(a,b) && !comp(b,a)를 사용하므로 반드시 엄격한 미만(<)만 사용해야 한다. 위반 시 정의되지 않은 동작이다.

C++17/20 추가 기능

Generic 람다 (C++14)

파라미터 타입을 auto로 받는다. 컴파일러는 operator()를 템플릿으로 만든다.

auto print = [](const auto& x)
{
    std::cout << x << std::endl;
};

print(42);          // int
print("hello");     // const char*
print(3.14);        // double

constexpr 람다 (C++17)

constexpr auto square = [](int x) { return x * x; };

constexpr int v = square(5);  // 컴파일 타임 평가
static_assert(v == 25);

템플릿 파라미터 람다 (C++20)

generic 람다보다 명시적으로 타입 파라미터를 다룬다.

auto getSize = []<typename T>(const std::vector<T>& v)
{
    return v.size();
};

std::vector<int> vi = {1, 2, 3};
auto n = getSize(vi);

흔한 함정

1. Dangling reference

std::function<int()> makeBad()
{
    int local = 42;
    return [&local]() { return local; };  // 위험!
}

auto f = makeBad();
int x = f();  // UB — local은 이미 소멸
람다 수명이 캡처 대상보다 길어지면 UB 함수에서 람다를 반환하거나 비동기로 넘길 때 참조 캡처는 위험하다. 이런 경우는 값 캡처나 shared_ptr 캡처를 사용해야 한다.

2. mutable 누락 컴파일 에러

int n = 0;
auto bad = [n]() { n++; return n; };       // 컴파일 에러
auto good = [n]() mutable { n++; return n; };  // OK

3. this 캡처 후 객체 소멸

class Worker
{
    int m_id = 1;
public:
    auto makeTask()
    {
        // this 캡처 — Worker가 죽으면 UB
        return [this]() { return m_id; };

        // C++17: *this로 객체를 값 복사 (안전)
        // return [*this]() { return m_id; };
    }
};
비동기 콜백에서는 [*this]를 고려하라 멤버 함수 안에서 람다를 만들어 다른 스레드/이벤트 루프로 넘길 때, 객체 수명을 보장할 수 없다면 [*this] 또는 shared_from_this()를 활용한다.

4. std::function 남용

// 나쁨 — sort 호출마다 type erasure 비용
std::sort(v.begin(), v.end(),
    std::function<bool(int,int)>([](int a, int b) { return a < b; }));

// 좋음 — 람다를 직접 전달, 인라인 됨
std::sort(v.begin(), v.end(),
    [](int a, int b) { return a < b; });

// 콜백을 받는 함수도 가능하면 템플릿으로
template<typename F>
void process(F&& callback) { callback(); }
정리 한 곳에서만 호출/전달되면 람다 그대로, 다양한 callable을 변수에 저장해야 하면 std::function, 성능이 중요하면 템플릿 파라미터로 받는다.
